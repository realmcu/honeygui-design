# 代码生成架构总结（重构后）

## 一、整体架构

```
触发入口
  ├─ Webview 设计器 (generateCode 按钮)
  ├─ 命令面板 (honeygui.codegen)
  └─ 编译仿真 (自动触发)
         ↓
    CodeGenerator (服务层)
         ↓
    CodeGeneratorFactory (工厂)
         ↓
    HoneyGuiCCodeGenerator (具体实现)
         ↓
    生成的 C 代码文件
```

## 二、生成目录结构（重构后）

```
src/
├── ui/                      # UI 代码（每次覆盖）
│   ├── main_ui.h
│   ├── main_ui.c
│   ├── settings_ui.h
│   └── settings_ui.c
├── callbacks/               # 回调代码（保护区）
│   ├── main_callbacks.h
│   ├── main_callbacks.c
│   ├── settings_callbacks.h
│   └── settings_callbacks.c
├── user/                    # 用户代码（只生成一次）
│   ├── main.h
│   ├── main.c
│   ├── settings.h
│   └── settings.c
└── {ProjectName}Entry.c     # 项目入口文件
```

**优势**：
- 按类型分组，结构清晰
- 便于编译配置（可以分别指定编译选项）
- 便于版本控制（user/ 目录可以单独管理）

## 三、核心文件说明

### 1. 接口层
**`src/codegen/ICodeGenerator.ts`**
- 定义代码生成器接口
- 简化的配置选项：
  ```typescript
  interface CodeGenOptions {
    srcDir: string;           // src 根目录
    designName: string;       // 设计名称
    enableProtectedAreas?: boolean;
  }
  ```

### 2. 服务层
**`src/services/CodeGenerator.ts`**
- 职责：协调代码生成流程
- 功能：
  - 扫描 HML 文件
  - 构造生成配置
  - 调用工厂创建生成器
  - 收集结果和错误

### 3. 工厂层
**`src/codegen/CodeGeneratorFactory.ts`**
- 根据目标引擎创建生成器
- 支持：`honeygui`, `lvgl`

### 4. 实现层
**`src/codegen/honeygui/HoneyGuiCCodeGenerator.ts`**
- 生成 HoneyGUI C 代码
- 自动创建目录结构
- 处理 include 路径

### 5. 入口文件生成器
**`src/codegen/EntryFileGenerator.ts`**
- 生成项目入口文件
- 位置：`src/{ProjectName}Entry.c`

## 四、Include 路径规则

```c
// ui/main_ui.c
#include "main_ui.h"
#include "../callbacks/main_callbacks.h"

// callbacks/main_callbacks.c
#include "main_callbacks.h"
#include "../ui/main_ui.h"

// user/main.c
#include "main.h"
#include "../ui/main_ui.h"
```

## 五、重构改进点

### ✅ 已解决的问题

1. **目录结构优化**
   - 从嵌套结构改为按类型分组
   - 更清晰、更易维护

2. **配置简化**
   - 移除 `outputDir`, `hmlFileName`, `userCodeDir`
   - 统一使用 `srcDir` + `designName`

3. **职责清晰**
   - `CodeGenerator` 只负责协调
   - 具体生成逻辑在 `HoneyGuiCCodeGenerator`

4. **移除冗余**
   - 删除 `honeygui/index.ts` 的便捷函数
   - 统一使用工厂模式

5. **Include 路径自动化**
   - 生成器自动处理相对路径
   - 无需手动配置

## 六、调用流程

```
用户触发
  ↓
CodeGenerator.generate(projectRoot)
  ↓
扫描 ui/{designName}/*.hml
  ↓
对每个 HML：
  - 加载组件树
  - 构造 CodeGenOptions { srcDir, designName }
  - CodeGeneratorFactory.create()
  - generator.generate()
    ├─ 创建 src/ui/
    ├─ 创建 src/callbacks/
    ├─ 创建 src/user/
    ├─ 生成 UI 代码（覆盖）
    ├─ 生成回调代码（保护区）
    └─ 生成用户代码（只生成一次）
  ↓
生成 src/{ProjectName}Entry.c
  ↓
返回结果
```

## 七、Clean 功能

**注意**：不清理 `src/user/` 目录（用户代码）
