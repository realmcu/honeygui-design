# HoneyGUI 需求表（可累加）

本文件以体验者视角描述产品需求，不涉及实现方案。新增需求请复制下方模板，按编号递增（如 R-004）。

## 新增需求模板
- 编号：R-XXX
- 标题：一句话概括
- 背景/痛点：为什么需要它、用户困扰是什么
- 用户场景：在哪些操作或页面会用到它
- 功能要求：系统需要提供的能力（清晰、可验证）
- 交互规则：用户与界面如何互动、反馈如何呈现
- 约束与边界：不做或暂不做的内容、离线约束
- 验收标准：体验者可按步骤确认是否满足
- 优先级：P1/P2/P3（高/中/低）
- 状态：待评审/开发中/已完成

## 需求列表

### R-001 硬件交互接口预留（项目级）
- 背景/痛点：作为 GUI 界面生成器，项目需在产品层面预留统一的硬件交互接口，使界面元素能绑定“设备交互能力”并获得一致反馈，而不聚焦具体设备的控制细节。
- 用户场景：在设计器或预览中，体验者为按钮、开关等组件绑定通用的设备交互（例如：触发动作、查询状态、订阅事件），以便后续与任意设备映射。
- 功能要求：
  - 提供通用的设备交互能力类型：动作（触发）、查询（获取状态）、事件（接收通知）。
  - 在设备不可用或未就绪时，界面给出明确提示或模拟反馈，不中断主要流程。
  - 全程离线可用；交互过程支持取消与重试；错误信息中文且不遮挡操作。
  - 绑定目标可为设备、设备组或占位符（后续再映射到具体设备）。
- 交互规则：
  - 触发后显示进行中与最终结果（成功/失败），反馈样式统一、可辨识。
  - 支持单个与批量目标的选择与触发（如项目存在多设备）。
- 约束与边界：不限定设备品牌、类型与参数模型；不描述具体控制行为，仅要求具备统一绑定与反馈能力。
- 验收标准：
  - 体验者能在界面为元素绑定并触发一类设备交互，并获得清晰的结果提示。
  - 无实际设备时不崩溃，提供模拟或说明性反馈。
  - 离线可用，文案清晰，不暴露实现术语。
- 优先级：P2
- 状态：待评审

### R-002 画布交互：单击选中即可拖动
- 背景/痛点：当前需要“点击一下再移动”，不符合直觉，拖动效率低。
- 用户场景：在画布中对组件进行微调或布局调整。
- 功能要求：
  - 单击组件即选中；按住不松即可直接拖动，无需额外步骤。
  - 支持多选（如按住 Shift）与框选；支持取消选择（点击空白或按 Esc）。
  - 拖动过程中有明确的视觉反馈（选中态、对齐线、网格吸附如开启）。
  - 提供设置项，可关闭“单击即拖动”以兼容旧习惯。
- 交互规则：
  - 选中对象显示明显边框或手柄；拖动时位置变化实时可见。
  - 框选与多选时，反馈一致且易辨识。
- 约束与边界：不要求在本需求中处理旋转/变形，专注于移动体验。
- 验收标准：
  - 体验者无需切换到“移动”工具即可完成拖动操作。
  - 多选、框选、取消选择均工作正常，反馈清晰。
  - 拖动定位准确，对齐反馈正常。
- 优先级：P1
- 状态：待评审

### R-003 资源库统一管理与属性选择器
- 背景/痛点：属性面板需要直接填“图片路径”，易错且不便，无法预览与管理。
- 用户场景：体验者导入图片/字体等资源，在属性面板通过选择器引用。
- 功能要求：
  - 资源库面板：支持导入、预览、搜索、删除，资源以中文名称展示。
  - 属性选择器：在属性面板点击按钮弹出资源选择器，无需手输路径。
  - 支持分组/标签与显示引用状态；可识别未引用资源并清理。
  - 导出或构建时，资源被完整打包到项目资产目录，引用不丢失。
- 交互规则：
  - 导入后立即可预览缩略图与基础信息；选择后属性自动填充。
  - 清理操作需确认提示，避免误删在用资源。
- 约束与边界：本需求不限定资源文件的具体存储结构，仅关注用户导入、选择与清理体验。
- 验收标准：
  - 体验者能导入图片并在属性面板通过选择器完成引用。
  - 导出/构建结果包含所需资源，界面展示正常。
  - 可列出未引用资源并一键清理，过程有确认与结果反馈。
- 优先级：P1
- 状态：待评审
