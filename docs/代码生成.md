# HoneyGUI 代码生成

本文档描述 HoneyGUI Design 的代码生成功能，包括架构设计、使用方法和实现细节。

## 概述

从HML设计器生成调用HoneyGUI API的C代码，实现可视化设计到嵌入式GUI代码的转换。

## 代码生成流程

```
HML文件 → 解析器 → 组件树 → C代码生成器 → HoneyGUI API调用代码
```

## 架构设计（2025-12-01 更新）

### 核心组件

```
CodeGenerator (服务层)
  ├─ 扫描项目中的所有 HML 文件
  ├─ 生成项目入口文件 (EntryFileGenerator)
  ├─ 生成构建脚本 (SConscriptGenerator)
  └─ 为每个 HML 生成代码 (HoneyGuiCCodeGenerator)

EntryFileGenerator (独立工具)
  └─ 生成 {ProjectName}Entry.c

SConscriptGenerator (独立工具)
  └─ 生成 src/SConscript

HoneyGuiCCodeGenerator (核心生成器)
  ├─ 生成 {hmlFileName}_ui.h
  ├─ 生成 {hmlFileName}_ui.c
  ├─ 生成 {hmlFileName}_callbacks.h
  ├─ 生成 {hmlFileName}_callbacks.c (含保护区)
  ├─ 生成 {hmlFileName}_user.h
  └─ 生成 {hmlFileName}_user.c
```

### 关键变更

**v1.2.2 (2025-12-01)**：
- ✅ 移除"生成当前"功能，统一为"生成代码"
- ✅ 入口文件生成独立为 `EntryFileGenerator`
- ✅ 移除冗余的 `CodeGenManager` 中间层
- ✅ 简化命名：`BatchCodeGenerator` → `CodeGenerator`

## 文件命名和目录规则

### 命名规则
- 使用HML文件名作为生成文件的基础名称
- 例如：`main.hml` → `main.h`, `main.c`, `main_callbacks.h`, `main_callbacks.c`

### 目录映射
- HML文件在 `ui/xxx/` 目录 → C代码生成到 `src/` 目录的对应子目录
- 每个HML文件生成到独立的子目录
- 例如：`ui/main/main.hml` → `src/ui/main_ui.c`, `src/callbacks/main_callbacks.c`, `src/user/main_user.c`
- 自动创建不存在的子目录

### 生成文件结构
```
project/
├── ui/
│   ├── main/
│   │   └── main.hml
│   └── settings/
│       └── settings.hml
└── src/
    ├── {ProjectName}Entry.c    # 项目入口文件
    ├── SConscript              # 构建脚本
    ├── ui/                     # UI代码（每次覆盖）
    │   ├── main_ui.h
    │   ├── main_ui.c
    │   ├── settings_ui.h
    │   └── settings_ui.c
    ├── callbacks/              # 回调代码（保护区）
    │   ├── main_callbacks.h
    │   ├── main_callbacks.c
    │   ├── settings_callbacks.h
    │   └── settings_callbacks.c
    └── user/                   # 用户代码（只生成一次）
        ├── main_user.h
        ├── main_user.c
        ├── settings_user.h
        └── settings_user.c
```

## 使用方法

### 生成代码

1. 在设计器中打开任意 HML 文件
2. 点击工具栏的"生成代码"按钮
3. 自动扫描项目中所有 HML 文件并生成代码
4. 显示生成进度和结果

**生成内容**：
- 所有设计稿的 C 代码文件
- 项目入口文件 `{ProjectName}Entry.c`
- 构建脚本 `SConscript`

**快捷方式**：
- 命令面板：`Ctrl+Shift+P` → `HoneyGUI: Generate Code`
- 工具栏按钮：点击"生成代码"

## 代码保护区

### 保护区语法

```c
// HONEYGUI PROTECTED START [unique_id]
// 这里的代码在重新生成时会被保留
// HONEYGUI PROTECTED END [unique_id]
```

### 示例

**首次生成**：
```c
// HONEYGUI PROTECTED START [on_start_click]
void on_start_click(gui_obj_t *obj) {
    // TODO: 实现事件处理逻辑
}
// HONEYGUI PROTECTED END [on_start_click]
```

**用户修改后**：
```c
// HONEYGUI PROTECTED START [on_start_click]
void on_start_click(gui_obj_t *obj) {
    printf("Starting application...\n");
    init_app_state();
    load_user_settings();
}
// HONEYGUI PROTECTED END [on_start_click]
```

**重新生成后**：保护区内的代码被保留，其他代码更新。

## 批量代码生成

### 架构设计

**CodeGenerator** (`src/services/CodeGenerator.ts`)
- 扫描项目中的所有 HML 文件
- 执行批量代码生成
- 收集和报告生成结果
- 独立于 VSCode UI，可测试

### API

```typescript
class CodeGenerator {
    // 扫描项目中的所有 HML 文件
    scanHmlFiles(projectRoot: string): Promise<string[]>
    
    // 批量生成代码
    generate(
        projectRoot: string,
        onProgress?: (progress: GenerationProgress) => void
    ): Promise<GenerationResult>
}
```

### 数据结构

```typescript
interface GenerationResult {
    success: boolean;           // 是否全部成功
    successCount: number;       // 成功数量
    totalFiles: number;         // 生成的文件总数
    errors: Array<{             // 错误列表
        designName: string;
        error: string;
    }>;
}

interface GenerationProgress {
    current: number;            // 当前进度
    total: number;              // 总数
    designName: string;         // 当前处理的设计稿名称
}
```

### 使用示例

```typescript
const generator = new CodeGenerator();
const projectRoot = ProjectUtils.findProjectRoot(filePath);

const result = await generator.generate(projectRoot, (progress) => {
    console.log(`${progress.current}/${progress.total}: ${progress.designName}`);
});

if (result.success) {
    console.log(`成功生成 ${result.successCount} 个设计稿`);
} else {
    console.log(`部分失败: ${result.errors.length} 个错误`);
}
```

## 生成的代码结构

### 头文件 (.h)

```c
#ifndef MAIN_H
#define MAIN_H

#include "gui_api.h"

// 组件声明
extern gui_obj_t *mainView;
extern gui_obj_t *titleLabel;
extern gui_obj_t *startButton;

// 初始化函数
void main_init(void);

// 更新函数
void main_update(void);

#endif
```

### 实现文件 (.c)

```c
#include "main.h"
#include "main_callbacks.h"

// 组件定义
gui_obj_t *mainView = NULL;
gui_obj_t *titleLabel = NULL;
gui_obj_t *startButton = NULL;

void main_init(void) {
    // 创建组件
    mainView = gui_view_create(NULL, "mainView", 0, 0, 480, 272);
    titleLabel = gui_text_create(mainView, "titleLabel", 10, 10, 460, 40);
    startButton = gui_button_create(mainView, "startButton", 190, 200, 100, 40);
    
    // 设置属性
    gui_text_set(titleLabel, "欢迎使用");
    gui_button_set_text(startButton, "开始");
    
    // 绑定事件
    gui_obj_add_event_cb(startButton, on_start_click, GUI_EVENT_CLICKED, NULL);
}

void main_update(void) {
    // 更新逻辑
}
```

### 回调文件 (_callbacks.h / _callbacks.c)

```c
// main_callbacks.h
#ifndef MAIN_CALLBACKS_H
#define MAIN_CALLBACKS_H

#include "gui_api.h"

void on_start_click(gui_obj_t *obj);

#endif

// main_callbacks.c
#include "main_callbacks.h"

// HONEYGUI PROTECTED START [on_start_click]
void on_start_click(gui_obj_t *obj) {
    // TODO: 实现事件处理逻辑
}
// HONEYGUI PROTECTED END [on_start_click]
```

## 调用生成的代码

```c
#include "src/ui/main_ui.h"
#include "src/callbacks/main_callbacks.h"
#include "src/user/main_user.h"

int main(void) {
    // 初始化HoneyGUI
    gui_init();
    
    // 初始化界面
    main_init();
    
    // 主循环
    while (1) {
        gui_task();
        main_update();
    }
    
    return 0;
}
```

## 注意事项

1. **目录结构**：HML文件必须在 `ui/` 目录下的子目录中
2. **文件命名**：HML文件名应使用有效的C标识符
3. **保护区**：不要手动修改保护区标记
4. **重新生成**：修改HML后重新生成，用户代码会被保留
5. **多文件**：每个HML独立生成，互不影响
