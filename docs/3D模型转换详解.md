# 3D 模型转换详解：OBJ vs GLTF

## 一、格式对比

### OBJ 格式

**特点**：
- 文本格式，人类可读
- 几何数据在 .obj 文件中
- 材质定义在 .mtl 文件中
- 纹理图片为独立文件

**文件结构**：
```
earth_tria.obj      # 主文件（顶点、面、纹理坐标）
earth_tria.mtl      # 材质文件（颜色、纹理引用）
earth.png           # 纹理图片
```

**优点**：
- ✅ 简单易懂
- ✅ 广泛支持
- ✅ 易于手动编辑
- ✅ 文件独立，易于管理

**缺点**：
- ❌ 文件多，容易丢失
- ❌ 文本格式，文件较大
- ❌ 不支持动画、骨骼

### GLTF 格式

**特点**：
- JSON 描述 + 二进制数据
- 场景图结构
- 支持动画、骨骼、蒙皮
- 现代 3D 格式标准

**文件结构**：
```
Pikachu.gltf        # JSON 描述（场景、材质、引用）
Pikachu.bin         # 二进制数据（顶点、索引）
texture.png         # 纹理图片（可选）
```

**优点**：
- ✅ 现代标准格式
- ✅ 支持复杂特性
- ✅ 二进制数据，加载快
- ✅ 可嵌入数据（GLB）

**缺点**：
- ❌ 格式复杂
- ❌ 需要多个文件
- ❌ 不易手动编辑

## 二、OBJ 转换流程

### 2.1 文件关系

```
earth_tria.obj
  ↓ (mtllib earth_tria.mtl)
earth_tria.mtl
  ↓ (map_Kd earth.png)
earth.png
```

### 2.2 详细流程

#### 步骤 1：解析 OBJ 文件

**输入**：`earth_tria.obj`

**解析内容**：
```obj
# 顶点坐标
v -0.000000 -0.831470 -0.555570
v -0.000000 -0.555570 -0.831470
...

# 纹理坐标
vt 0.000000 0.000000
vt 0.062500 0.000000
...

# 法线
vn 0.0000 -0.8507 -0.5257
vn 0.0000 -0.5257 -0.8507
...

# 面（索引）
f 1/1/1 2/2/2 3/3/3
f 2/2/2 4/4/4 3/3/3
...

# 材质库引用
mtllib earth_tria.mtl

# 使用材质
usemtl 材质.007
```

**提取数据**：
```typescript
{
  vertices: [
    { x: -0.000000, y: -0.831470, z: -0.555570 },
    { x: -0.000000, y: -0.555570, z: -0.831470 },
    ...
  ],
  normals: [...],
  texcoords: [...],
  indices: [
    { vertex: 0, texcoord: 0, normal: 0 },
    { vertex: 1, texcoord: 1, normal: 1 },
    ...
  ],
  faceNumVerts: [3, 3, 3, ...],  // 每个面的顶点数
  materialIds: [0, 0, 0, ...]     // 每个面使用的材质 ID
}
```

#### 步骤 2：解析 MTL 文件

**输入**：`earth_tria.mtl`

**解析内容**：
```mtl
newmtl 材质.007
Ns 250.000000              # 高光指数
Ka 1.0 1.0 1.0            # 环境光
Kd 0.8 0.8 0.8            # 漫反射
Ks 0.5 0.5 0.5            # 镜面反射
Ke 0.0 0.0 0.0            # 自发光
Ni 1.500000               # 折射率
d 1.000000                # 透明度
illum 2                   # 光照模型
map_Kd earth.png          # 漫反射纹理 ⭐
```

**提取数据**：
```typescript
{
  ambient: { x: 1.0, y: 1.0, z: 1.0 },
  diffuse: { x: 0.8, y: 0.8, z: 0.8 },
  specular: { x: 0.5, y: 0.5, z: 0.5 },
  shininess: 250.0,
  dissolve: 1.0,
  illum: 2,
  texturePath: "earth.png"  // ⭐ 关键：纹理路径
}
```

#### 步骤 3：加载纹理

**关键代码**：
```typescript
// 从 MTL 获取纹理路径
const texturePath = material.texturePath;  // "earth.png"

// 转换为 .bin 路径
const binPath = texturePath.replace(/\.(png|jpe?g)$/i, '.bin');
// "earth.bin"

// 检查文件是否存在
if (fs.existsSync(binPath)) {
  // 读取 .bin 文件内容
  const textureData = fs.readFileSync(binPath);
  textures.push(textureData);
} else {
  console.warn(`纹理 bin 文件未找到: ${binPath}`);
  textures.push(Buffer.alloc(0));  // 空纹理
}
```

**重要**：
- ⚠️ 纹理图片必须**先转换**为 .bin 格式
- ⚠️ .bin 文件名必须与原图片名一致
- ⚠️ .bin 文件必须在同一目录

#### 步骤 4：生成二进制文件

**输出结构**：
```
[文件头 16 bytes]
  - Magic: 0x3344 ('3D')
  - ModelType: 0 (OBJ)
  - Version: 3
  - FileSize: 实际大小
  - FaceType: 0=矩形, 1=三角形, 2=混合
  - PayloadOffset: 16

[属性计数 24 bytes]
  - Vertices count: 482
  - Normals count: 527
  - Texcoords count: 559
  - Indices count: 2880
  - Faces count: 960

[顶点数据 482 * 12 bytes]
  - 每个顶点: float x, float y, float z

[法线数据 527 * 12 bytes]
  - 每个法线: float x, float y, float z

[纹理坐标 559 * 8 bytes]
  - 每个坐标: float u, float v

[索引数据 2880 * 12 bytes]
  - 每个索引: int32 vertex, int32 texcoord, int32 normal

[面顶点数 960 * 4 bytes]
  - 每个面的顶点数: int32

[材质 ID 960 * 4 bytes]
  - 每个面的材质: int32

[形状信息]
  - Shape count: 1
  - Shape offset: 0
  - Shape length: 960

[材质信息]
  - Material count: 1
  - Material data: 76 bytes
    - ambient (12 bytes)
    - diffuse (12 bytes)
    - specular (12 bytes)
    - transmittance (12 bytes)
    - emission (12 bytes)
    - shininess (4 bytes)
    - ior (4 bytes)
    - dissolve (4 bytes)
    - illum (4 bytes)

[纹理大小]
  - Texture count: 1
  - Texture size: 262152 bytes

[纹理数据]
  - Texture binary data: 262152 bytes ⭐
```

**文件大小计算**：
```
earth_tria.obj 转换结果：
  几何数据: 58,956 bytes
  纹理数据: 262,152 bytes
  总计: 321,108 bytes
```

### 2.3 完整示例

**原始文件**：
```
assets/
├── earth_tria.obj      76 KB
├── earth_tria.mtl     258 B
└── earth.png           51 KB (实际是 JPEG)
```

**转换步骤**：
```bash
# 1. 先转换纹理
earth.png → earth.bin (262 KB, RGB565 格式)

# 2. 转换模型
earth_tria.obj
  ├─ 解析 OBJ (482 顶点, 960 面)
  ├─ 解析 MTL (发现 map_Kd earth.png)
  ├─ 加载 earth.bin (262 KB)
  └─ 生成 desc_earth_tria.bin (321 KB)
```

**输出文件**：
```
build/assets/
├── earth.bin                  262 KB (纹理)
└── desc_earth_tria.bin        321 KB (模型+纹理)
```

## 三、GLTF 转换流程

### 3.1 文件关系

```
Pikachu_resize.gltf (JSON 描述)
  ↓ (buffers[0].uri)
Pikachu_resize.bin (几何数据)
```

### 3.2 详细流程

#### 步骤 1：解析 GLTF JSON

**输入**：`Pikachu_resize.gltf`

**JSON 结构**：
```json
{
  "asset": {
    "version": "2.0",
    "generator": "Blender"
  },
  "scene": 0,
  "scenes": [...],
  "nodes": [...],
  "meshes": [
    {
      "primitives": [
        {
          "attributes": {
            "POSITION": 0,    // 顶点位置 accessor
            "NORMAL": 1,      // 法线 accessor
            "TEXCOORD_0": 2   // 纹理坐标 accessor
          },
          "indices": 3,       // 索引 accessor
          "material": 0       // 材质索引
        }
      ]
    }
  ],
  "accessors": [
    {
      "bufferView": 0,
      "componentType": 5126,  // FLOAT
      "count": 482,
      "type": "VEC3",
      "max": [1.0, 1.0, 1.0],
      "min": [-1.0, -1.0, -1.0]
    },
    ...
  ],
  "bufferViews": [
    {
      "buffer": 0,
      "byteOffset": 0,
      "byteLength": 5784
    },
    ...
  ],
  "buffers": [
    {
      "byteLength": 116788,
      "uri": "Pikachu_resize.bin"  // ⭐ 外部 buffer
    }
  ],
  "materials": [...],
  "textures": [...],
  "images": [...]
}
```

#### 步骤 2：加载 Buffer 数据

**关键代码**：
```typescript
// 检查 buffer 文件
const bufferPath = path.join(baseDir, buffer.uri);
if (!fs.existsSync(bufferPath)) {
  throw new Error(
    `GLTF buffer file not found: ${buffer.uri}\n` +
    `Expected at: ${bufferPath}`
  );
}

// 读取二进制数据
const bufferData = fs.readFileSync(bufferPath);
```

**Buffer 类型**：

1. **外部 Buffer**（需要 .bin 文件）：
```json
"buffers": [{
  "byteLength": 116788,
  "uri": "Pikachu_resize.bin"  // 外部文件
}]
```

2. **嵌入式 Buffer**（不需要 .bin 文件）：
```json
"buffers": [{
  "byteLength": 116788,
  "uri": "data:application/octet-stream;base64,AAABAAIAA..."
}]
```

#### 步骤 3：通过 Accessor 读取数据

**Accessor 系统**：
```
Buffer (原始二进制)
  ↓ BufferView (指定范围)
  ↓ Accessor (指定类型和格式)
  ↓ 实际数据
```

**示例**：读取顶点位置
```typescript
// 1. 获取 accessor
const posAccessor = gltfData.accessors[primitive.attributes.POSITION];
// {
//   bufferView: 0,
//   componentType: 5126,  // FLOAT
//   count: 482,
//   type: "VEC3"
// }

// 2. 获取 bufferView
const bufferView = gltfData.bufferViews[posAccessor.bufferView];
// {
//   buffer: 0,
//   byteOffset: 0,
//   byteLength: 5784
// }

// 3. 从 buffer 读取数据
const buffer = buffers[bufferView.buffer];
const offset = (bufferView.byteOffset || 0) + (posAccessor.byteOffset || 0);

// 4. 解析数据
for (let i = 0; i < posAccessor.count; i++) {
  const x = buffer.readFloatLE(offset + i * 12 + 0);
  const y = buffer.readFloatLE(offset + i * 12 + 4);
  const z = buffer.readFloatLE(offset + i * 12 + 8);
  vertices.push({ x, y, z });
}
```

#### 步骤 4：处理材质和纹理

**材质**：
```json
"materials": [
  {
    "pbrMetallicRoughness": {
      "baseColorFactor": [1.0, 1.0, 1.0, 1.0],
      "baseColorTexture": {
        "index": 0  // 纹理索引
      }
    }
  }
]
```

**纹理**：
```json
"textures": [
  {
    "source": 0  // 图片索引
  }
],
"images": [
  {
    "uri": "texture.png"  // 纹理图片
  }
]
```

**加载纹理**：
```typescript
const texture = gltfData.textures[texIndex];
const image = gltfData.images[texture.source];
const imagePath = path.join(baseDir, image.uri);

// 查找 .bin 版本
const binPath = imagePath.replace(/\.(png|jpg)$/i, '.bin');
if (fs.existsSync(binPath)) {
  textures.push(fs.readFileSync(binPath));
}
```

#### 步骤 5：生成二进制文件

**输出结构**：与 OBJ 相同，但：
- ModelType: 1 (GLTF)
- FaceType: 通常是 1 (三角形)

### 3.3 完整示例

**原始文件**：
```
assets/
├── Pikachu_resize.gltf    203 KB (JSON)
└── Pikachu_resize.bin     115 KB (几何数据) ⭐ 必需！
```

**转换步骤**：
```bash
# 检查 .bin 文件
if (!exists(Pikachu_resize.bin)) {
  throw Error("GLTF buffer file not found");
}

# 转换
Pikachu_resize.gltf
  ├─ 解析 JSON
  ├─ 加载 Pikachu_resize.bin
  ├─ 通过 accessor 读取顶点、法线、索引
  ├─ 解析材质
  └─ 生成 desc_Pikachu_resize.bin (97 KB)
```

**输出文件**：
```
build/assets/
└── desc_Pikachu_resize.bin    97 KB
```

## 四、关键差异对比

| 特性 | OBJ | GLTF |
|------|-----|------|
| **文件数量** | 3 个（.obj + .mtl + 纹理） | 2 个（.gltf + .bin） |
| **几何数据** | 文本格式，在 .obj 中 | 二进制格式，在 .bin 中 |
| **材质定义** | .mtl 文件 | JSON 中的 materials |
| **纹理引用** | MTL 中的 map_Kd | JSON 中的 textures/images |
| **数据访问** | 直接解析文本 | 通过 accessor 系统 |
| **面类型** | 可以是三角形或矩形 | 通常是三角形 |
| **复杂特性** | 不支持 | 支持动画、骨骼等 |

## 五、常见问题和解决方案

### 问题 1：OBJ 模型没有纹理

**现象**：
```
转换成功，但模型文件很小，没有纹理数据
```

**原因**：
```
纹理图片未转换为 .bin 格式
```

**解决**：
```bash
# 1. 先转换纹理
ImageConverter.convert('earth.png', 'earth.bin')

# 2. 再转换模型
OBJConverter.convert('earth_tria.obj', 'desc_earth_tria.bin')
```

**验证**：
```
无纹理：desc_earth_tria.bin = 59 KB
有纹理：desc_earth_tria.bin = 321 KB (59 + 262)
```

### 问题 2：GLTF 转换失败 "buffer file not found"

**现象**：
```
Error: GLTF buffer file not found: Pikachu_resize.bin
```

**原因**：
```
GLTF 文件不完整，缺少 .bin 几何数据文件
```

**解决**：
```bash
# 方案 1：获取完整文件
从原始来源重新导出，确保 .gltf 和 .bin 都存在

# 方案 2：使用嵌入式 GLTF
在 Blender 等工具中导出时选择 "Embed buffers"
```

### 问题 3：拖拽文件后缺少关联文件

**现象**：
```
拖拽 earth_tria.obj 后，earth_tria.mtl 和 earth.png 没有一起复制
拖拽 Pikachu.gltf 后，Pikachu.bin 没有一起复制
```

**原因**：
```
系统只复制了选中的文件，未自动复制关联文件
```

**解决**：
```bash
# 手动复制所有关联文件
OBJ: 复制 .obj + .mtl + 纹理图片
GLTF: 复制 .gltf + .bin + 纹理图片
```

**未来改进**：
```typescript
// 自动检测并提示复制关联文件
if (file.endsWith('.gltf')) {
  const binFile = file.replace('.gltf', '.bin');
  if (exists(binFile)) {
    prompt("是否一起复制 .bin 文件？");
  }
}
```

### 问题 4：纹理文件名不匹配

**现象**：
```
MTL 引用: map_Kd earth.png
实际文件: earth.bin
找不到纹理
```

**原因**：
```
.bin 文件名必须与原图片名完全一致（除了扩展名）
```

**解决**：
```bash
# 确保文件名一致
earth.png → earth.bin  ✅
earth.png → texture.bin  ❌
```

## 六、最佳实践

### OBJ 工作流程

```bash
# 1. 准备文件
assets/
├── model.obj
├── model.mtl
└── texture.png

# 2. 转换纹理
texture.png → texture.bin

# 3. 转换模型（自动加载纹理）
model.obj → desc_model.bin (包含纹理)
```

### GLTF 工作流程

```bash
# 1. 准备完整文件
assets/
├── model.gltf
└── model.bin  ← 必需！

# 2. 转换模型
model.gltf → desc_model.bin
```

### 导出建议

**Blender 导出 OBJ**：
- ✅ 勾选 "Write Materials"
- ✅ 勾选 "Include UVs"
- ✅ 勾选 "Write Normals"
- ✅ 确保纹理图片在同一目录

**Blender 导出 GLTF**：
- ✅ 格式选择 "glTF 2.0 (.gltf/.bin)"
- ✅ 或选择 "glTF Binary (.glb)" 自包含
- ✅ 勾选 "Remember Export Settings"

## 七、性能对比

| 模型 | 格式 | 原始大小 | 转换后大小 | 转换时间 |
|------|------|---------|-----------|---------|
| butterfly | OBJ | 8 KB | 1.4 KB | ~5ms |
| earth_tria | OBJ | 76 KB | 321 KB* | ~20ms |
| face | OBJ | 89 KB | 89 KB | ~30ms |
| Pikachu | GLTF | 203+115 KB | 97 KB | ~15ms |

*包含 262 KB 纹理数据

## 八、总结

### OBJ 特点
- ✅ 简单直观，易于理解
- ✅ 文本格式，便于调试
- ⚠️ 需要多个文件，容易丢失
- ⚠️ 纹理必须先转换

### GLTF 特点
- ✅ 现代标准，功能强大
- ✅ 二进制数据，加载快
- ⚠️ 必须有 .bin 文件
- ⚠️ 格式复杂，不易手动编辑

### 选择建议
- **简单模型**：使用 OBJ
- **复杂场景**：使用 GLTF
- **带动画**：必须使用 GLTF
- **易于编辑**：使用 OBJ
