# 模板开发工作流

## 概述

本文档说明如何开发和完善项目模板。

## 目录结构

```
honeygui-design/
├── template-projects/              # 模板开发目录 ⭐
│   ├── smartwatch/                # 完整的可运行项目
│   ├── settings/
│   ├── dashboard/
│   └── README.md
├── src/template/templates/         # 模板系统（自动生成）
│   └── wearable/smartwatch/
│       ├── SmartWatchTemplate.ts  # 模板类
│       └── project/               # 从 template-projects 同步
└── scripts/
    └── sync-templates.js          # 同步脚本
```

## 完整工作流程

### 场景1：完善现有模板

#### 步骤1：从模板创建项目

```bash
# 在 VSCode 中
Ctrl+Shift+P → HoneyGUI: New Project
→ Create template project
→ 选择 "Smart Watch UI"
→ 项目名：smartwatch
→ 保存位置：/path/to/honeygui-design/template-projects/
```

#### 步骤2：开发和完善

```bash
# 打开项目
cd template-projects/smartwatch
code .

# 正常开发：
# 1. 在设计器中编辑 UI
# 2. 添加图片到 assets/
#    - assets/watch_bg.png
#    - assets/icons/battery.png
# 3. 生成代码
# 4. 编译仿真测试
```

#### 步骤3：同步到模板系统

```bash
# 回到主项目目录
cd /path/to/honeygui-design

# 同步模板
npm run sync:templates smartwatch

# 输出：
# 📦 Syncing smartwatch...
#   - Copying files...
#   - Cleaning up...
#   - Removed: src/autogen
#   - Removed: build
#   - Adding variables...
#   - Updated: project.json
#   - Updated: SmartWatchMain.hml
#   - Updated: README.md
# ✅ smartwatch synced successfully
```

#### 步骤4：编译和测试

```bash
# 编译扩展
npm run compile

# 重新加载 VSCode
Ctrl+Shift+P → Developer: Reload Window

# 测试：创建新项目验证模板
HoneyGUI: New Project → Create template project → Smart Watch UI
```

### 场景2：创建新模板

#### 步骤1：创建项目

```bash
# 方式1：从空项目开始
HoneyGUI: New Project → Create empty project
→ 项目名：music-player
→ 保存位置：/path/to/honeygui-design/template-projects/
→ 分辨率：800X480

# 方式2：从现有模板修改
HoneyGUI: New Project → Create template project
→ 选择相似的模板
→ 修改为新模板
```

#### 步骤2：开发模板内容

```bash
cd template-projects/music-player

# 设计 UI
# - 播放/暂停按钮
# - 进度条
# - 歌曲列表
# - 专辑封面

# 添加资源
# - assets/play_icon.png
# - assets/pause_icon.png
# - assets/album_cover.png
```

#### 步骤3：创建模板类

```bash
# 创建目录
mkdir -p src/template/templates/entertainment/music-player

# 创建模板类
cat > src/template/templates/entertainment/music-player/MusicPlayerTemplate.ts << 'EOF'
import { BaseTemplate } from '../../BaseTemplate';

export class MusicPlayerTemplate extends BaseTemplate {
    id = 'music-player';
    name = 'Music Player';
    description = 'Music player with playback controls';
    category = 'Entertainment';
    recommendedResolution = '800X480';
}
EOF
```

#### 步骤4：注册模板

编辑 `src/template/templates/index.ts`：

```typescript
import { MusicPlayerTemplate } from './entertainment/music-player/MusicPlayerTemplate';

export const TEMPLATE_REGISTRY: ITemplate[] = [
    new SmartWatchTemplate(),
    new SettingsTemplate(),
    new DashboardTemplate(),
    new MusicPlayerTemplate(),  // 新增
];
```

#### 步骤5：更新同步脚本

编辑 `scripts/sync-templates.js`：

```javascript
const TEMPLATE_MAPPINGS = {
    'smartwatch': 'wearable/smartwatch',
    'settings': 'utility/settings',
    'dashboard': 'utility/dashboard',
    'music-player': 'entertainment/music-player'  // 新增
};
```

#### 步骤6：同步和编译

```bash
# 同步模板
npm run sync:templates music-player

# 编译
npm run compile

# 测试
# 重新加载 VSCode，应该能看到新模板
```

## 同步脚本详解

### 功能

`npm run sync:templates` 会：

1. **拷贝项目** - 从 `template-projects/` 到 `src/template/templates/*/project/`
2. **清理文件** - 删除：
   - `src/autogen/` - 生成的代码
   - `src/user/` - 用户代码
   - `build/` - 编译输出
   - `.vscode/` - VSCode 配置
3. **替换变量** - 将固定值替换为占位符：
   - 项目名 → `{{PROJECT_NAME}}`
   - APP ID → `{{APP_ID}}`
   - SDK 路径 → `{{SDK_PATH}}`
   - 创建时间 → `{{CREATED_TIME}}`

### 用法

```bash
# 同步所有模板
npm run sync:templates

# 只同步指定模板
npm run sync:templates smartwatch
npm run sync:templates settings
npm run sync:templates dashboard
```

## 常见问题

### Q: 修改了模板项目，但创建的新项目没有变化？

A: 需要执行完整流程：
```bash
npm run sync:templates
npm run compile
# 重新加载 VSCode
```

### Q: 同步后提示 "Source not found"？

A: 检查：
1. `template-projects/` 目录下是否有对应的项目
2. 项目名称是否与 `TEMPLATE_MAPPINGS` 中的 key 一致

### Q: 如何删除模板？

A: 
1. 删除 `template-projects/` 中的项目
2. 删除 `src/template/templates/` 中的模板类
3. 从 `index.ts` 中移除注册
4. 从 `sync-templates.js` 中移除映射

### Q: 可以在模板中包含预生成的代码吗？

A: 不建议。同步脚本会清理 `src/autogen/` 和 `src/user/`。
   模板应该只包含 HML 和资源文件。

## 总结

模板开发工作流：

1. **开发** - 在 `template-projects/` 中正常开发项目
2. **同步** - `npm run sync:templates` 同步到模板系统
3. **编译** - `npm run compile` 编译扩展
4. **测试** - 创建新项目验证模板

这个流程确保模板项目是完整的、可测试的，同时自动化了同步和清理过程。
