# 模板开发工作流

## 概述

模板项目采用 **双文件策略**，解决了"可复制"和"可开发"的矛盾：

- **开发版文件**：包含真实值，可以直接在 VSCode 中打开、编辑、编译
- **模板版文件**：包含占位符（如 `{{PROJECT_NAME}}`），用于复制给用户

## 文件命名规则

```
template-projects/dashboard/
├── project.json              # 开发版（真实值）
├── project.template.json     # 模板版（占位符）
├── ui/main/
│   ├── DashboardMain.hml           # 开发版
│   └── DashboardMain.template.hml  # 模板版
└── README.md / README.template.md  # 可选
```

## 工作流程

### 1. 开发模板

直接在 `template-projects/<模板名>/` 目录下开发：

```bash
# 在 VSCode 中打开模板项目
cd template-projects/dashboard
code .

# 正常开发流程：
# 1. 编辑 HML 文件（DashboardMain.hml）
# 2. 在设计器中调整布局
# 3. 添加资源到 assets/
# 4. 生成代码
# 5. 编译测试
```

**开发版配置示例**：
```json
{
  "name": "DashboardTemplate",
  "appId": "com.honeygui.dashboard",
  "honeyguiSdkPath": "/home/howie_wang/.HoneyGUI-SDK"
}
```

### 2. 同步到模板版

开发完成后，将修改同步到模板版文件：

#### 方式1：使用脚本（推荐）

```bash
node scripts/sync-template.js dashboard
```

脚本会自动：
- 读取开发版文件
- 替换真实值为占位符
- 写入模板版文件

#### 方式2：手动同步

```bash
# 拷贝文件
cp project.json project.template.json
cp ui/main/DashboardMain.hml ui/main/DashboardMain.template.hml

# 手动编辑 .template.* 文件，替换为占位符：
# DashboardTemplate -> {{PROJECT_NAME}}
# com.honeygui.dashboard -> {{APP_ID}}
# /home/howie_wang/.HoneyGUI-SDK -> {{SDK_PATH}}
```

### 3. 提交到 Git

两种文件都需要提交：

```bash
git add template-projects/dashboard/project.json
git add template-projects/dashboard/project.template.json
git add template-projects/dashboard/ui/main/*.hml
git commit -m "feat: 更新 dashboard 模板"
```

### 4. 用户使用

用户创建项目时，`BaseTemplate` 会：
1. 检查是否存在 `.template.*` 文件
2. 如果存在，使用模板版（带占位符）
3. 如果不存在，使用开发版
4. 拷贝到用户项目目录
5. 替换占位符为用户输入的值

## 占位符列表

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{PROJECT_NAME}}` | 项目名称 | MyProject |
| `{{APP_ID}}` | 应用ID | com.example.myapp |
| `{{SDK_PATH}}` | SDK路径 | /path/to/sdk |
| `{{CREATED_TIME}}` | 创建时间 | 2025-12-11 16:00:00 |

## 添加新模板

### 1. 创建目录结构

```bash
cd template-projects
mkdir my-template
cd my-template
mkdir -p ui/main assets src
```

### 2. 创建开发版文件

**project.json**（开发版）：
```json
{
  "name": "MyTemplate",
  "appId": "com.honeygui.mytemplate",
  "resolution": "480X272",
  "mainHmlFile": "ui/main/MyTemplateMain.hml",
  "honeyguiSdkPath": "/home/howie_wang/.HoneyGUI-SDK"
}
```

**ui/main/MyTemplateMain.hml**（开发版）：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<hml>
    <meta>
        <project name="MyTemplate" appId="com.honeygui.mytemplate" ... />
    </meta>
    <view>
        <!-- 你的组件 -->
    </view>
</hml>
```

### 3. 开发和测试

在 VSCode 中打开，正常开发。

### 4. 创建模板版

```bash
node scripts/sync-template.js my-template
```

### 5. 注册模板类

创建 `src/template/templates/MyTemplate.ts`：

```typescript
import * as path from 'path';
import { BaseTemplate } from './BaseTemplate';

export class MyTemplate extends BaseTemplate {
    id = 'my-template';
    name = 'My Template';
    description = 'Template description';
    category = 'Utility';
    recommendedResolution = '480X272';
    
    protected getTemplateProjectPath(): string {
        return path.join(__dirname, '..', '..', '..', 'template-projects', 'my-template');
    }
}
```

在 `src/template/templates/index.ts` 注册：

```typescript
import { MyTemplate } from './MyTemplate';

export const TEMPLATE_REGISTRY: ITemplate[] = [
    // ...
    new MyTemplate(),
];
```

### 6. 编译和测试

```bash
npm run compile
# 在 VSCode 中按 F5 测试
```

## 常见问题

### Q: 为什么需要两个文件？

A: 
- **开发版**：让开发者能直接运行和测试模板
- **模板版**：让用户创建项目时能自定义项目名等信息

### Q: 修改后忘记同步怎么办？

A: 运行 `node scripts/sync-template.js <模板名>` 即可重新同步。

### Q: 可以只提交模板版吗？

A: 不建议。开发版文件让其他开发者能快速测试模板，提高协作效率。

### Q: 同步脚本会覆盖手动修改吗？

A: 会。如果你手动修改了模板版文件，运行脚本会覆盖。建议只修改开发版，然后用脚本同步。

## 最佳实践

1. **只修改开发版** - 所有开发工作在开发版文件中进行
2. **定期同步** - 每次提交前运行同步脚本
3. **测试两个版本** - 确保开发版能运行，模板版能正确创建项目
4. **提交两个版本** - 两种文件都提交到 Git

## 示例：完整开发流程

```bash
# 1. 开发模板
cd template-projects/dashboard

# 在 VSCode 中编辑 DashboardMain.hml
# 添加组件、调整布局、测试

# 2. 同步到模板版

node scripts/sync-template.js dashboard

# 3. 编译扩展
npm run compile

# 4. 测试
# 按 F5 启动调试
# 创建新项目，选择 dashboard 模板
# 验证生成的项目是否正确

# 5. 提交
git add template-projects/dashboard/
git commit -m "feat: 完善 dashboard 模板"
git push
```

简单高效！
