# 拖拽文件处理说明

## 问题描述

用户从文件管理器拖拽文件到资源面板时，只复制了选中的文件，**没有自动复制关联文件**。

### 典型场景

**GLTF 模型**：
```
用户拖拽：Pikachu.gltf
实际需要：Pikachu.gltf + Pikachu.bin
结果：只复制了 .gltf，缺少 .bin → 转换失败
```

**OBJ 模型**：
```
用户拖拽：earth_tria.obj
实际需要：earth_tria.obj + earth_tria.mtl + earth.png
结果：已处理 ✅（之前已实现）
```

## 解决方案

### 修改前

```typescript
// 只处理 OBJ 依赖
const objFiles = files.filter(f => f.name.endsWith('.obj'));
if (objFiles.length > 0) {
  // 处理 OBJ 及其依赖
  await processObjWithDependencies(...);
}
```

### 修改后

```typescript
// 同时处理 OBJ 和 GLTF 依赖
const objFiles = files.filter(f => f.name.endsWith('.obj'));
const gltfFiles = files.filter(f => f.name.endsWith('.gltf'));

if (objFiles.length > 0 || gltfFiles.length > 0) {
  // 处理 OBJ
  for (const objFile of objFiles) {
    await processObjWithDependencies(objFile, fileList, '');
  }
  
  // 处理 GLTF ⭐ 新增
  for (const gltfFile of gltfFiles) {
    await processGltfWithDependencies(gltfFile, fileList, '');
  }
}
```

## 实现细节

### processGltfWithDependencies 函数

```typescript
const processGltfWithDependencies = async (
  gltfFile: File, 
  fileList: FileList, 
  relativePath: string
) => {
  // 1. 查找对应的 .bin 文件
  const binFileName = gltfFile.name.replace(/\.gltf$/i, '.bin');
  const binFile = Array.from(fileList).find(f => f.name === binFileName);
  
  // 2. 上传 GLTF 文件
  await uploadFile(gltfFile, relativePath);
  
  // 3. 上传 .bin 文件（如果存在）
  if (binFile) {
    await uploadFile(binFile, relativePath);
  } else {
    // 4. 警告用户缺少 .bin 文件
    window.vscodeAPI?.postMessage({
      command: 'notify',
      text: `${gltfFile.name} 缺少对应的 ${binFileName} 文件。
             模型可能无法正常转换。
             请同时选中 .gltf 和 .bin 文件一起拖拽。`
    });
  }
};
```

### 处理流程

```
用户拖拽多个文件
  ↓
检测文件类型
  ├─ 包含 .obj → processObjWithDependencies
  │   ├─ 上传 .obj
  │   ├─ 查找并上传 .mtl
  │   └─ 查找并上传纹理图片
  │
  └─ 包含 .gltf → processGltfWithDependencies ⭐
      ├─ 上传 .gltf
      ├─ 查找 .bin 文件
      ├─ 如果找到 → 上传 .bin
      └─ 如果未找到 → 显示警告
```

## 文件类型支持

| 格式 | 主文件 | 依赖文件 | 状态 |
|------|--------|---------|------|
| **OBJ** | .obj | .mtl + 纹理图片 | ✅ 已支持 |
| **GLTF** | .gltf | .bin | ✅ 新增支持 |
| **GLB** | .glb | 无（自包含） | ✅ 无需处理 |
| **图片** | .png/.jpg | 无 | ✅ 直接复制 |
| **视频** | .mp4/.avi | 无 | ✅ 直接复制 |

### 2. 解析 GLTF JSON

读取 GLTF 文件内容，查找所有引用的资源：

```typescript
// 解析 GLTF JSON
const gltfContent = await gltfFile.text();
const gltfData = JSON.parse(gltfContent);

// 查找 buffer 文件
const buffers = gltfData.buffers || [];
const bufferFiles = buffers
  .filter(b => b.uri && !b.uri.startsWith('data:'))
  .map(b => b.uri);

// 查找纹理文件
const images = gltfData.images || [];
const imageFiles = images
  .filter(i => i.uri && !i.uri.startsWith('data:'))
  .map(i => i.uri);

// 自动上传所有依赖
for (const fileName of [...bufferFiles, ...imageFiles]) {
  const file = Array.from(fileList).find(f => f.name === fileName);
  if (file) {
    await uploadFile(file, relativePath);
  }
}
```

### 3. 目录拖拽支持

支持拖拽整个文件夹：

```
拖拽文件夹：models/
  ├── Pikachu.gltf
  ├── Pikachu.bin
  └── textures/
      └── texture.png

结果：保持目录结构复制所有文件
```
