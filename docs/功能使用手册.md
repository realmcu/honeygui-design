## 组件使用

### 几何控件

#### 圆弧控件 (hg_arc)

**基本信息：**
- 图标：🌙
- 用途：进度环、仪表盘指示器、装饰性圆弧

**属性：**
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| radius | number | 40 | 圆弧半径（像素） |
| startAngle | number | 0 | 起始角度（度，0° 为右侧） |
| endAngle | number | 270 | 结束角度（度） |
| strokeWidth | number | 8 | 线条宽度（像素） |
| color | color | #007acc | 圆弧颜色 |
| opacity | number | 255 | 透明度（0-255） |

**使用示例：**
```xml
<!-- 进度环 -->
<hg_arc id="progress" x="50" y="50" radius="40" startAngle="0" endAngle="270" strokeWidth="8" color="#00ff00"/>

<!-- 完整圆环 -->
<hg_arc id="ring" x="200" y="200" radius="70" startAngle="0" endAngle="360" strokeWidth="15" color="#ff0000"/>
```

#### 圆角矩形控件 (hg_rect)

**基本信息：**
- 图标：⬜
- 用途：卡片、按钮背景、装饰框

**属性：**
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| width | number | 100 | 矩形宽度 |
| height | number | 60 | 矩形高度 |
| cornerRadius | number | 10 | 圆角半径 |
| color | color | #007acc | 填充颜色 |
| strokeWidth | number | 0 | 边框宽度（0为无边框） |
| strokeColor | color | #000000 | 边框颜色 |

### 渐变功能

支持的控件：`hg_arc`、`hg_circle`、`hg_rect`

**启用渐变：**
1. 在属性面板的"样式"分组中勾选"启用渐变"
2. 选择渐变类型/方向
3. 配置起始和结束颜色

**渐变类型：**
- **hg_arc**：角度渐变（沿弧线方向）
- **hg_circle**：径向渐变 + 角度渐变
- **hg_rect**：线性渐变（上到下、左到右、对角线等）

**示例：**
```xml
<hg_rect id="card" x="0" y="0" width="200" height="100"
         gradientEnabled="true"
         gradientDirection="topToBottom"
         gradientStartColor="#ff6b6b"
         gradientEndColor="#4ecdc4"/>
```

## 事件系统

HoneyGUI Design 的事件系统采用 **事件-动作(Event-Action)** 模式，在设计器中配置事件响应，无需编写代码。

### 核心概念

1. **事件(Event)**：组件触发的交互行为（如点击、滑动）
2. **动作(Action)**：事件触发后执行的操作（如跳转界面、发送消息）
3. **消息(Message)**：全局广播机制，组件间通信

### 事件类型

**通用事件（所有组件支持）：**
| 事件 | 标识符 | 说明 |
|------|--------|------|
| 单击 | `onClick` | 点击后抬起触发 |
| 长按 | `onLongPress` | 按住超过阈值时间触发 |
| 按下 | `onTouchDown` | 手指按下时触发 |
| 抬起 | `onTouchUp` | 手指抬起时触发 |

**滑动事件（容器组件）：**
| 事件 | 标识符 | 支持组件 |
|------|--------|----------|
| 左滑 | `onSwipeLeft` | hg_view, hg_window |
| 右滑 | `onSwipeRight` | hg_view, hg_window |
| 上滑 | `onSwipeUp` | hg_view, hg_window |
| 下滑 | `onSwipeDown` | hg_view, hg_window |

**状态事件（特定组件）：**
| 事件 | 标识符 | 支持组件 |
|------|--------|----------|
| 值改变 | `onValueChange` | hg_slider, hg_progressbar, hg_input |
| 选中改变 | `onCheckedChange` | hg_checkbox, hg_switch, hg_radio |

### 动作类型

**视图切换动作：**
```typescript
switchView: {
  target: "目标视图ID",
  switchOutStyle: "none|fadeout|slideout_left|slideout_right|slideout_top|slideout_bottom",
  switchInStyle: "none|fadein|slidein_left|slidein_right|slidein_top|slidein_bottom"
}
```

**组件控制动作：**
- `show`: 显示组件
- `hide`: 隐藏组件
- `enable`: 启用组件
- `disable`: 禁用组件

**消息动作：**
- `sendMessage`: 发送全局消息，其他组件可通过 `onMessage` 事件监听

### 配置事件

1. 选中组件
2. 切换到属性面板的"事件"标签页
3. 选择事件类型
4. 添加动作并配置参数
5. 保存设计

**示例：点击按钮跳转到设置界面**
```
组件：hg_button (id="btnSettings")
事件：onClick
动作：switchView
  - target: "settingsView"
  - switchOutStyle: "slideout_left"
  - switchInStyle: "slidein_right"
```

## 代码生成

### 生成流程

```
HML 文件 → HmlParser → 组件树 → HoneyGuiCCodeGenerator → C 代码
```

### 生成文件结构

```
project/
├── ui/
│   └── main/
│       └── main.hml
└── src/
    ├── {ProjectName}Entry.c    # 项目入口（每次覆盖）
    ├── SConscript              # 构建脚本（每次覆盖）
    ├── ui/                     # UI代码（每次覆盖）
    │   ├── main_ui.h
    │   └── main_ui.c
    ├── callbacks/              # 回调代码（保护区机制）
    │   ├── main_callbacks.h
    │   └── main_callbacks.c
    └── user/                   # 用户代码（首次生成，不覆盖）
        ├── main_user.h
        └── main_user.c
```

### 保护区机制

在 `*_callbacks.c` 文件中，用户代码使用保护区标记：

```c
void btnSubmit_onClick(void *obj, gui_event_t e)
{
    /* USER CODE BEGIN btnSubmit_onClick */
    // 在此编写自定义逻辑
    // 这部分代码不会被覆盖
    /* USER CODE END btnSubmit_onClick */
}
```

**规则：**
- `/* USER CODE BEGIN */` 和 `/* USER CODE END */` 之间的代码会被保留
- 其他部分在重新生成时会被覆盖
- 支持多个保护区

### 生成代码示例

**HML 输入：**
```xml
<hg_button id="btn1" x="10" y="10" width="100" height="40" label="Click"/>
```

**生成的 C 代码：**
```c
// main_ui.c
gui_obj_t *btn1;

void main_ui_create(gui_obj_t *parent)
{
    btn1 = (gui_obj_t *)gui_button_create(parent, "btn1", 10, 10, 100, 40);
    gui_button_text_set((gui_button_t *)btn1, "Click", "Arial", 0xFFFFFF, 14);
}
```

### 触发代码生成

**方式一：命令面板**
```
Ctrl+Shift+P → HoneyGUI: Generate Code
```

**方式二：设计器工具栏**
点击工具栏的"生成代码"按钮

## 组件树操作

### 拖拽功能

在组件树面板中，支持通过拖拽改变组件的层级关系和顺序。

**操作方法：**
1. 按住鼠标左键拖动组件
2. 根据鼠标位置选择放置方式：
   - **上部 1/4**：插入到目标前面（显示蓝色上边框）
   - **中间 1/2**：作为目标的子组件（显示蓝色虚线框）
   - **下部 1/4**：插入到目标后面（显示蓝色下边框）
3. 释放鼠标完成操作

**拖拽规则：**
- ❌ `hg_view` 不能被拖拽（只能作为根组件）
- ❌ `hg_list_item` 不能拖动（由 hg_list 自动管理）
- ✅ 只有容器控件可以接收子组件：`hg_view`、`hg_window`、`hg_list_item`
- ✅ 兄弟组件可以自由调整顺序
- ❌ 不能拖到自己或自己的子组件中（避免循环引用）

**效果：**
- 拖拽会立即保存到文件
- 组件顺序影响渲染顺序（后面的组件在上层）
- 不改变组件的坐标，只改变层级关系

## 特殊功能

### 时间接口（hg_label）

为 `hg_label` 组件添加自动更新时间的功能。

**配置步骤：**
1. 添加 `hg_label` 组件
2. 在属性面板的"数据"分组中，选择"时间格式"：
   - `HH:mm:ss` - 时分秒（14:30:25）
   - `HH:mm` - 时分（14:30）
   - `YYYY-MM-DD` - 年月日（2024-12-24）
   - `YYYY-MM-DD HH:mm:ss` - 完整日期时间
   - `MM-DD HH:mm` - 月日时分
3. 生成代码

**生成的代码：**
```c
// UI 代码：创建定时器
static void mainView_switch_in(gui_view_t *view)
{
    time_label = gui_text_create((gui_obj_t *)view, "time", 10, 10, 100, 24);
    gui_text_content_set((gui_text_t *)time_label, (void *)"00:00:00", 8);

    // 创建1秒间隔的定时器
    gui_obj_create_timer(GUI_BASE(time_label), 1000, true, time_label_update_cb);
}

// 回调代码：更新时间
void time_label_update_cb(void *p)
{
    time_t now = time(NULL);
    struct tm *t = localtime(&now);
    static char time_str[9];

    sprintf(time_str, "%02d:%02d:%02d", t->tm_hour, t->tm_min, t->tm_sec);
    gui_text_content_set((gui_text_t *)time_label, (void *)time_str, strlen(time_str));
}
```

**特点：**
- 自动创建定时器，无需手动配置
- 定时器绑定到 label 对象，生命周期由对象管理
- 每个时间 label 有独立的定时器

### 拖拽文件处理

从文件管理器拖拽文件到资源面板时，自动复制关联文件。

**支持的场景：**

**GLTF 模型：**
```
拖拽：Pikachu.gltf
自动复制：Pikachu.gltf + Pikachu.bin
```

**OBJ 模型：**
```
拖拽：earth.obj
自动复制：earth.obj + earth.mtl + 纹理图片（通过解析 .mtl 获取）
```

**MP4 视频：**
```
拖拽：demo.mp4
自动复制：demo.mp4（及元数据）
```

**处理流程：**
1. 用户拖拽文件到资源面板
2. 系统检测文件类型
3. 自动查找并复制关联文件
4. 显示复制结果（成功/失败）

## 常见问题

**Q: 为什么圆弧不显示？**
A: 检查圆心坐标和半径是否在父容器范围内，透明度是否设置正确。

**Q: 如何修改生成的代码？**
A: 在 `*_callbacks.c` 文件的 `USER CODE BEGIN/END` 保护区内修改，或在 `*_user.c` 文件中添加自定义代码。

**Q: 事件动作不执行怎么办？**
A: 确认目标组件 ID 正确，检查生成的代码是否正确注册了回调函数。

**Q: 如何删除组件的时间更新功能？**
A: 在属性面板中将"时间格式"设置为空，重新生成代码。

**Q: 拖拽文件后资源未显示？**
A: 刷新资源面板，或检查文件是否成功复制到 assets 目录。
