## 概述

HoneyGUI Design 提供纯 JavaScript/TypeScript 实现的资源转换工具，无需 Python 环境。

### 优势

- ✅ 纯 JS/TS 实现，无需 Python
- ✅ 插件自包含，开箱即用
- ✅ 性能提升 25-40%
- ✅ 统一技术栈，易于维护

### 打开资源转换工具

**方式一：命令面板**
```
Ctrl+Shift+P → HoneyGUI: Resource Conversion Tools
```

**方式二：侧边栏**
点击 HoneyGUI 侧边栏的"工具"按钮

## 图片转换

### 支持格式

| 输入格式 | 输出格式 | 状态 |
|----------|----------|------|
| PNG | BIN | ✅ |
| JPEG | BIN | ✅ |
| BMP | BIN | ✅ |

### 像素格式

| 格式 | 位深 | 说明 |
|------|------|------|
| RGB565 | 16 位 | 无透明通道，适合静态图片 |
| RGB888 | 24 位 | 真彩色，无透明通道 |
| ARGB8888 | 32 位 | 真彩色 + Alpha 通道 |
| ARGB8565 | 24 位 | 压缩的 ARGB 格式 |
| A8 | 8 位 | 仅透明度通道 |

### 使用方法

1. 在资源转换工具中选择"图片转换"
2. 选择源图片文件（可多选）
3. 选择像素格式（推荐 Auto 自动选择）
4. 选择输出目录（默认：`assets/images/`）
5. 点击"开始转换"

### 自动格式选择规则

- 无透明通道 → RGB565（最省空间）
- 有透明通道 → ARGB8888
- 灰度图 → A8

### 输出文件

```
assets/images/
├── icon.png.bin      # 转换后的二进制文件
└── icon.png.txt      # 元数据（尺寸、格式等）
```

## 3D模型转换

### 支持格式

| 格式 | 状态 | 说明 |
|------|------|------|
| OBJ | ✅ 完成 | 支持 MTL 材质和纹理 |
| GLTF | ⏳ 计划中 | 下一版本支持 |
| GLB | ⏳ 计划中 | GLTF 二进制格式 |

### OBJ 格式详解

#### 文件结构

```
模型文件/
├── model.obj         # 主文件（顶点、面、纹理坐标）
├── model.mtl         # 材质文件（颜色、纹理引用）
└── texture.png       # 纹理图片
```

#### 文件关系

```
model.obj
  ↓ (mtllib model.mtl)
model.mtl
  ↓ (map_Kd texture.png)
texture.png
```

#### 转换流程

**1. 解析 OBJ 文件**
```obj
# 顶点坐标
v -0.000000 -0.831470 -0.555570
v -0.000000 -0.555570 -0.831470

# 纹理坐标
vt 0.000000 0.000000
vt 0.062500 0.000000

# 法线
vn 0.0000 -0.8507 -0.5257

# 面（索引）
f 1/1/1 2/2/2 3/3/3

# 材质引用
mtllib model.mtl
usemtl material_001
```

**2. 解析 MTL 文件**
```mtl
newmtl material_001
Ka 1.0 1.0 1.0         # 环境光
Kd 0.8 0.8 0.8         # 漫反射
Ks 0.5 0.5 0.5         # 镜面反射
Ns 250.000000          # 高光指数
map_Kd texture.png     # 漫反射纹理
```

**3. 生成二进制文件**

输出文件：
```
assets/models/
├── desc_model.bin     # 顶点、法线、纹理坐标、索引
├── desc_model.txt     # 元数据
└── texture.png        # 纹理图片（复制）
```

#### 使用方法

1. 准备 OBJ 模型（确保 .obj、.mtl、纹理图片在同一目录）
2. 在资源转换工具中选择"3D模型转换"
3. 选择 .obj 文件
4. 系统自动查找 .mtl 和纹理文件
5. 输出到 `assets/models/` 目录

#### 注意事项

- 确保 .obj 文件中引用的 .mtl 文件存在
- 确保 .mtl 文件中引用的纹理图片存在
- 纹理图片路径可以是相对路径或绝对路径
- 支持三角形和四边形面

### GLTF 格式详解

#### 文件结构

```
模型文件/
├── model.gltf        # JSON 描述（场景、材质、引用）
├── model.bin         # 二进制数据（顶点、索引）
└── texture.png       # 纹理图片
```

#### 特点

- 现代 3D 格式标准
- 支持动画、骨骼、蒙皮
- 二进制数据，加载快
- 可嵌入数据（GLB 格式）

#### 状态

⏳ GLTF 支持计划在 v1.2.0 版本实现

## 视频转换

### 支持格式

| 输入格式 | 输出格式 | 状态 |
|----------|----------|------|
| MP4 | MP4 (H.264) | ✅ |
| AVI | MP4 (H.264) | ✅ |
| MOV | MP4 (H.264) | ✅ |

### 使用方法

1. 在资源转换工具中选择"视频转换"
2. 选择源视频文件
3. 配置转换参数：
   - 分辨率（如 480x272）
   - 帧率（如 30fps）
   - 码率（如 1000kbps）
4. 选择输出目录
5. 点击"开始转换"

### 转换参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 分辨率 | 原始分辨率 | 目标分辨率 |
| 帧率 | 30fps | 每秒帧数 |
| 码率 | 1000kbps | 视频比特率 |
| 编码器 | H.264 | 视频编码格式 |

### 前置依赖

需要安装 FFmpeg：
- **Windows**: 从 [ffmpeg.org](https://ffmpeg.org) 下载并添加到 PATH
- **Linux**: `sudo apt install ffmpeg`
- **macOS**: `brew install ffmpeg`

## 字体转换

### 支持格式

| 输入格式 | 输出格式 | 状态 |
|----------|----------|------|
| TTF | BIN | ✅ |
| OTF | BIN | ✅ |

### 使用方法

1. 在资源转换工具中选择"字体转换"
2. 选择字体文件（TTF/OTF）
3. 配置转换参数：
   - 字号（如 16, 24, 32）
   - 字符集（常用汉字、ASCII、自定义）
   - 抗锯齿（启用/禁用）
4. 选择输出目录
5. 点击"开始转换"

### 字符集选项

- **ASCII**：基本英文字符
- **常用汉字**：GB2312 常用汉字 (~6000 字)
- **扩展汉字**：GBK 汉字 (~20000 字)
- **自定义**：指定字符列表

### 输出文件

```
assets/fonts/
├── font_16.bin       # 16号字体数据
├── font_16.txt       # 元数据
├── font_24.bin       # 24号字体数据
└── font_24.txt       # 元数据
```

## 常见问题

**Q: 为什么不需要安装 Python？**
A: 我们使用纯 JavaScript/TypeScript 实现了所有转换工具，无需额外依赖。

**Q: 转换后的文件在哪里？**
A: 默认输出到项目的 `assets/` 目录对应的子目录中（images、models、fonts、videos）。

**Q: 如何批量转换？**
A: 在文件选择对话框中按住 Ctrl 或 Shift 选择多个文件。

**Q: 转换失败怎么办？**
A: 查看 Output 面板的 "HoneyGUI Design" 通道，检查错误信息。常见原因：
- 文件格式不支持
- 关联文件缺失（如 .obj 缺少 .mtl）
- 文件权限问题

**Q: GLTF 什么时候支持？**
A: GLTF 支持计划在 v1.2.0 版本实现，预计 2025 年上半年。

**Q: 如何转换带动画的 3D 模型？**
A: 当前版本不支持动画。如需动画支持，请等待 GLTF 支持（v1.2.0）。

**Q: 视频转换需要 FFmpeg 吗？**
A: 是的，视频转换依赖 FFmpeg。请确保已安装并添加到系统 PATH。
