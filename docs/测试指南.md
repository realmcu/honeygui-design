# 测试指南

## 概述

HoneyGUI Design 使用模板端到端测试来验证核心功能：HML 解析、C 代码生成、编译和仿真运行。

## 运行测试

```bash
npm test
```

## 测试流程

每个模板测试执行以下步骤：

```
[1/5] 查找 HML 文件
[2/5] 解析 HML (HmlParser)
[3/5] 生成 C 代码 (HoneyGuiCCodeGenerator)
[4/5] 编译 (BuildCore + SCons)
[5/5] 运行仿真 (2秒)
```

## 测试覆盖的模板

| 模板 | 分辨率 | 描述 |
|------|--------|------|
| rotary | 480x480 | 圆形表盘界面 |
| settings | 480x272 | 设置界面 |
| smartwatch | 410x502 | 智能手表界面 |
| dashboard | 800x480 | 仪表盘界面 |
| chatbot | 480x320 | 聊天机器人界面 |

## 前置依赖

- **HoneyGUI SDK**: 安装到 `~/.HoneyGUI-SDK`
- **SCons**: `pip install scons`
- **GCC**: C 编译器
- **Python**: 用于 romfs 打包

## 测试文件位置

```
test/templates/run-templates-tests.ts  # 测试脚本
template-projects/                      # 模板项目
├── rotary/
├── settings/
├── smartwatch/
├── dashboard/
└── chatbot/
```

## 添加新模板测试

1. 在 `template-projects/` 下创建模板目录
2. 添加 `ui/main/XXXMain.hml` 文件
3. 在测试脚本的 `templates` 数组中添加模板名称

## 常见问题

### SDK 不存在
```
错误: SDK 不存在
```
解决：安装 HoneyGUI SDK 到 `~/.HoneyGUI-SDK`

### 编译失败
检查：
- HML 文件是否有嵌套的 `hg_view`（不允许）
- 使用的组件是否被 SDK 支持

### FFmpeg 未找到
这是警告，不影响测试。如需视频转换功能，请安装 FFmpeg。
