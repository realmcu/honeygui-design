# HoneyGUI Design 测试指南

本文档描述 HoneyGUI Design 项目的测试策略、测试类型和执行方法。

## 测试概述

项目采用多层次测试策略：
- **端到端测试 (E2E)**：验证完整工作流程（HML 生成 → C 代码生成 → SDK 编译 → 仿真运行）
- **单元测试**：测试独立模块和函数
- **集成测试**：测试模块间交互
- **手动测试**：在 VS Code 扩展宿主中进行功能验证

## 端到端测试

### 测试脚本

位置：`scripts/e2e-test.ts`

### 测试流程

E2E 测试自动执行以下完整流程：

1. **环境检查**
   - 验证 HoneyGUI SDK 存在（`~/.HoneyGUI-SDK`）
   - 检查 SCons 工具链
   - 检查 GCC 编译器

2. **创建临时项目**
   - 生成项目目录结构：`ui/main/`, `src/autogen/main/`, `assets/`
   - 创建 `project.json` 配置文件

3. **生成 HML 文件**
   - 使用 `HmlSerializer` 将组件数据序列化为 HML 格式
   - 模拟设计器保存操作
   - 测试组件：`hg_view` (主视图) + `hg_image` (图片组件)

4. **解析并生成 C 代码**
   - 使用 `HmlParser` 解析 HML 文件（验证序列化/反序列化闭环）
   - 使用 `HoneyGuiCCodeGenerator` 生成 C 代码
   - 输出到 `src/autogen/main/` 目录

5. **编译验证**
   - 使用 `BuildCore` 准备编译环境
   - 复制生成的代码到 SDK 的 `win32_sim` 目录
   - 执行 `scons` 编译
   - 验证可执行文件生成

6. **仿真运行**（可选）
   - 检测图形环境（DISPLAY 或 Windows）
   - 运行仿真程序 3 秒
   - 无图形环境时跳过此步骤

7. **清理**
   - 保留生成的工程以便检查（默认不清理）
   - 可选：删除临时文件

### 运行 E2E 测试

```bash
# 执行完整的端到端测试
npm run test:e2e
```

### 前置条件

**必需**：
- HoneyGUI SDK 安装在 `~/.HoneyGUI-SDK`
- GCC 编译器
- SCons 构建工具
- Node.js 和 npm

**可选**（用于仿真运行）：
- SDL2 库
- 图形环境（X11 或 Windows）

### 测试配置

配置项位于 `scripts/e2e-test.ts` 的 `CONFIG` 对象：

```typescript
const CONFIG = {
  sdkPath: path.join(os.homedir(), '.HoneyGUI-SDK'),  // SDK 路径
  resolution: { width: 480, height: 272 },             // 屏幕分辨率
  timeout: 120000,                                     // 编译超时（毫秒）
  runDuration: 3000,                                   // 仿真运行时长（毫秒）
};
```

### 测试组件数据

测试使用 `createMockDesignerData()` 函数生成模拟组件：

```typescript
// 主视图（黑色背景，全屏）
{
  id: 'mainView',
  type: 'hg_view',
  position: { x: 0, y: 0, width: 480, height: 272 },
  style: { backgroundColor: '#000000' }
}

// 图片组件
{
  id: 'testImage',
  type: 'hg_image',
  position: { x: 100, y: 50, width: 280, height: 172 },
  data: { src: 'assets/test.png' },
  parent: 'mainView'
}
```

### 测试验证点

E2E 测试验证以下关键功能：

✅ **HML 序列化**：Component[] → HML String  
✅ **HML 解析**：HML String → Component[]（闭环验证）  
✅ **C 代码生成**：Component[] → C 源文件  
✅ **代码正确性**：生成的代码能通过 GCC 编译  
✅ **SDK 集成**：生成的代码能与 HoneyGUI SDK 链接  
✅ **运行时验证**：编译后的程序能正常启动（有图形环境时）

### 故障排查

**SDK 未找到**：
```bash
# 检查 SDK 路径
ls ~/.HoneyGUI-SDK

# 或修改 CONFIG.sdkPath
```

**编译失败**：
```bash
# 检查工具链
scons --version
gcc --version

# 查看生成的代码
cat e2e_test_project/src/autogen/main/main.c
```

**仿真无法启动**：
```bash
# 检查图形环境
echo $DISPLAY

# 检查 SDL2
ldconfig -p | grep SDL2
```

```
test/
├── unit/              # 单元测试
│   ├── hml/          # HML 解析器/序列化器测试
```

## 手动测试

### 在 VS Code 中测试

1. **启动扩展开发宿主**
   ```bash
   # 按 F5 或运行
   npm run watch
   # 然后在 VS Code 中按 F5
   ```

2. **测试新项目创建**
   - 打开命令面板（Ctrl+Shift+P）
   - 执行：`HoneyGUI: New Project`
   - 验证项目结构和 HML 文件

3. **测试设计器**
   - 右键点击 `.hml` 文件
   - 选择"使用 HoneyGUI 设计器打开"
   - 测试拖放、属性编辑、保存

4. **测试代码生成**
   - 在设计器中点击"生成代码"
   - 验证生成的 C/C++ 文件
   - 检查代码保护区是否保留

5. **测试预览功能**
   - 执行：`HoneyGUI: Preview`
   - 验证仿真窗口启动

### Webview 独立测试

```bash
# 启动 webview 开发服务器
npm run serve:webview

# 在浏览器中访问
# http://localhost:3000
```

在浏览器中测试：
- 组件拖放
- 属性编辑
- 撤销/重做
- 键盘快捷键
- 响应式布局

### 调试技巧

**扩展端调试**：
- 在 TypeScript 代码中设置断点
- 查看 Output 面板 → "HoneyGUI Design"
- 使用 `console.log()` 输出调试信息

**Webview 端调试**：
- 在开发宿主中：Help → Toggle Developer Tools
- 在独立模式：浏览器开发者工具（F12）
- 使用 React DevTools 扩展

**通信调试**：
```typescript
// 扩展端
console.log('[Extension] Sending:', message);

// Webview 端
console.log('[Webview] Received:', event.data);
```
