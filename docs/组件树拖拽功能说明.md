# 组件树拖拽功能说明

## 功能概述

在组件树面板中，现在支持通过拖拽来改变组件的父子关系和兄弟关系，包括：
- 将组件作为另一个容器的子组件
- 在兄弟组件之间插入（调整顺序）

## 使用方法

1. **拖拽组件**：在组件树中按住鼠标左键拖动组件
2. **选择放置位置**：
   - **插入到前面**：鼠标移到目标组件的上部（显示上边框）
   - **插入到后面**：鼠标移到目标组件的下部（显示下边框）
   - **作为子组件**：鼠标移到容器组件的中间（显示虚线框）
3. **释放鼠标**：完成拖放操作

## 视觉反馈

- **插入前**：目标组件显示蓝色上边框
- **插入后**：目标组件显示蓝色下边框
- **作为子组件**：目标容器显示蓝色虚线框

## 拖拽规则

### 1. hg_view 限制
- **hg_view 不能被拖拽**
- hg_view 只能作为根组件存在

### 2. hg_list_item 限制
- **hg_list_item 不能拖动**
- list_item 由 hg_list 控件自动管理
- 通过修改 hg_list 的 noteNum 属性来调整 list_item 数量

### 3. 容器规则
- 只有容器控件可以接收子组件：
  - hg_view
  - hg_window
  - hg_list_item
- 非容器控件不能作为父组件
- 容器控件支持三种放置方式：
  - 上部 1/4：插入到容器前面（作为兄弟）
  - 中间 1/2：作为容器的子组件
  - 下部 1/4：插入到容器后面（作为兄弟）

### 4. 兄弟组件排序
- 可以在同级组件之间自由调整顺序
- 拖到组件上半部分：插入到该组件前面
- 拖到组件下半部分：插入到该组件后面

### 5. 其他限制
- 不能将组件拖到自己身上
- 不能将组件拖到自己的子组件中（避免循环引用）

## 实现细节

### 前端实现
- `ComponentTree.tsx`：增强了拖拽事件处理
  - `handleDragOver`：根据鼠标位置计算放置方式（before/after/inside）
  - `handleDrop`：根据放置方式执行不同操作
  - `insertComponentAt`：在指定位置插入组件
  - 支持三种放置位置的视觉反馈

### 后端实现
- `store.ts`：新增方法
  - `reorderSiblings`：重新排序同级组件，重建 components 数组保持正确顺序
  - `moveComponentToPosition`：移动组件到指定位置，使用更新后的状态计算索引
  - 增强了 `moveComponent` 的验证逻辑

### CSS 样式
- `.drag-before`：插入前的指示器（上边框）
- `.drag-after`：插入后的指示器（下边框）
- `.drag-inside`：作为子组件的指示器（虚线框）

## 注意事项

1. 拖拽操作会立即保存到文件
2. 不符合规则的拖拽会被忽略（无提示）
3. 拖拽不会改变组件的位置坐标，只改变层级关系和顺序
4. 组件顺序会影响渲染顺序（后面的组件在上层）
