# HoneyGUI 插件调试指南

## 如何收集调试信息并反馈给 AI

当你遇到问题时，请按以下步骤收集日志信息。这些信息能帮助 AI 快速定位问题。

---

## 📋 收集日志的 4 个关键来源

### 1️⃣ **扩展输出通道（最重要）**

所有重要的日志都会输出到这里。

**查看方法：**
1. 打开 VS Code 的 **输出面板**（菜单：查看 → 输出，或按 `Ctrl+Shift+U`）
2. 在输出面板右上角的下拉菜单中，选择  **"HoneyGUI Design"**

**应该提供什么：**
- 完整的日志内容（从启动到出错的所有信息）
- 时间戳和错误消息

**示例：**
```
[2025-01-16 10:30:15] HoneyGUI Designer 启动
[2025-01-16 10:30:16] 加载 HML 文件: /home/user/project/ui/main.hml
[2025-01-16 10:30:18] 错误: 无法解析组件定义
```

---

### 2️⃣ **Webview 开发者工具**

Webview UI（设计器界面）的控制台日志。

**查看方法：**
1. 打开 HoneyGUI 设计器（在 HML 文件中右键 → "Open in HoneyGUI Designer"）
2. 在设计器界面中，按 **Ctrl+Shift+I**（Windows/Linux）或 **Cmd+Option+I**（Mac）
3. 这将打开 Chrome 开发者工具
4. 切换到 **Console** 标签页

**应该提供什么：**
- 所有的错误和警告（红色和黄色消息）
- 相关的 JavaScript 错误堆栈

**示例：**
```
[Error] TypeError: Cannot read property 'id' of undefined
    at DesignerCanvas.tsx:156
    at Array.map (<anonymous>)
```

---

### 3️⃣ **VS Code 开发者工具**

整个 VS Code 的日志，包含扩展加载信息。

**查看方法：**
1. 在 VS Code 中，按 **Ctrl+Shift+P** 打开命令面板
2. 输入并运行：**Developer: Toggle Developer Tools**
3. 切换到 **Console** 标签页

**应该提供什么：**
- 扩展加载错误（红色错误消息）
- 模块加载失败信息

---

### 4️⃣ **调试控制台**

当你使用 F5 调试模式时的专用控制台。

**查看方法：**
1. 按 **F5** 启动调试（打开新的 VS Code 窗口）
2. 在原始窗口底部，切换到 **"调试控制台"** 标签

**应该提供什么：**
- 断点信息
- 变量值
- 调用堆栈

---

## 🎯 高效反馈问题的方法

### 极简版（快速反馈）

复制粘贴以下内容，填入你的信息：

```
**问题描述：**
[简单描述遇到了什么问题]

**复现步骤：**
1. [第一步]
2. [第二步]
3. [发生错误]

**扩展输出通道的关键日志：**
```
[粘贴最后 10-20 行日志]
```

**环境信息：**
- 操作系统：[Windows/Mac/Linux]
- VS Code 版本：[帮助 → 关于]
```

### 完整版（复杂问题）

对于复杂问题，请按以下结构提供信息：

#### 1. 基本信息
```markdown
- **问题类型**: [崩溃/功能异常/性能问题/其他]
- **严重程度**: [ blocker / high / medium / low ]
- **首次出现**: [今天/上周/一直存在]
- **能否稳定复现**: [能/不能]
```

#### 2. 问题描述
```markdown
**期望行为：**
[描述你期望发生什么]

**实际行为：**
[描述实际发生了什么]

**错误信息：**
[如果有具体的错误消息，请完整复制]
```

#### 3. 复现步骤
```markdown
1. [精确的第一步，包含文件路径]
2. [精确的第二步，包含所有操作]
3. ...
4. [出现问题的结果]
```

#### 4. 日志信息（按重要性排序）

**优先级1: 扩展输出通道**
```
[粘贴完整的 "HoneyGUI Design" 输出]
```

**优先级2: Webview Console**
```
[粘贴开发者工具 Console 中的错误]
```

**优先级3: 截图（如果适用）**
[在开发者工具中按 F12 截图，或右键 → 检查 → 截图控制台]

#### 5. 环境信息
```markdown
- OS: [Windows 10/ macOS 14/ Ubuntu 22.04]
- VS Code Version: [帮助 → 关于]
- Node.js Version: `node --version`
- 是否在开发模式下: [是 (F5调试) / 否 (直接安装)]
```

#### 6. 相关文件（如果适用）
```markdown
如果问题与特定文件相关，请提供：
- HML 文件内容
- 项目结构截图
- package.json
```

## 📸 如何截屏（最有帮助）

### 对于 UI 问题
1. 直接截图整个 VS Code 窗口
2. 用红框标注问题区域

### 对于代码生成问题
1. 截图扩展输出通道的完整错误日志
2. 截图生成的代码（如果有）

### 对于 Webview 问题
1. 按 **Ctrl+Shift+I** 打开开发者工具
2. 截图 Console 标签页的错误信息
3. 截图 Network 标签页（如果加载失败）

## 🔧 常见问题快速诊断

### 问题1: 扩展无法启动

**检查清单:**
- [ ] 运行 `npm run compile` 是否成功？
- [ ] 扩展输出通道是否有 "HoneyGUI Designer 启动"？
- [ ] VS Code 开发者工具中有红色错误吗？

**需要提供的信息:**
- `npm run compile` 的完整输出
- VS Code 开发者工具 Console 的截图

---

### 问题2: Webview 一片空白

**检查清单:**
- [ ] 是否运行了 `npm run serve:webview`？(开发模式)
- [ ] 按 Ctrl+Shift+I 查看 Console 有什么错误？
- [ ] 浏览器中访问 http://localhost:3000 是否正常？

**需要提供的信息:**
- Webview 开发者工具 Console 的完整截图
- `npm run serve:webview` 终端的输出

---

### 问题3: 代码生成失败

**检查清单:**
- [ ] HML 文件语法是否正确？
- [ ] 输出目录是否有写入权限？
- [ ] 扩展输出通道显示什么错误？

**需要提供的信息:**
- 完整的扩展输出日志
- HML 文件内容
- 目标目录的文件权限

---

## 📤 如何向 AI 发送日志

### 最佳实践

1. **使用 Markdown 代码块**
   ```markdown
   ```
   [您的日志内容]
   ```
   ```

2. **只提供相关信息**：不要粘贴2000行日志，只提供错误前后的关键部分

3. **多个日志来源**：分区域标注日志来源

4. **如果日志太长**：
   - 只粘贴错误消息和前后10行
   - 或者上传到 GitHub Gist，提供链接

### 示例反馈格式

```markdown
**发现问题：** 点击 "Open in HoneyGUI Designer" 后，设计器界面空白

**复现步骤：**
1. 在 HML 文件中右键
2. 选择 "Open in HoneyGUI Designer"
3. 面板打开，但内容区域空白

**扩展输出通道日志：**
```
[2025-01-16 10:45:21] HoneyGUI Designer 启动
[2025-01-16 10:45:22] 正在创建 webview 面板
[2025-01-16 10:45:23] 错误: Cannot find module './webview/index.html'
```

**Webview Console 截图：**
[List item]
 0: "GET http://localhost:3000/index.html net::ERR_CONNECTION_REFUSED"

**环境：** Windows 11, VS Code 1.85.0
```

## 🛠️ 扩展调试命令

### 已配置的命令

我们添加了以下快捷命令来帮助你调试：

#### 1. 打开扩展日志目录
```
命令面板 → "HoneyGUI: Open Logs Directory"
```
这会打开日志文件所在文件夹，你可以：
- 查看历史日志文件
- 删除旧日志文件
- 导出日志

#### 2. 显示扩展信息
```
命令面板 → "HoneyGUI: Show Extension Info"
```
显示：
- 扩展版本
- 安装路径
- 配置信息

#### 3. 重启扩展（开发模式）
```
命令面板 → "Developer: Reload Window"
```
这会重新加载整个 VS Code 窗口，相当于重启扩展。

## 🚀 开发模式调试技巧

### 当你使用 F5 调试时

1. **设置断点**：
   - 在 TypeScript 源码中点击行号左侧
   - 红点表示断点已设置

2. **查看变量**：
   - 在调试控制台输入变量名
   - 或使用调试面板的 "Variables" 视图

3. **单步执行**：
   - F10: Step Over (跳过函数)
   - F11: Step Into (进入函数)
   - Shift+F11: Step Out (跳出函数)

4. **观看表达式**：
   - 在调试面板的 "Watch" 添加表达式
   - 实时查看变量值变化
