# 视频转换方案升级总结

## 🔄 方案变更概述

将视频转换从 **HoneyGUI SDK Python 工具** 升级为 **FFmpeg**，提供更好的跨平台兼容性和功能。

## 📋 主要变更

### 1. VideoConverterService.ts - 完全重构

**构造函数变更**：
```typescript
// 旧版本
constructor(sdkPath: string)

// 新版本  
constructor() // 不再需要 SDK 路径
```

**新增功能**：
- `checkFFmpegAvailable()` - 检查 FFmpeg 可用性
- `getVideoInfo()` - 获取视频详细信息
- `parseFFmpegError()` - 智能错误解析
- 支持更多输入格式（15+ 种）
- 转换性能监控（耗时统计）
- **视频裁剪功能** - 支持裁剪指定区域
- **视频缩放功能** - 支持缩放到指定尺寸，可选保持宽高比
- **组合处理** - 支持同时应用裁剪和缩放

### 2. BuildCore.ts - 集成 FFmpeg 检查

```typescript
// 新增 FFmpeg 可用性检查
const ffmpegAvailable = await videoConverter.checkFFmpegAvailable();
if (!ffmpegAvailable) {
    this.logger.log('FFmpeg 未找到，跳过视频转换', true);
}
```

### 5. 文档更新

- `视频控件使用说明.md` - 更新安装和故障排查

## 🔧 命令对比

### MJPEG 格式
```bash
# 旧方案
python -m video_converter -i input.mp4 -o output.mjpeg -f mjpeg -q 85

# 新方案
ffmpeg -i input.mp4 -c:v mjpeg -q:v 8 -c:a pcm_s16le output.mjpeg
```

### AVI 格式
```bash
# 旧方案
python -m video_converter -i input.mp4 -o output.avi -f avi_mjpeg -r 25

# 新方案
ffmpeg -i input.mp4 -c:v mjpeg -r 25 -c:a pcm_s16le -f avi output.avi
```

### H.264 格式
```bash
# 旧方案
python -m video_converter -i input.mp4 -o output.h264 -f h264

# 新方案
ffmpeg -r 30 -i input.mp4 -c:v libx264 -x264-params "cabac=0:ref=3:deblock=1:0:0:analyse=0x1:0x111:me=hex:subme=7:psy=1:psy_rd=1.0:0.0:mixed_ref=1:me_range=16:chroma_me=1:trellis=1:8x8dct=0:deadzone-inter=21:deadzone-intra=11:fast_pskip=1:chroma_qp_offset=-2:threads=11:lookahead_threads=1:sliced_threads=0:nr=0:decimate=1:interlaced=0:bluray_compat=0:constrained_intra=0:bframes=0:weightp=0:keyint=40:min-keyint=4:scenecut=40:intra_refresh=0:rc_lookahead=40:mbtree=1:crf=23:qcomp=0.60:qpmin=0:qpmax=69:qpstep=4:ipratio=1.40:aq-mode=1:aq-strength=1.00" -an -f rawvideo output.h264
```

## 🐛 潜在问题

### 1. FFmpeg 版本兼容性
**问题**：不同版本的 FFmpeg 可能有不同的编码器支持
**解决**：建议使用 FFmpeg 4.0+，在文档中说明

### 2. 系统 PATH 配置
**问题**：用户可能未正确配置 FFmpeg 到 PATH
**解决**：提供详细的安装指南和故障排查

### 3. 编码器缺失
**问题**：某些系统可能缺少特定编码器（如 libx264）
**解决**：在错误信息中提供安装建议
