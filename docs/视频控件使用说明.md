# 视频控件使用说明

## 概述

HoneyGUI Design 支持视频控件 (`hg_video`)，可以在嵌入式 GUI 应用中播放视频。视频文件会在编译时自动转换为 HoneyGUI 支持的格式。

## 功能特性

### 支持的输入格式
- MP4
- AVI
- MOV
- MKV
- WebM
- FLV
- WMV

### 支持的输出格式
1. **MJPEG** - Motion JPEG
   - 高质量视频
   - 文件较大
   - 支持质量设置 (0-100)
   - **编码要求**: baseline yuv420p 像素格式
   
2. **AVI (MJPEG)** - AVI 容器 + MJPEG 编码
   - 兼容性好
   - 支持帧率设置
   - **编码要求**: baseline yuv420p 像素格式，PCM 16-bit 音频
   
3. **H.264** - H.264 编码
   - 压缩率高
   - 文件较小
   - 输出原始 H.264 流（.h264 文件）
   - **编码要求**: 特定的 x264 参数配置，无音频输出

## 使用流程

### 1. 添加视频组件

**方法 1：从组件库拖拽**
1. 打开设计器
2. 从左侧组件库的"多媒体"分类中找到"视频"组件
3. 拖拽到画布上的容器中（必须在 `hg_view` 或 `hg_window` 内）

**方法 2：从资源面板拖拽**
1. 将视频文件（如 `.mp4`）放入项目的 `assets/` 目录
2. 打开资源面板（左侧 Tab 2）
3. 直接拖拽视频文件到画布上的容器中
4. 自动创建视频组件并设置路径

### 2. 配置视频属性

在右侧属性面板中配置（支持滚轮滚动查看所有选项）：

#### 基础属性
- **ID**: 组件唯一标识（自动生成）
- **名称**: 组件名称（用于代码生成）
- **视频路径**: 视频文件路径（如 `assets/video.mp4`）

#### 位置和尺寸
- **X, Y**: 组件位置
- **宽度, 高度**: 组件尺寸

#### 转换设置
- **输出格式**: 选择 MJPEG / AVI / H.264
- **质量** (仅 MJPEG/AVI): 0-100，默认 85
- **帧率**: 输出帧率，默认值因格式而异
  - MJPEG: 默认 30fps
  - AVI: 默认 25fps  
  - H.264: 默认 30fps

#### 视频处理（高级功能）
- **裁剪**: 从原视频中裁剪指定区域
  - 裁剪坐标 (x, y): 裁剪起始位置
  - 裁剪尺寸 (width, height): 裁剪区域大小
- **缩放**: 将视频缩放到指定尺寸
  - 目标尺寸 (width, height): 输出视频尺寸
  - 保持宽高比: 是否保持原始比例（不足部分用黑边填充）

#### 播放设置
- **播放帧率**: 运行时播放帧率，默认 30
- **自动播放**: 组件创建后自动开始播放

### 3. 生成代码

点击工具栏"生成代码"按钮，自动生成 C 代码：

```c
// 创建视频组件
gui_video_t *video = gui_video_create_from_fs(
    view, 
    "video", 
    "/video.mjpeg",  // VFS 路径
    0, 0, 410, 502
);

// 设置帧率
gui_video_set_frame_rate(video, 30.f);

// 设置播放状态
gui_video_set_state(video, GUI_VIDEO_STATE_PLAYING);
```

### 4. 编译仿真

执行 `Ctrl+Shift+P → HoneyGUI: Compile & Simulate`

编译流程会自动：
1. **转换视频格式**: 调用 SDK 的 `video_converter` 工具
2. **打包 romfs**: 将转换后的视频打包到 romfs 文件系统
3. **编译代码**: 生成可执行文件
4. **启动仿真**: 运行 SDL2 仿真窗口

## 视频转换详解

### 转换命令示例

编译时会自动调用 FFmpeg：

```bash
# MJPEG 格式（带质量设置）
ffmpeg -i birds.mp4 -c:v mjpeg -q:v 8 -c:a pcm_s16le birds.mjpeg

# AVI 格式（带帧率设置）
ffmpeg -i birds.mp4 -c:v mjpeg -r 25 -c:a pcm_s16le -f avi birds.avi

# H.264 格式（带质量设置）
ffmpeg -i birds.mp4 -c:v libx264 -crf 23 -c:a aac -f mp4 birds.mp4
```

### 前置依赖

**必需**：
- **FFmpeg**：视频转换工具，需要添加到系统 PATH 环境变量
  - Windows: 下载 FFmpeg 并添加到 PATH
  - Linux: `sudo apt install ffmpeg` 或 `sudo yum install ffmpeg`
  - macOS: `brew install ffmpeg`

**可选**：
- **FFprobe**：视频信息获取工具（通常与 FFmpeg 一起安装）

### 项目级配置

在 `project.json` 中可以设置默认转换选项：

```json
{
  "name": "my-project",
  "videoFormat": "mjpeg",
  "videoQuality": 85,
  "videoFrameRate": 30
}
```

这些设置会作为所有视频组件的默认值，但可以在组件属性中单独覆盖。

## 代码生成策略

### 路径转换

设计器中使用原始视频路径，代码生成时自动转换：

```
设计器路径: assets/video.mp4
    ↓ (根据格式设置)
转换后路径: assets/video.mjpeg
    ↓ (去掉 assets/ 前缀)
VFS 路径: /video.mjpeg
```

### 生成的 C 代码结构

```c
// 在 *_ui.c 中
gui_obj_t *video_component = NULL;

void create_ui_components(gui_obj_t *parent) {
    // 创建视频组件
    video_component = (gui_obj_t *)gui_video_create_from_fs(
        parent,
        "video_name",
        "/video.mjpeg",
        x, y, width, height
    );
    
    // 设置帧率
    gui_video_set_frame_rate((gui_video_t *)video_component, 30.f);
    
    // 设置自动播放
    gui_video_set_state((gui_video_t *)video_component, GUI_VIDEO_STATE_PLAYING);
}
```

## 最佳实践

### 1. 选择合适的格式

| 场景 | 推荐格式 | 原因 |
|------|----------|------|
| 高质量短视频 | MJPEG | 质量好，解码简单 |
| 长视频 | H.264 | 文件小，节省存储 |
| 兼容性优先 | AVI | 广泛支持 |

### 2. 优化视频尺寸

- 视频分辨率不要超过屏幕分辨率
- 帧率建议 15-30 FPS
- 质量设置 70-85 之间平衡文件大小和质量

### 3. 测试流程

1. 先用小视频测试转换流程
2. 确认播放正常后再使用完整视频
3. 注意检查 romfs 打包后的文件大小

## 故障排查

### 问题 1: 视频转换失败

**错误信息**: `FFmpeg not found in system PATH`

**解决方案**:
1. 安装 FFmpeg：
   - Windows: 从 https://ffmpeg.org/download.html 下载并添加到 PATH
   - Linux: `sudo apt install ffmpeg`
   - macOS: `brew install ffmpeg`
2. 验证安装：在命令行运行 `ffmpeg -version`
3. 重启 VS Code 以刷新环境变量

### 问题 2: 视频格式不支持

**错误信息**: `Invalid data found when processing input`

**解决方案**:
1. 检查输入视频文件是否损坏
2. 尝试使用其他视频格式（MP4, AVI, MOV）
3. 检查视频编码是否被 FFmpeg 支持

### 问题 3: 视频不播放

**可能原因**:
1. 视频路径错误
2. 视频格式不支持
3. romfs 未正确挂载

**解决方案**:
1. 检查生成的 C 代码中的路径
2. 确认视频已转换为支持的格式
3. 检查入口文件中的 `gui_vfs_mount_romfs` 调用

## 示例项目

### 完整示例

```xml
<!-- ui/main.hml -->
<hg_view id="main_view" x="0" y="0" w="480" h="272">
  <hg_video 
    id="video1" 
    name="demo_video" 
    x="40" 
    y="36" 
    w="400" 
    h="200">
    <property name="src" value="assets/demo.mp4"/>
    <property name="format" value="mjpeg"/>
    <property name="quality" value="85"/>
    <property name="frameRate" value="30"/>
    <property name="autoPlay" value="true"/>
  </hg_video>
</hg_view>
```

生成的代码会自动处理视频转换和播放设置。

## API 参考

### HoneyGUI 视频 API

```c
// 创建视频组件
gui_video_t *gui_video_create_from_fs(
    gui_obj_t *parent,
    const char *name,
    const char *path,
    int x, int y,
    int w, int h
);

// 设置帧率
void gui_video_set_frame_rate(gui_video_t *video, float fps);

// 设置播放状态
void gui_video_set_state(gui_video_t *this, GUI_VIDEO_STATE state);

// 播放状态枚举
typedef enum
{
    GUI_VIDEO_STATE_ERR,
    GUI_VIDEO_STATE_INIT,
    GUI_VIDEO_STATE_PLAYING,
    GUI_VIDEO_STATE_PAUSE,
    GUI_VIDEO_STATE_STOP,
} GUI_VIDEO_STATE;
```
