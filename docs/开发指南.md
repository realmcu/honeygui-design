# HoneyGUI Design 开发指南

详细架构设计请参考 [架构设计](架构设计.md)。

## 开发命令

### 日常开发
```bash
npm install               # 安装依赖
npm run watch             # 监听扩展代码
npm run watch:webview     # 监听 webview 代码
# 按 F5 启动调试
```

### 构建
```bash
npm run compile           # 编译扩展
npm run build:webview     # 构建 webview（生产）
npm run vscode:prepublish # 发布前完整构建
vsce package              # 打包 VSIX
```

### 测试
```bash
npm test                  # 运行测试
npm run lint              # 代码检查
```

## 添加新组件

1. **类型定义** (`src/webview/types.ts`)：添加组件类型

2. **组件库** (`src/webview/components/ComponentLibrary.tsx`)：
   ```typescript
   { type: 'hg_xxx', name: '组件名', icon: '📦', category: 'basic' }
   ```

3. **渲染器** (`src/webview/components/widgets/`)：创建 `XxxWidget.tsx`

4. **属性编辑** (`src/webview/components/properties/`)：创建属性面板

5. **代码生成** (`src/codegen/honeygui/HoneyGuiCCodeGenerator.ts`)：添加生成逻辑

## 添加新模板

1. 在 `template-projects/` 创建模板目录
2. 添加 `project.json` 和 `ui/main/XXXMain.hml`
3. 在 `src/template/templates/` 创建模板类
4. 在 `src/template/templates/index.ts` 注册

## 调试技巧

### 扩展端
- 在 VS Code 中设置断点，按 F5 启动调试
- 查看 Output 面板 "HoneyGUI Design" 通道

### Webview 端
- 运行 `npm run serve:webview`，访问 http://localhost:3000
- 使用浏览器开发者工具调试

### 常见问题
- **通信问题**：检查 postMessage 日志
- **构建问题**：删除 `out/` 目录后重新构建
- **依赖问题**：删除 `node_modules/` 后重新 `npm install`

## 代码规范

- TypeScript 严格模式
- React 函数组件 + Hooks
- CSS 模块化（每个组件独立 CSS）
- 使用 VSCode 主题变量 (`var(--vscode-*)`)

## 代码复用

- **分辨率解析**：`ProjectUtils.parseResolution(resolution)`
- **项目配置**：`ProjectUtils.loadProjectConfig(projectRoot)`
- **路径获取**：`ProjectUtils.getAssetsDir()` / `getUiDir()` / `getSrcDir()`
