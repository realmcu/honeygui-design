## 项目概述

HoneyGUI Visual Designer 是一个 VS Code 扩展，提供可视化拖放界面来创建 HoneyGUI UI 界面。它能生成适用于嵌入式应用的 C 代码，并支持代码保护区功能。

**关键技术：**
- TypeScript 5.3
- React 19 + Zustand（用于 webview UI）
- VS Code 扩展 API
- Webpack 5（webview 构建）
- HML（HoneyGUI 标记语言 - 自定义类 XML 格式）

## 架构设计

### 模块结构

```
src/
├── extension.ts                    # 扩展入口点
├── core/                           # 核心管理模块
│   ├── ExtensionManager.ts         # 扩展生命周期管理
│   ├── CommandManager.ts           # 命令注册和处理
│   └── CollaborationService.ts     # 协同开发服务
├── webview/                        # React webview UI
│   ├── index.tsx                   # React 应用入口
│   ├── App.tsx                     # 主应用组件
│   ├── store.ts                    # Zustand 状态管理
│   ├── components/                 # UI 组件（画布、属性面板等）
│   ├── hooks/                      # React Hooks
│   └── utils/                      # 工具函数
├── designer/                       # 设计器模块
│   ├── DesignerPanel.ts            # Webview 面板管理
│   ├── MessageHandler.ts           # 消息处理器
│   └── SaveManager.ts              # 保存管理
├── hml/                            # HML处理模块
│   ├── HmlParser.ts                # 解析 HML 到组件
│   └── HmlSerializer.ts            # 序列化组件到 HML
├── codegen/                        # 代码生成模块
│   └── honeygui/                   # HoneyGUI C代码生成
├── simulation/                     # 编译仿真模块
│   ├── SimulationService.ts        # 仿真服务
│   ├── BuildCore.ts                # 编译核心逻辑
│   └── EnvironmentChecker.ts       # 环境检查
├── tools/                          # 资源转换工具
└── template/                       # 项目模板
```

### 状态管理

**扩展端：**
- `HmlController` - 管理HML文档状态和版本
- `SaveManager` - 管理保存事务和状态
- `CollaborationService` - 管理协同会话状态

**Webview 端（Zustand）：**
- 组件增删改查操作
- 选择状态管理
- 撤销/重做（命令模式，50 步历史）
- 画布状态（缩放、平移、网格设置）

### 通信流程

**扩展 → Webview:**
```typescript
{ command: 'loadHml', components: [], projectConfig: {} }
{ command: 'collaboration:update', components: [] }
{ command: 'codegenComplete', success: boolean }
```

**Webview → 扩展:**
```typescript
{ command: 'save', components: [] }
{ command: 'codegen', language: 'c' }
{ command: 'updateComponent', id: '', updates: {} }
```

### HML 文件格式

```xml
<?xml version="1.0" encoding="UTF-8"?>
<document>
  <meta>
    <resolution width="480" height="272"/>
  </meta>
  <view>
    <hg_button id="btn1" x="10" y="10" width="100" height="40" label="Click"/>
  </view>
</document>
```

## 开发命令

### 日常开发
```bash
npm install               # 安装依赖
npm run watch             # 监听扩展代码
npm run watch:webview     # 监听 webview 代码
# 按 F5 启动调试
```

### 构建
```bash
npm run compile           # 编译扩展
npm run build:webview     # 构建 webview（生产）
npm run vscode:prepublish # 发布前完整构建
npm run package           # 打包 VSIX
```

### 测试与检查
```bash
npm test                  # 运行模板测试
npm run lint              # 代码检查
npm run clean             # 清理缓存
npm run rebuild           # 完整重建
```

## 开发流程

### 添加新组件

1. **类型定义** (`src/webview/types.ts`)：
   ```typescript
   export type ComponentType = 'hg_button' | 'hg_xxx' | ...;
   ```

2. **组件库** (`src/webview/components/ComponentLibrary.tsx`)：
   ```typescript
   { type: 'hg_xxx', name: '组件名', icon: '📦', category: 'basic' }
   ```

3. **渲染器** (`src/webview/components/widgets/`)：
   创建 `XxxWidget.tsx` 实现组件在画布上的渲染

4. **属性编辑** (`src/webview/components/properties/`)：
   创建属性面板组件

5. **代码生成** (`src/codegen/honeygui/HoneyGuiCCodeGenerator.ts`)：
   添加生成 C 代码的逻辑

### 添加新模板

1. 在 `template-projects/` 创建模板目录
2. 添加 `project.json` 和 HML 文件
3. 在 `src/template/TemplateConfig.ts` 注册模板信息

### 代码复用工具

- **分辨率解析**：`ProjectUtils.parseResolution(resolution)`
- **项目配置**：`ProjectUtils.loadProjectConfig(projectRoot)`
- **路径获取**：`ProjectUtils.getAssetsDir()` / `getUiDir()` / `getSrcDir()`

## 测试指南

### 运行测试

```bash
npm test
```

### 测试流程

每个模板测试执行以下步骤：

```
[1/5] 查找 HML 文件
[2/5] 解析 HML (HmlParser)
[3/5] 生成 C 代码 (HoneyGuiCCodeGenerator)
[4/5] 编译 (BuildCore + SCons)
[5/5] 运行仿真 (2秒)
```

### 测试覆盖的模板

| 模板 | 分辨率 | 描述 |
|------|--------|------|
| rotary | 480x480 | 圆形表盘界面 |
| settings | 480x272 | 设置界面 |
| smartwatch | 410x502 | 智能手表界面 |
| dashboard | 800x480 | 仪表盘界面 |
| chatbot | 480x320 | 聊天机器人界面 |

### 前置依赖

- **SCons**: `pip install scons`
- **GCC**: C 编译器
- **Python**: 用于 romfs 打包

### 环境检查

在 VSCode 侧边栏的 **ENVIRONMENT** 面板中可以检查：
1. Python - 必需
2. SCons - 必需
3. 编译器 (GCC/MinGW) - 必需
4. FFmpeg - 可选（用于视频转换）

### 添加新模板测试

1. 在 `template-projects/` 下创建模板目录
2. 添加 `ui/main/XXXMain.hml` 文件
3. 在测试脚本的 `templates` 数组中添加模板名称

## 调试技巧

### 扩展端调试
- 在 VS Code 中设置断点，按 F5 启动调试
- 查看 Output 面板 "HoneyGUI Design" 通道
- 使用 `logger.info()` / `logger.error()` 输出日志

### Webview 端调试
- 在设计器中按 F12 打开开发者工具
- 或运行 `npm run serve:webview`，访问 http://localhost:3000
- 使用浏览器开发者工具调试 React 应用

### 常见问题

**通信问题：**
- 检查 postMessage 日志
- 确认消息格式是否正确

**构建问题：**
- 删除 `out/` 目录后重新构建
- 运行 `npm run clean` 清理缓存

**依赖问题：**
- 删除 `node_modules/` 后重新 `npm install`
- 检查 package.json 中的版本是否兼容

**编译失败：**
- 检查 HML 文件是否有嵌套的 `hg_view`（不允许）
- 确认使用的组件类型是否被支持
- 查看编译输出中的错误信息

## 代码规范

### TypeScript
- 使用严格模式 (`strict: true`)
- 优先使用接口而非类型别名
- 明确标注返回类型

### React
- 使用函数组件 + Hooks
- 避免不必要的重渲染（使用 useMemo/useCallback）
- Props 使用接口定义

### 样式
- CSS 模块化（每个组件独立 CSS 文件）
- 使用 VSCode 主题变量 (`var(--vscode-*)`)
- 避免硬编码颜色值

### 命名约定
- 组件文件：PascalCase（如 `ComponentLibrary.tsx`）
- 工具函数：camelCase（如 `parseResolution`）
- 常量：UPPER_SNAKE_CASE（如 `DEFAULT_GRID_SIZE`）

## 已知限制

1. **预览功能**：快速预览尚未实现，当前仅支持完整编译仿真
2. **性能**：大型组件树（100+组件）可能影响性能
3. **协同开发**：仅同步HML结构，资源文件需手动同步
4. **平台支持**：主要在Windows和Linux测试，macOS支持有限
