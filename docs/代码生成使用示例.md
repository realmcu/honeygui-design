# 代码生成使用示例

## 基本使用

### 1. 单个HML文件

**项目结构：**
```
my-project/
└── ui/
    └── main/
        └── main.hml
```

**生成后：**
```
my-project/
├── ui/
│   └── main/
│       └── main.hml
└── src/
    └── main/
        ├── main.h
        ├── main.c
        ├── main_callbacks.h
        └── main_callbacks.c
```

**调用方式：**
```c
#include "src/main/main.h"

int main(void) {
    // 初始化HoneyGUI
    gui_init();
    
    // 初始化界面
    main_init();
    
    // 主循环
    while (1) {
        gui_task();
        main_update();
    }
    
    return 0;
}
```

## 多HML文件

### 2. 多个界面

**项目结构：**
```
my-project/
└── ui/
    ├── main/
    │   └── main.hml
    ├── settings/
    │   └── settings.hml
    └── about/
        └── about.hml
```

**生成后：**
```
my-project/
├── ui/
│   ├── main/
│   │   └── main.hml
│   ├── settings/
│   │   └── settings.hml
│   └── about/
│       └── about.hml
└── src/
    ├── main/
    │   ├── main.h
    │   ├── main.c
    │   ├── main_callbacks.h
    │   └── main_callbacks.c
    ├── settings/
    │   ├── settings.h
    │   ├── settings.c
    │   ├── settings_callbacks.h
    │   └── settings_callbacks.c
    └── about/
        ├── about.h
        ├── about.c
        ├── about_callbacks.h
        └── about_callbacks.c
```

**调用方式：**
```c
#include "src/main/main.h"
#include "src/settings/settings.h"
#include "src/about/about.h"

int main(void) {
    gui_init();
    
    // 初始化所有界面
    main_init();
    settings_init();
    about_init();
    
    // 显示主界面
    gui_obj_show(mainView, true);
    gui_obj_show(settingsView, false);
    gui_obj_show(aboutView, false);
    
    while (1) {
        gui_task();
    }
    
    return 0;
}
```

## 目录结构说明

### 3. 标准项目结构

**完整项目：**
```
my-project/
├── ui/                      # UI定义目录
│   ├── main/               # 主界面模块
│   │   └── main.hml
│   ├── settings/           # 设置界面模块
│   │   └── settings.hml
│   └── dialogs/            # 对话框模块
│       └── confirm.hml
├── src/                    # 生成的C代码目录
│   ├── main/
│   │   ├── main.h
│   │   ├── main.c
│   │   ├── main_callbacks.h
│   │   └── main_callbacks.c
│   ├── settings/
│   │   ├── settings.h
│   │   ├── settings.c
│   │   ├── settings_callbacks.h
│   │   └── settings_callbacks.c
│   └── dialogs/
│       ├── confirm.h
│       ├── confirm.c
│       ├── confirm_callbacks.h
│       └── confirm_callbacks.c
├── assets/                 # 资源文件
│   ├── images/
│   └── fonts/
├── project.json           # 项目配置
└── README.md              # 项目说明
```

**关键规则：**
- 每个HML文件必须在自己的二级目录下
- 目录名通常与HML文件名相同
- `ui/xxx/xxx.hml` → `src/xxx/xxx.c`

## 代码保护区

### 4. 保留用户代码

**首次生成 (main_callbacks.c)：**
```c
#include "main_callbacks.h"
#include <stdio.h>

/* @protected start on_start_click */
void on_start_click(gui_obj_t *obj) {
    printf("Start button clicked\n");
    // TODO: 实现事件处理逻辑
}
/* @protected end on_start_click */
```

**用户修改后：**
```c
#include "main_callbacks.h"
#include <stdio.h>

/* @protected start on_start_click */
void on_start_click(gui_obj_t *obj) {
    printf("Starting application...\n");
    
    // 用户添加的代码
    init_app_state();
    load_user_settings();
    show_welcome_screen();
}
/* @protected end on_start_click */
```

**重新生成后：**
- 保护区内的代码被保留
- 保护区外的代码被更新
- 新增的组件会生成新的回调函数

## 代码生成器API

### TypeScript调用示例

```typescript
import { HoneyGuiCCodeGenerator } from './codegen/honeygui/HoneyGuiCCodeGenerator';
import { getOutputDir, getBaseName } from './codegen/honeygui/utils';

// HML文件路径
const hmlFilePath = '/path/to/project/ui/main/main.hml';
const projectRoot = '/path/to/project';

// 计算输出目录
const outputDir = getOutputDir(hmlFilePath, projectRoot);  // '/path/to/project/src/main'
const baseName = getBaseName(hmlFilePath);                 // 'main'

// 组件数据
const components = [
  {
    id: 'mainView',
    type: 'hg_view',
    name: '主视图',
    position: { x: 0, y: 0, width: 480, height: 272 },
    parent: null,
    children: ['titleLabel', 'startButton']
  },
  // ... 更多组件
];

// 创建生成器
const generator = new HoneyGuiCCodeGenerator(components, {
  outputDir,
  hmlFileName: baseName,
  enableProtectedAreas: true
});

// 生成代码
const result = await generator.generate();

if (result.success) {
  console.log('生成成功！');
  console.log('生成的文件：', result.files);
  // 输出：
  // /path/to/project/src/main/main.h
  // /path/to/project/src/main/main.c
  // /path/to/project/src/main/main_callbacks.h
  // /path/to/project/src/main/main_callbacks.c
} else {
  console.error('生成失败：', result.errors);
}
```

## 注意事项

1. **目录结构**：
   - HML文件必须在 `ui/` 目录下的二级目录中
   - 例如：`ui/main/main.hml`，而不是 `ui/main.hml`
   - 目录名通常与HML文件名相同

2. **文件命名**：HML文件名应使用有效的C标识符（字母、数字、下划线）

3. **保护区**：不要手动修改保护区标记，否则代码可能丢失

4. **重新生成**：修改HML后重新生成，用户代码会被保留

5. **多文件**：每个HML独立生成，互不影响

6. **路径映射**：
   - `ui/xxx/xxx.hml` → `src/xxx/xxx.c`
   - 自动创建不存在的目录
