# HoneyGUI Design 技术栈和工作流程

## 技术栈概览

### 前端技术

#### Webview UI
- **React 19.2.0** - UI框架
- **Zustand 5.0.8** - 状态管理
- **Lucide React 0.553.0** - 图标库
- **TypeScript 5.3** - 类型系统

#### 构建工具
- **Webpack 5.102.1** - 模块打包
- **ts-loader 9.5.4** - TypeScript编译
- **css-loader / style-loader** - 样式处理
- **HtmlWebpackPlugin** - HTML生成

### 后端技术

#### VSCode扩展
- **VSCode Extension API 1.80+** - 扩展开发
- **TypeScript 5.3** - 开发语言
- **Node.js 20+** - 运行环境

#### 数据处理
- **fast-xml-parser 4.3.2** - XML解析
- **xml-js 1.6.11** - XML处理
- **chokidar 3.6.0** - 文件监听

#### 网络通信
- **ws 8.18.3** - WebSocket服务器/客户端

### 编译工具链

#### C代码编译
- **SCons** - 构建系统
- **GCC / MinGW** - C编译器
- **HoneyGUI SDK** - GUI框架

#### 仿真运行
- **SDL2** - 图形库
- **OpenGL ES** - 图形渲染

## 完整工作流程

### 1. 项目创建流程

```
用户触发命令
  ↓
CommandManager.registerCommand('honeygui.newProject')
  ↓
CreateProjectPanel.createOrShow()
  ↓
显示Webview表单（项目名称、路径、分辨率）
  ↓
用户提交表单
  ↓
TemplateManager.createProject()
  ↓
创建目录结构：
  - ui/main/main.hml
  - src/autogen/
  - assets/
  - project.json
  ↓
打开main.hml在设计器中
```

### 2. 设计器打开流程

```
用户双击.hml文件
  ↓
HmlEditorProvider.resolveCustomTextEditor()
  ↓
DesignerPanel.createOrShow()
  ↓
创建Webview面板
  ↓
WebviewContentProvider.getHtmlContent()
  ↓
加载React应用（webview.js）
  ↓
Webview发送'ready'消息
  ↓
HmlController.loadFile()
  ↓
HmlParser.parse() → Component[]
  ↓
发送'loadHml'消息到Webview
  ↓
Zustand store.setComponents()
  ↓
DesignerCanvas渲染组件
```

### 3. 组件操作流程

#### 添加组件
```
用户从ComponentLibrary拖拽
  ↓
DesignerCanvas.onDrop()
  ↓
store.addComponent()
  ↓
发送'addComponent'消息到扩展
  ↓
MessageHandler.handleMessage()
  ↓
ComponentManager.addComponent()
  ↓
HmlController更新内存模型
  ↓
SaveManager.save()
  ↓
HmlSerializer.serialize()
  ↓
写入.hml文件
```

#### 更新属性
```
用户在PropertiesPanel修改
  ↓
store.updateComponent()
  ↓
发送'updateComponent'消息
  ↓
MessageHandler处理
  ↓
ComponentManager.updateComponent()
  ↓
自动保存（防抖）
```

#### 删除组件
```
用户按Delete键
  ↓
keyboardShortcuts处理
  ↓
store.removeComponent()
  ↓
发送'removeComponent'消息
  ↓
MessageHandler处理
  ↓
ComponentManager.removeComponent()
  ↓
自动保存
```

### 4. 代码生成流程

```
用户点击"生成代码"按钮
  ↓
Webview发送'codegen'消息
  ↓
CodeGenManager.generateCode()
  ↓
HmlController.getCurrentDocument()
  ↓
HoneyGuiCCodeGenerator.generate()
  ↓
遍历组件树：
  - 生成main.h（组件声明）
  - 生成main.c（初始化代码）
  - 生成main_callbacks.h（回调声明）
  - 生成main_callbacks.c（回调实现，含保护区）
  ↓
写入src/autogen/main/
  ↓
发送'codegenComplete'消息
  ↓
显示成功提示
```

### 5. 编译仿真流程

```
用户执行'honeygui.simulation'命令
  ↓
SimulationService.startSimulation()
  ↓
EnvironmentChecker.checkEnvironment()
  - 检查SDK路径（project.json → settings → 默认）
  - 检查SCons
  - 检查GCC
  ↓
CodeGenManager.generateCode()
  ↓
BuildCore.setupBuildDir()
  - 创建.honeygui-build/
  - 复制SDK的win32_sim/
  ↓
BuildCore.copyGeneratedCode()
  - 复制src/autogen/到编译目录
  - 生成autogen/SConscript
  ↓
SConstructGenerator.generate()
  - 生成SConstruct编译脚本
  ↓
BuildCore.compile()
  - 执行scons命令
  - 显示编译输出
  ↓
SimulationRunner.start()
  - 启动可执行文件
  - 显示SDL2窗口
  ↓
用户查看仿真效果
  ↓
用户停止仿真
  ↓
SimulationRunner.stop()
```

### 6. 协同开发流程

#### 主机模式
```
用户执行'honeygui.collaboration.startHost'
  ↓
CollaborationService.startHost(port)
  ↓
创建WebSocket服务器
  ↓
监听端口（默认3000）
  ↓
获取本机IP地址
  ↓
显示连接地址（如192.168.1.10:3000）
  ↓
复制地址到剪贴板
  ↓
等待访客连接
  ↓
访客连接成功
  ↓
发送完整组件数据（sync:full）
  ↓
监听Webview操作
  ↓
广播操作到所有访客
  ↓
保存文件到磁盘
```

#### 访客模式
```
用户执行'honeygui.collaboration.joinSession'
  ↓
输入主机地址
  ↓
CollaborationService.joinSession(address)
  ↓
创建WebSocket客户端
  ↓
连接到主机
  ↓
接收完整组件数据
  ↓
HmlController.applyRemoteUpdate()
  ↓
打开设计器显示内容
  ↓
用户进行编辑
  ↓
Webview操作被拦截
  ↓
发送到主机
  ↓
主机广播给所有客户端
  ↓
接收其他人的修改
  ↓
实时更新Webview
```

### 7. 资源管理流程

#### 导入资源
```
用户拖拽图片到AssetsPanel
  ↓
processImageFiles()
  ↓
发送'importAssets'消息
  ↓
AssetManager.importAssets()
  ↓
复制文件到assets/
  ↓
扫描资源目录
  ↓
发送资源列表到Webview
  ↓
AssetsPanel显示缩略图
```

#### 拖拽创建组件
```
用户从AssetsPanel拖拽图片到画布
  ↓
DesignerCanvas.onDrop()
  ↓
createImageComponentAtPosition()
  ↓
创建hg_image组件
  ↓
设置src属性为资源路径
  ↓
store.addComponent()
  ↓
自动保存
```

#### 删除资源
```
用户点击删除按钮
  ↓
确认对话框
  ↓
发送'deleteAsset'消息
  ↓
AssetManager.deleteAsset()
  ↓
查找引用该资源的组件
  ↓
删除关联组件
  ↓
删除资源文件
  ↓
更新资源列表
```

## 关键设计模式

### 1. 命令模式
- 撤销/重做系统
- 每个操作封装为Command对象
- 维护历史栈

### 2. 观察者模式
- Zustand状态订阅
- 文件监听（chokidar）
- WebSocket消息监听

### 3. 工厂模式
- DesignerPanelFactory
- 组件创建工厂

### 4. 单例模式
- StatusBarManager
- Logger

### 5. 策略模式
- 不同的代码生成器（C/C++）
- 不同的运行模式（预览/仿真）

### 6. 适配器模式
- VSCodeLogger适配Logger接口
- WebviewContentProvider适配不同环境

## 性能优化策略

### 1. 防抖和节流
- 自动保存使用防抖（500ms）

## 部署流程

### 1. 开发构建
```bash
npm run watch          # 监听扩展代码
npm run watch:webview  # 监听Webview代码
# 按F5启动调试
```

### 2. 生产构建
```bash
npm run vscode:prepublish  # 完整构建
vsce package               # 打包VSIX
```

### 3. 发布
```bash
vsce publish              # 发布到市场
# 或手动分发VSIX文件
```
