# 视频控件实现总结

## 实现概述

本次更新为 HoneyGUI Design 添加了完整的视频控件支持，参考图片控件的处理流程，实现了从设计器到编译运行的完整工作流。

## 新增文件

### 1. VideoConverterService.ts
**路径**: `src/services/VideoConverterService.ts`

**功能**: 视频格式转换服务
- 支持多种输入格式（MP4, AVI, MOV, MKV, WebM, FLV, WMV）
- 支持三种输出格式（MJPEG, AVI, H.264）
- 支持质量和帧率配置
- 批量转换功能
- 与 SDK 的 `video_converter` 工具集成

**关键方法**:
```typescript
// 单个视频转换
async convert(inputPath, outputPath, options): Promise<VideoConvertResult>

// 批量转换
async convertBatch(items): Promise<VideoConvertResult[]>

// 扫描目录转换
async convertAssetsDir(assetsDir, outputDir, defaultOptions): Promise<VideoConvertResult[]>

// 根据组件配置转换
async convertWithComponentConfig(inputPath, outputDir, componentData): Promise<VideoConvertResult>
```

### 2. HgVideoProperties.tsx
**路径**: `src/webview/components/properties/HgVideoProperties.tsx`

**功能**: 视频组件专用属性面板
- 基础属性（ID, 名称, 路径）
- 位置和尺寸设置
- 转换设置（格式, 质量, 帧率）
- 播放设置（播放帧率, 自动播放）
- 格式说明和使用提示

**特性**:
- 根据选择的格式动态显示相关选项
- MJPEG 格式显示质量设置
- AVI 格式显示帧率设置
- 友好的用户提示和说明

### 3. 视频控件使用说明.md
**路径**: `docs/视频控件使用说明.md`

**内容**: 完整的用户文档
- 功能特性介绍
- 使用流程说明
- 视频转换详解
- 代码生成策略
- 最佳实践
- 故障排查
- API 参考

## 修改的文件

### 1. BuildCore.ts
**修改内容**:
- 导入 `VideoConverterService`
- 在 `convertAssets()` 方法中添加视频转换逻辑
- 从项目配置读取默认视频转换选项
- 视频转换失败时提供详细错误信息

**代码片段**:
```typescript
// 转换视频资源
const videoConverter = new VideoConverterService(this.sdkPath);
const defaultVideoOptions = {
    format: this.projectConfig.videoFormat || 'mjpeg',
    quality: this.projectConfig.videoQuality || 85,
    frameRate: this.projectConfig.videoFrameRate || 30
};
const videoResults = await videoConverter.convertAssetsDir(
    assetsDir, outputDir, defaultVideoOptions
);
```

export interface ProjectConfig {

## 完整工作流

### 1. 设计器阶段
```
用户拖拽视频文件到画布
    ↓
创建 hg_video 组件
    ↓
在属性面板配置：
  - 输出格式 (mjpeg/avi/h264)
  - 质量 (0-100)
  - 帧率 (FPS)
  - 自动播放
    ↓
保存到 HML 文件
```

### 2. 代码生成阶段
```
读取 HML 文件
    ↓
解析 hg_video 组件
    ↓
路径转换：
  assets/video.mp4 → /video.mjpeg
    ↓
生成 C 代码：
  - gui_video_create_from_fs()
  - gui_video_set_frame_rate()
  - gui_video_set_state()
```

### 3. 编译阶段
```
扫描 assets/ 目录
    ↓
调用 VideoConverterService
    ↓
执行 SDK 的 video_converter 工具：
  python -m video_converter -i video.mp4 -o video.mjpeg -f mjpeg -q 85
    ↓
输出到 build/assets/video.mjpeg
    ↓
打包到 romfs
    ↓
编译链接
```

### 4. 运行阶段
```
挂载 romfs 文件系统
    ↓
从 VFS 加载 /video.mjpeg
    ↓
创建视频组件并播放
```

## 技术要点

### 1. 格式转换策略
- **MJPEG**: 高质量，支持质量参数 `-q`
- **AVI**: 兼容性好，支持帧率参数 `-r`
- **H.264**: 高压缩率，文件小

### 2. 路径处理
```typescript
// 原始路径
src: "assets/video.mp4"

// 根据格式替换扩展名
format: "mjpeg" → "assets/video.mjpeg"

// 去掉 assets/ 前缀
→ "video.mjpeg"

// 添加 VFS 根路径
→ "/video.mjpeg"
```

### 3. 组件数据结构
```typescript
{
    type: 'hg_video',
    data: {
        src: 'assets/video.mp4',      // 原始路径
        format: 'mjpeg',               // 输出格式
        quality: 85,                   // 质量 (MJPEG)
        frameRate: 30,                 // 帧率
        autoPlay: true                 // 自动播放
    }
}
```

### 4. 错误处理
- 视频转换失败时提供详细错误信息
- 区分图片转换和视频转换的错误
- Python 环境检查和友好提示

## 与图片控件的对比

| 特性 | 图片控件 | 视频控件 |
|------|----------|----------|
| 输入格式 | PNG, JPG, BMP, GIF 等 | MP4, AVI, MOV, MKV 等 |
| 输出格式 | .bin (单一格式) | MJPEG, AVI, H.264 (可选) |
| 转换工具 | image_converter.py | video_converter |
| 配置选项 | 无 | 格式, 质量, 帧率 |
| API 函数 | gui_img_create_from_fs | gui_video_create_from_fs |
| 额外设置 | 无 | 帧率, 播放状态 |

## 已知限制

1. **转换工具依赖**: 需要 SDK 中的 `video_converter` 工具
2. **Python 环境**: 需要 Python 3.x 环境
3. **格式支持**: 取决于 SDK 工具支持的格式
4. **文件大小**: 视频文件可能较大，影响 romfs 大小
