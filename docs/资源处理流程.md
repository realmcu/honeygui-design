# 资源处理流程说明

## 完整编译流程

```
用户触发编译
    ↓
BuildManager.compile()
    ↓
BuildCore 执行以下步骤：
    1. setupBuildDir()      - 准备编译目录
    2. copyGeneratedCode()  - 检查生成的代码
    3. convertAssets()      - 转换资源文件
    4. compile()            - SCons 编译
```

## 1. 准备编译目录 (setupBuildDir)

### 目的
准备 `build/` 目录，包含 SDK 的 win32_sim 环境

### 流程
```
1. 检查 SDK 路径
   └─ ~/.HoneyGUI-SDK/win32_sim/

2. 拷贝 win32_sim 到 build/
   ├─ 如果 build/ 不存在 → 完整拷贝
   └─ 如果 build/ 已存在 → 跳过拷贝

3. 拷贝 Kconfig.gui
   └─ SDK/Kconfig.gui → build/Kconfig.gui

4. 生成配置文件
   └─ build/.config

5. 修改 SConstruct
   └─ 调整编译配置
```

### 输出
```
build/
├── gui/              (SDK 源码)
├── lvgl/             (LVGL 源码)
├── SConstruct        (编译脚本)
├── Kconfig.gui       (配置文件)
└── .config           (生成的配置)
```

## 2. 检查生成的代码 (copyGeneratedCode)

### 目的
验证用户已生成 C 代码

### 流程
```
1. 检查 src/ 目录是否存在
2. 检查 src/SConscript 是否存在
3. 如果不存在 → 抛出错误，提示先生成代码
```

### 要求
用户必须先在设计器中生成代码：
```
src/
├── ui/
│   ├── main_ui.h
│   └── main_ui.c
├── callbacks/
│   └── main_callbacks.c
└── SConscript
```

## 3. 转换资源文件 (convertAssets) ⭐

### 目的
将 `assets/` 中的原始资源转换为 HoneyGUI 格式

### 3.1 图片转换

**输入**：`assets/*.png`, `assets/*.jpg`

**处理**：
```typescript
ImageConverterService.convertAssetsDir(assetsDir, outputDir)
  ↓
扫描 assets/ 目录
  ↓
对每个图片文件：
  1. 检测实际格式（基于文件魔数，不依赖扩展名）
  2. 解析图片（PNG/JPEG）
  3. 转换为 HoneyGUI 格式（RGB565/RGB888/ARGB8888 等）
  4. 写入 build/assets/*.bin
```

**输出**：`build/assets/*.bin`

**示例**：
```
assets/icon.png (10 KB)
  ↓ [ImageConverter]
build/assets/icon.bin (8 KB, RGB565 格式)
```

**特性**：
- ✅ 自动格式检测（不依赖扩展名）
- ✅ 支持 5 种像素格式
- ✅ 可选压缩（RLE/FastLZ/YUV）
- ✅ 纯 TS/JS 实现，无需 Python

### 3.2 视频转换

**输入**：`assets/*.mp4`, `assets/*.avi`

**处理**：
```typescript
VideoConverterService.convertAssetsDir(assetsDir, outputDir)
  ↓
检查 FFmpeg 是否可用
  ↓
对每个视频文件：
  1. 使用 FFmpeg 转换格式
  2. 调整帧率、质量
  3. 写入 build/assets/*.avi
```

**输出**：`build/assets/*.avi`

**依赖**：需要系统安装 FFmpeg

### 3.3 3D 模型转换

**输入**：`assets/*.obj`, `assets/*.gltf`

**处理**：
```typescript
Model3DConverterService.convertAssetsDir(assetsDir, outputDir)
  ↓
扫描 assets/ 目录
  ↓
对每个模型文件：
  
  【OBJ 文件】
  1. 解析 .obj 文件（顶点、法线、纹理坐标）
  2. 查找并解析 .mtl 文件（材质）
  3. 读取 MTL 中的纹理引用（map_Kd）
  4. 查找对应的 .bin 纹理文件
  5. 嵌入纹理数据
  6. 写入 build/assets/desc_*.bin
  
  【GLTF 文件】
  1. 检查 .bin buffer 文件是否存在
  2. 解析 .gltf JSON 描述
  3. 读取 .bin 几何数据
  4. 解析材质和纹理
  5. 写入 build/assets/desc_*.bin
```

**输出**：`build/assets/desc_*.bin`

**示例**：
```
OBJ 模型：
assets/
├── earth_tria.obj      (76 KB)
├── earth_tria.mtl      (258 B, 引用 earth.png)
└── earth.png           (51 KB, 实际是 JPEG)
  ↓
1. 转换纹理：earth.png → earth.bin (262 KB)
2. 转换模型：earth_tria.obj → desc_earth_tria.bin (321 KB)
   └─ 包含：几何数据 (59 KB) + 纹理数据 (262 KB)

GLTF 模型：
assets/
├── Pikachu_resize.gltf (203 KB, JSON 描述)
└── Pikachu_resize.bin  (115 KB, 几何数据) ← 必需！
  ↓
转换：desc_Pikachu_resize.bin (97 KB)
```

**特性**：
- ✅ OBJ 完整支持（含 MTL 材质）
- ✅ GLTF 2.0 支持（需要 .bin 文件）
- ✅ 自动纹理加载和嵌入
- ✅ 面类型自动检测
- ✅ 纯 TS/JS 实现，无需 Python

**注意事项**：
1. **纹理必须先转换**：OBJ 引用的纹理图片必须先转换为 .bin
2. **GLTF 需要完整文件**：.gltf 和 .bin 必须同时存在
3. **文件命名**：纹理 .bin 文件名必须与原图片名一致

### 3.4 拷贝 UI 目录

**输入**：`ui/` 目录（HML 设计文件）

**处理**：
```
直接拷贝 ui/ 目录到 build/assets/ui/
```

**输出**：`build/assets/ui/*.hml`

### 3.5 打包 romfs

**目的**：将所有转换后的资源打包为文件系统

**处理**：
```
1. 调用 SDK mkromfs 工具
   └─ tool/mkromfs/mkromfs_for_honeygui.py

2. 扫描 build/assets/ 目录

3. 生成两个文件：
   ├─ root_image.c   (C 数组格式)
   └─ app_romfs.bin  (二进制格式)
```

**输出**：
```
build/
├── root_image.c      (romfs C 文件)
└── app_romfs.bin     (romfs 二进制)
```

## 4. 编译 (compile)

### 目的
使用 SCons 编译整个项目

### 流程
```
1. 在 build/ 目录执行：scons -j4

2. SCons 编译：
   ├─ SDK 源码 (gui/, lvgl/)
   ├─ 用户代码 (src/)
   └─ romfs 数据 (root_image.c)

3. 链接生成可执行文件
```

### 输出
```
build/gui.exe  (Windows)
build/gui      (Linux)
```

## 常见问题

### Q1: 图片转换失败 "Invalid file signature"
**原因**：文件扩展名与实际格式不符（如 .png 实际是 JPEG）
**解决**：新版本已自动处理，基于文件魔数检测

### Q2: GLTF 转换失败 "missing .bin"
**原因**：GLTF 文件不完整，缺少几何数据文件
**解决**：
1. 从原始来源获取完整文件（.gltf + .bin）
2. 或使用嵌入式 GLTF（所有数据在 JSON 中）

### Q3: OBJ 模型没有纹理
**原因**：纹理图片未转换为 .bin 格式
**解决**：
1. 先转换纹理：`earth.png → earth.bin`
2. 再转换模型：`earth_tria.obj → desc_earth_tria.bin`

### Q4: 拖拽 GLTF 文件后缺少 .bin
**原因**：系统只复制了选中的文件，未复制关联文件
**解决**：手动复制 .bin 文件到 assets/ 目录
