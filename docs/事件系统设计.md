# 事件系统设计

## 概述

HoneyGUI Design 的事件系统采用 **事件-动作(Event-Action)** 模式,用户可以在设计器中为组件配置事件响应,无需编写代码即可实现常见交互逻辑。

## 核心概念

### 1. 事件(Event)
组件触发的交互行为。

### 2. 动作(Action)
事件触发后执行的操作,一个事件可以配置多个动作,按顺序执行。

### 3. 消息(Message)
全局广播机制,组件可以发送消息,其他组件可以监听并响应。

## UI 设计变更

### 属性面板标签页统一
原有的"视图切换"功能将统一到 Event-Action 体系中:

```
┌─────────────────────────────────────┐
│ [属性] [事件]                       │  ← 所有组件统一使用
├─────────────────────────────────────┤
```

- **属性**: 组件基础属性(位置、大小、样式等)
- **事件**: Event-Action 配置(包含原"视图切换"功能)

### 兼容性说明
- 原 `view_switch` 字段映射为 `onSwipeXxx` + `switchView` 动作
- HML 解析时自动转换旧格式

## 事件类型(Event Types)

### 通用事件(所有组件支持)
| 事件名 | 标识符 | 说明 |
|--------|--------|------|
| 单击 | `onClick` | 点击后抬起触发 |
| 长按 | `onLongPress` | 按住超过阈值时间触发 |
| 按下 | `onTouchDown` | 手指按下时触发 |
| 抬起 | `onTouchUp` | 手指抬起时触发 |

### 滑动事件(容器组件支持)
| 事件名 | 标识符 | 说明 | 支持组件 |
|--------|--------|------|----------|
| 左滑 | `onSwipeLeft` | 向左滑动触发 | hg_view, hg_window |
| 右滑 | `onSwipeRight` | 向右滑动触发 | hg_view, hg_window |
| 上滑 | `onSwipeUp` | 向上滑动触发 | hg_view, hg_window |
| 下滑 | `onSwipeDown` | 向下滑动触发 | hg_view, hg_window |

### 状态事件(特定组件支持)
| 事件名 | 标识符 | 说明 | 支持组件 |
|--------|--------|------|----------|
| 值改变 | `onValueChange` | 值变化时触发 | hg_slider, hg_progressbar |
| 选中改变 | `onCheckedChange` | 选中状态变化 | hg_checkbox, hg_switch, hg_radio |

### 生命周期事件(容器组件支持)
| 事件名 | 标识符 | 说明 | 支持组件 |
|--------|--------|------|----------|
| 显示 | `onShow` | 组件显示时触发 | hg_view, hg_window |
| 隐藏 | `onHide` | 组件隐藏时触发 | hg_view, hg_window |

### 消息事件(所有组件支持)
| 事件名 | 标识符 | 说明 |
|--------|--------|------|
| 消息监听 | `onMessage` | 收到指定消息时触发 |

## 动作类型(Action Types)

### 视图切换动作
| 动作名 | 标识符 | 参数 | 说明 |
|--------|--------|------|------|
| 跳转界面 | `switchView` | `target`, `switchOutStyle`, `switchInStyle` | 带动画的视图切换 |

**switchOutStyle 选项:**
- 平移: `SWITCH_OUT_TO_LEFT/RIGHT/TOP/BOTTOM_USE_TRANSLATION`
- 立方体: `SWITCH_OUT_TO_LEFT/RIGHT/TOP/BOTTOM_USE_CUBE`
- 旋转: `SWITCH_OUT_TO_LEFT/RIGHT/TOP/BOTTOM_USE_ROTATE`
- 缩放: `SWITCH_OUT_TO_LEFT/RIGHT/TOP/BOTTOM_USE_REDUCTION`
- 其他: `SWITCH_OUT_NONE_ANIMATION`, `SWITCH_OUT_ANIMATION_FADE`, `SWITCH_OUT_ANIMATION_ZOOM`

**switchInStyle 选项:**
- 平移: `SWITCH_IN_FROM_LEFT/RIGHT/TOP/BOTTOM_USE_TRANSLATION`
- 立方体: `SWITCH_IN_FROM_LEFT/RIGHT/TOP/BOTTOM_USE_CUBE`
- 旋转: `SWITCH_IN_FROM_LEFT/RIGHT/TOP/BOTTOM_USE_ROTATE`
- 缩放: `SWITCH_IN_FROM_LEFT/RIGHT/TOP/BOTTOM_USE_REDUCTION`
- 其他: `SWITCH_IN_NONE_ANIMATION`, `SWITCH_IN_ANIMATION_FADE`, `SWITCH_IN_CENTER_ZOOM_FADE`

### 消息动作
| 动作名 | 标识符 | 参数 | 说明 |
|--------|--------|------|------|
| 发送消息 | `sendMessage` | `message` | 广播消息 |

### 自定义动作
| 动作名 | 标识符 | 参数 | 说明 |
|--------|--------|------|------|
| 调用函数 | `callFunction` | `functionName` | 调用用户自定义C函数 |

## 消息系统(Message System)

### 概念
消息系统提供组件间的解耦通信机制:
- **发送方**: 通过 `sendMessage` 动作广播消息
- **接收方**: 通过 `onMessage` 事件监听消息

### 使用场景
1. **跨界面通信**: 界面A的操作影响界面B的显示
2. **全局状态同步**: 多个组件响应同一状态变化
3. **模块解耦**: 避免组件间直接引用

### 示例
```
场景: 登录按钮点击后,多个界面需要更新

按钮配置:
  事件: onClick
  动作: sendMessage
    消息名: "login_success"

界面A配置:
  事件: onMessage("login_success")
  动作: show(welcomePanel)

界面B配置:
  事件: onMessage("login_success")  
  动作: setProperty(userLabel, text, "已登录")
```

## 数据结构

### HML 中的事件配置
```xml
<!-- 按钮点击跳转 -->
<hg_button id="btn1" x="10" y="10" w="100" h="40" text="Click">
  <events>
    <event type="onClick">
      <action type="navigateTo" target="screen2" />
    </event>
  </events>
</hg_button>

<!-- 视图滑动切换(原view_switch功能) -->
<hg_view id="main_view" x="0" y="0" w="480" h="272">
  <events>
    <event type="onSwipeLeft">
      <action type="switchView" target="screen2" 
              switchOutStyle="SWITCH_OUT_TO_LEFT_USE_TRANSLATION"
              switchInStyle="SWITCH_IN_FROM_RIGHT_USE_TRANSLATION" />
    </event>
  </events>
</hg_view>

<!-- 消息监听 -->
<hg_text id="label1">
  <events>
    <event type="onMessage" message="button_clicked">
      <action type="setProperty" target="label1" property="text" value="按钮被点击了" />
    </event>
  </events>
</hg_text>
```

### TypeScript 类型定义
```typescript
// 事件类型
type EventType = 
  // 通用事件
  | 'onClick' | 'onLongPress' | 'onTouchDown' | 'onTouchUp'
  // 滑动事件
  | 'onSwipeLeft' | 'onSwipeRight' | 'onSwipeUp' | 'onSwipeDown'
  // 状态事件
  | 'onValueChange' | 'onCheckedChange'
  // 生命周期事件
  | 'onShow' | 'onHide'
  // 消息事件
  | 'onMessage';

// 动作类型
type ActionType = 
  // 视图切换(带动画)
  | 'switchView'
  // 导航
  | 'navigateTo' | 'goBack'
  // 组件控制
  | 'show' | 'hide' | 'toggle' | 'setProperty'
  // 消息
  | 'sendMessage'
  // 自定义
  | 'callFunction';

// 动作定义
interface Action {
  type: ActionType;
  target?: string;           // 目标组件/界面ID
  property?: string;         // 属性名(setProperty用)
  value?: any;               // 属性值(setProperty用)
  message?: string;          // 消息名(sendMessage用)
  functionName?: string;     // 函数名(callFunction用)
  // switchView专用
  switchOutStyle?: string;   // 退出动画
  switchInStyle?: string;    // 进入动画
}

// 事件定义
interface EventConfig {
  type: EventType;
  message?: string;          // 仅 onMessage 事件需要
  actions: Action[];
}

// 组件事件配置(添加到 ComponentData)
interface ComponentData {
  // ... 现有字段
  events?: EventConfig[];
}
```
