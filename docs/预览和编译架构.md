# HoneyGUI Design 预览和编译仿真架构

## 概述

本文档描述 HoneyGUI Design 的预览和编译仿真架构，实现从可视化设计到真实运行效果验证的完整流程。

## 架构演进

### 历史架构（v1.1.x 之前）
```
src/preview/
├── PreviewService.ts    # 混合了预览和编译逻辑
└── PreviewRunner.ts     # 单一运行器
```

**问题**：
- 职责不清：PreviewService 混合了两种不同的运行模式
- 难以维护：预览和编译逻辑耦合在一起
- 用户困惑：不清楚"预览"具体做什么
- 扩展性差：难以独立优化两种模式

### 当前架构（v1.2.1 - 2025-11-29）
```
src/
├── preview/                    # 预览模块（独立）
│   ├── PreviewService.ts      # 纯预览服务
│   └── PreviewRunner.ts       # 预览运行器
│
├── simulation/                 # 编译仿真模块
│   ├── SimulationService.ts   # 仿真服务
│   ├── SimulationRunner.ts    # 仿真运行器
│   ├── EnvironmentChecker.ts  # 环境检查
│   ├── BuildCore.ts           # 编译核心逻辑（不依赖VSCode）
│   ├── BuildManager.ts        # 编译管理（VSCode版本，继承BuildCore）
│   └── SConstructGenerator.ts # 编译脚本生成
│
└── common/
    ├── RunnerStatus.ts        # 共享状态定义
    └── ProjectConfig.ts       # 项目配置
```

**改进**：
- ✅ 职责清晰：预览和仿真完全分离
- ✅ 易于维护：各自独立开发和测试
- ✅ 用户友好：两种模式定义明确
- ✅ 易于扩展：可独立优化和增强

## 架构决策记录 (2025-11-27)

### 两种运行模式

| 模式 | 名称 | 英文 | 流程 | 用途 |
|------|------|------|------|------|
| 模式1 | **预览** | Preview | HML → 解释器 → 直接渲染 | 快速验证设计 |
| 模式2 | **编译仿真** | Compile & Simulate | HML → C代码 → 编译 → 运行 | 真实效果验证 |

### 核心决策

#### 1. 编译仿真流程
```
用户点击"编译仿真"
  ↓
读取当前HML文件
  ↓
调用C代码生成器 (已有的HoneyGuiCCodeGenerator)
  ↓
生成C代码到 {projectRoot}/src/autogen/{designName}/
  ↓
设置编译目录 {projectRoot}/.honeygui-build/
  ↓
拷贝SDK的win32_sim到编译目录
  ↓
生成SConstruct编译脚本
  ↓
执行SCons编译
  ↓
运行生成的可执行文件
  ↓
显示SDL2仿真窗口
```

#### 2. HoneyGUI SDK路径获取（优先级）
1. 项目配置：`project.json` 中的 `honeyguiSdkPath`
2. VSCode设置：`honeygui.sdk.path`
3. 默认路径：`~/.HoneyGUI-SDK`

#### 3. 编译工具链检查（方案A - 严格检查）
启动编译仿真前检查：
- SCons 命令是否存在
- GCC/MinGW 编译器是否存在
- 如果缺失，显示安装指南并中止

#### 4. UI入口（选项A - 两个按钮）
```
设计器工具栏：
[👁️ 预览] [🚀 编译仿真]
```

#### 5. 编译输出位置（选项C - 项目目录 + gitignore）
```
{projectRoot}/.honeygui-build/
├── src/
│   └── autogen/          # 从项目src/autogen拷贝
├── win32_sim/            # 从SDK拷贝
│   ├── main.c
│   └── port/
├── SConstruct            # 自动生成
└── build/
    └── gui               # 编译输出的可执行文件
```
- 自动添加到 `.gitignore`
- 支持增量编译
- 不污染版本控制

## 技术路线

### 预览模式（未实现）
```
设计器 → HML → 解释器 → 直接渲染到SDL2窗口
```

### 编译仿真模式（优先实现）
```
设计器 → HML → C代码生成 → SCons编译 → 模拟器运行 → SDL2窗口
```

**核心原理**：
- 设计器生成 HML 文件描述界面
- C 代码生成器将 HML 转换为调用 HoneyGUI API 的 C 代码
- 使用 SCons 编译用户代码和 HoneyGUI SDK
- 在 PC 上运行编译后的模拟器，显示真实运行效果

## 使用说明

### 编译仿真

1. **配置SDK路径**（三选一）：
   - 项目配置：在 `project.json` 中添加 `honeyguiSdkPath`
   - VSCode设置：配置 `honeygui.sdk.path`
   - 默认路径：`~/.HoneyGUI-SDK`

2. **安装编译工具**：
   - SCons: `pip install scons`
   - GCC (Linux): `sudo apt-get install build-essential`
   - MinGW (Windows): 下载安装 MinGW-w64

3. **启动编译仿真**：
   - 打开 HML 文件
   - 按 `Ctrl+Shift+P`
   - 选择 `HoneyGUI: Compile & Simulate`
   - 或点击状态栏的 `$(rocket) 编译仿真` 按钮

4. **查看输出**：
   - 编译输出显示在 "HoneyGUI Simulation" 输出通道
   - 编译产物在 `.honeygui-build/build/gui`

### 预览

- 当前显示"功能开发中"提示
- 建议使用编译仿真功能

### 实现代码

```typescript
class ProjectCreator {
  async createProject(config: ProjectConfig) {
    const projectPath = path.join(config.saveLocation, config.projectName);
    const sdkPath = config.honeyguiSdkPath;
    
    // 1. 创建目录结构
    this.createDirectories(projectPath);
    
    // 2. 拷贝模拟器入口
    await this.copySimulatorEntry(sdkPath, projectPath);
    
    // 3. 自定义 main.c
    await this.customizeMainC(projectPath, config);
    
    // 4. 生成 SConstruct
    this.generateSConstruct(projectPath, config);
    
    // 5. 生成初始 HML
    this.generateInitialHML(projectPath, config);
    
    // 6. 生成编译脚本
    this.generateBuildScripts(projectPath);
    
    // 7. 保存项目配置
    this.saveProjectConfig(projectPath, config);
  }
  
  private async copySimulatorEntry(sdkPath: string, projectPath: string) {
    const simSrcPath = path.join(sdkPath, 'win32_sim');
    const simDestPath = path.join(projectPath, 'win32_sim');
    
    // 拷贝必要的文件和目录
    const itemsToCopy = [
      'main.c',              // 主入口
      'port/',               // 平台适配层
      'RTE/',                // SDL2 等依赖库
    ];
    
    for (const item of itemsToCopy) {
      const src = path.join(simSrcPath, item);
      const dest = path.join(simDestPath, item);
      
      if (fs.statSync(src).isDirectory()) {
        // 递归拷贝目录
        fs.cpSync(src, dest, { recursive: true });
      } else {
        // 拷贝文件
        fs.copyFileSync(src, dest);
      }
    }
  }
  
  private async customizeMainC(projectPath: string, config: ProjectConfig) {
    const mainCPath = path.join(projectPath, 'win32_sim/main.c');
    let content = fs.readFileSync(mainCPath, 'utf-8');
    
    // 替换窗口配置
    const [width, height] = config.resolution.split('X');
    content = content.replace(
      /#define WINDOW_WIDTH\s+\d+/,
      `#define WINDOW_WIDTH  ${width}`
    );
    content = content.replace(
      /#define WINDOW_HEIGHT\s+\d+/,
      `#define WINDOW_HEIGHT ${height}`
    );
    content = content.replace(
      /#define WINDOW_TITLE\s+"[^"]*"/,
      `#define WINDOW_TITLE  "${config.projectName} - HoneyGUI Preview"`
    );
    
    fs.writeFileSync(mainCPath, content);
  }
}
```

### 可定制的 main.c 模板

```c
// win32_sim/main.c
#include "gui_api.h"
#include "gui_server.h"
#include <SDL2/SDL.h>

// 项目配置（由生成器自动填充）
#define WINDOW_WIDTH  480
#define WINDOW_HEIGHT 272
#define WINDOW_TITLE  "NewProject - HoneyGUI Preview"

int main(int argc, char *argv[]) {
    // 1. 初始化 SDL2
    if (SDL_Init(SDL_INIT_VIDEO) < 0) {
        printf("SDL2 初始化失败: %s\n", SDL_GetError());
        return -1;
    }
    
    // 2. 创建窗口
    SDL_Window *window = SDL_CreateWindow(
        WINDOW_TITLE,
        SDL_WINDOWPOS_CENTERED,
        SDL_WINDOWPOS_CENTERED,
        WINDOW_WIDTH,
        WINDOW_HEIGHT,
        SDL_WINDOW_OPENGL | SDL_WINDOW_SHOWN
    );
    
    if (!window) {
        printf("窗口创建失败: %s\n", SDL_GetError());
        SDL_Quit();
        return -1;
    }
    
    // 3. 初始化 OpenGL 上下文
    SDL_GLContext gl_context = SDL_GL_CreateContext(window);
    if (!gl_context) {
        printf("OpenGL 上下文创建失败: %s\n", SDL_GetError());
        SDL_DestroyWindow(window);
        SDL_Quit();
        return -1;
    }
    
    // 4. 初始化 HoneyGUI
    gui_server_init();
    
    // 5. 用户界面会通过 GUI_VIEW_INSTANCE 自动注册
    
    // 6. 主循环
    bool running = true;
    SDL_Event event;
    
    while (running) {
        // 处理事件
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) {
                running = false;
            }
            // 传递事件给 HoneyGUI
            gui_server_handle_event(&event);
        }
        
        // 更新和渲染
        gui_server_update();
        gui_server_render();
        
        SDL_GL_SwapWindow(window);
        SDL_Delay(16);  // ~60 FPS
    }
    
    // 7. 清理
    SDL_GL_DeleteContext(gl_context);
    SDL_DestroyWindow(window);
    SDL_Quit();
    
    return 0;
}
```

- 事件处理（添加自定义事件）

## 编译入口：SConstruct

import sys

# HoneyGUI SDK 路径（只用于链接 realgui 库）
HONEYGUI_SDK = '/home/howie_wang/workspace/HoneyGUI'

# 项目路径

# 环境配置

# 设置编译器

env['CXX'] = 'g++'

# 编译标志

env.Append(CPPPATH=[
    os.path.join(HONEYGUI_SDK, 'realgui'),
    os.path.join(HONEYGUI_SDK, 'realgui/server'),
    os.path.join(HONEYGUI_SDK, 'realgui/widget'),
    os.path.join(HONEYGUI_SDK, 'realgui/engine'),
    os.path.join(HONEYGUI_SDK, 'realgui/core'),

# 链接库路径（使用项目内的 SDL2）

env.Append(LIBPATH=[SDL2_LIB_PATH])
env.Append(LIBS=['SDL2main', 'SDL2', 'opengl32', 'mingw32'])

# 收集源文件
# 1. HoneyGUI SDK 源文件（只链接 realgui 库）
honeygui_src = Glob(os.path.join(HONEYGUI_SDK, 'realgui/**/*.c'))

## 仿真器定义

### 仿真器是什么

**仿真器 = 编译后的可执行文件**

- Windows: `.honeygui-build/build/gui.exe`
- Linux: `.honeygui-build/build/gui`

这是由 HoneyGUI SDK 的 win32_sim 和用户生成的C代码编译出来的可执行程序。

### 仿真器的职责

#### 1. 图形渲染
- 创建 SDL2 窗口
- 初始化 OpenGL 上下文
- 渲染 GUI 界面到窗口

#### 2. 执行用户代码
- 加载并执行生成的 C 代码
- 调用 `GUI_VIEW_INSTANCE` 注册的视图
- 执行 `switch_in` 回调创建组件

#### 3. 事件处理
- 捕获鼠标点击、移动
- 捕获键盘输入
- 传递给 HoneyGUI 事件系统

#### 4. 资源加载
- 从 `assets/` 目录加载图片、字体
- 解码图片格式（PNG、JPG等）
- 管理资源缓存

#### 5. 运行循环
```c
// 仿真器的主循环（简化版）
int main() {
    // 1. 初始化 SDL2 和 OpenGL
    SDL_Init(SDL_INIT_VIDEO);
    SDL_Window *window = SDL_CreateWindow(...);
    
    // 2. 初始化 HoneyGUI
    gui_server_init();
    
    // 3. 加载用户界面（调用生成的代码）
    // GUI_VIEW_INSTANCE 会自动注册
    
    // 4. 主循环
    while (running) {
        // 处理事件
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            handle_input(&event);
        }
        
        // 更新 GUI
        gui_server_update();
        
        // 渲染
        gui_server_render();
        SDL_GL_SwapWindow(window);
    }
    
    // 5. 清理
    SDL_Quit();
    return 0;
}
```

### src/autogen/SConscript 说明

`src/autogen/SConscript` 是在**代码生成阶段**自动创建的 SCons 构建脚本。

**生成时机**：`HoneyGuiCCodeGenerator.generate()` 生成 C 代码时自动创建

**生成器**：`SConscriptGenerator.generate()`（独立工具类）

**内容**：
```python
from building import *
import os

cwd = GetCurrentDir()
src = []
for root, dirs, files in os.walk(cwd):
    for f in files:
        if f.endswith('.c'):
            src.append(os.path.join(root, f))

CPPPATH = [cwd]
for root, dirs, files in os.walk(cwd):
    CPPPATH.append(root)

group = DefineGroup('autogen', src, depend=[''], CPPPATH=CPPPATH)
Return('group')
```

**工作原理**：
1. 代码生成时，`SConscriptGenerator` 在 `src/autogen/` 创建 `SConscript`
2. 编译时，修改后的 `build/SConstruct` 引用 `{projectRoot}/src/autogen/SConscript`
3. SCons 执行 `src/autogen/SConscript`，收集所有 `.c` 文件
4. 编译产物（.o 文件、可执行文件）输出到 `build/`
5. 源代码保持在 `src/autogen/`，只有一份

// BuildCore.modifySConstruct()
private modifySConstruct(): void {
    const sconstructPath = path.join(this.buildDir, 'SConstruct');
    let content = fs.readFileSync(sconstructPath, 'utf-8');
    
    const sdkPathNormalized = this.sdkPath.replace(/\\/g, '/');
    const projectRootNormalized = this.projectRoot.replace(/\\/g, '/');
    
    // 1. 将 PROJECT_ROOT 替换为 SDK 的绝对路径
    content = content.replace(
        /PROJECT_ROOT\s*=\s*os\.path\.dirname\(os\.getcwd\(\)\)/,
        `PROJECT_ROOT = '${sdkPathNormalized}'`
    );

    // 2. 在 DoBuilding 之前添加项目 autogen 代码的编译
    const autogenInclude = `
# Include project autogen code
PROJECT_AUTOGEN = '${projectRootNormalized}/src/autogen'
if os.path.exists(os.path.join(PROJECT_AUTOGEN, 'SConscript')):
    objs.extend(SConscript(os.path.join(PROJECT_AUTOGEN, 'SConscript')))
`;
    content = content.replace(
        /# Build\s*\nDoBuilding\(TARGET, objs\)/,
        `${autogenInclude}\n# Build\nDoBuilding(TARGET, objs)`
    );
    
    fs.writeFileSync(sconstructPath, content, 'utf-8');

- 跳过 win32_sim 拷贝（节省 1-2 秒）

4. **并行编译**：`scons -j4` 使用多核 CPU
5. **单一代码源**：避免代码同步问题，减少磁盘占用
