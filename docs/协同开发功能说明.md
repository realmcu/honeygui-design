# HoneyGUI 离线局域网协同开发功能说明

HoneyGUI 设计器现已支持 **离线局域网多人协同开发** 模式。该功能允许团队成员在没有互联网连接的情况下，通过局域网实时共同编辑同一个 HML 设计稿。

## 1. 功能特点

*   **完全离线**: 不依赖任何公网服务器或云服务，数据仅在局域网内传输。
*   **实时同步**: 毫秒级的操作同步，支持组件的增删改查。
*   **主机-访客模式**: 
    *   **主机 (Host)**: 拥有设计稿的“真理之源”，负责保存文件。
    *   **访客 (Guest)**: 连接到主机，在内存中协同编辑，修改实时回传给主机保存。
*   **零侵入**: 不开启协同模式时，插件完全保持原有的单机运行逻辑，性能不受影响。

## 2. 使用方法

### 2.1 开启协同 (作为主机)

1.  打开一个 HML 设计稿。
2.  按下 `Ctrl+Shift+P` (macOS: `Cmd+Shift+P`) 打开命令面板。
3.  输入并选择 **`HoneyGUI: Start Host`**。
4.  输入监听端口（默认 `3000`）。
5.  启动成功后，右下角会显示连接地址（例如 `192.168.1.10:3000`），并且该地址会自动复制到剪贴板。
6.  将地址分享给局域网内的其他同事。

### 2.2 加入协同 (作为访客)

1.  打开 VS Code（无需打开任何项目）。
2.  按下 `Ctrl+Shift+P` 打开命令面板。
3.  输入并选择 **`HoneyGUI: Join Session`**。
4.  粘贴主机分享的地址（例如 `192.168.1.10:3000`）。
5.  连接成功后，设计器会自动打开并同步显示主机当前的设计稿内容。

### 2.3 停止协同

*   点击 VS Code 底部状态栏右侧的协同图标（`$(broadcast)` 或 `$(plug)`）。
*   或者在命令面板中运行 **`HoneyGUI: Stop Collaboration`**。

## 3. 技术架构

采用 **WebSocket 直连** 架构，由插件主进程充当信令服务器。

```mermaid
graph TD
    subgraph Host [主机 (Server)]
        WV1[Webview] <--> MH1[MessageHandler]
        MH1 <--> CS1[**CollaborationService**]
        CS1 <--> HC1[HmlController]
        HC1 <--> FS[文件系统]
    end

    subgraph Guest [访客 (Client)]
        WV2[Webview] <--> MH2[MessageHandler]
        MH2 <--> CS2[**CollaborationService**]
        CS2 <--> HC2[HmlController (内存副本)]
    end

    CS1 == WebSocket (LAN) ==> CS2
    CS2 == WebSocket (LAN) ==> CS1
```

*   **CollaborationService**: 核心服务类，管理 WebSocket 连接和状态分发。
*   **MessageHandler**: 拦截 Webview 的操作指令（如 `updateComponent`），在协同模式下将其广播而非直接执行。
*   **HmlController**: 新增 `applyRemoteUpdate` 接口，支持从网络流更新内存模型。

## 4. 注意事项

1.  **网络环境**: 确保主机和访客在同一个局域网网段内，且主机防火墙允许 VS Code 监听指定端口。
2.  **版本一致性**: 建议所有成员使用相同版本的 HoneyGUI 插件，以免协议不兼容。
3.  **单机模式**: 不开启协同时，插件完全保持原有单机运行逻辑，性能不受影响。

## 5. 已知限制

1.  **资源文件**: 目前仅支持 HML 结构同步，图片等资源文件需手动同步。
2.  **冲突处理**: 采用"主机为真理之源"的简化策略，多人同时修改同一组件时以主机为准。
3.  **断线重连**: 断线后需手动重新加入会话。
