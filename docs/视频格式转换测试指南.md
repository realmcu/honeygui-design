# 视频格式转换测试指南

## 📋 概述

本指南说明如何测试三种视频格式（MJPEG、AVI、H.264）的转换功能。

## 🎯 测试目标

验证视频转换服务能够正确地将输入视频转换为三种不同的输出格式，并确保：
- 转换成功完成
- 输出文件格式正确
- 文件大小合理
- 转换性能可接受

## 🛠️ 准备工作

### 1. 配置 SDK 路径（可选，用于后处理）

测试脚本会尝试从以下位置获取 SDK 路径：

**方法 1: 环境变量**
```bash
# Windows
set HONEYGUI_SDK_PATH=C:\path\to\HoneyGUI-SDK

# Linux/macOS
export HONEYGUI_SDK_PATH=/path/to/HoneyGUI-SDK
```

**方法 2: 默认路径**
- Linux/macOS: `~/.HoneyGUI-SDK`
- Windows: `C:\HoneyGUI-SDK`

**SDK 后处理脚本位置**：
- `SDK/tool/video-convert-tool/video_converter.py` (推荐)
- `SDK/tool/video_converter.py`
- `SDK/tools/video_converter.py`
- `SDK/video_converter.py`

如果没有 SDK 后处理脚本，测试仍会继续，但会跳过后处理步骤。

### 2. 安装 FFmpeg

**Windows**:
```bash
# 下载 FFmpeg
# https://ffmpeg.org/download.html
# 解压并添加到 PATH
```

**Linux**:
```bash
sudo apt install ffmpeg
```

**macOS**:
```bash
brew install ffmpeg
```

### 2. 验证 FFmpeg 安装

```bash
ffmpeg -version
```

### 3. 准备测试视频

在项目根目录创建 `assets_test` 目录，并放入测试视频：

```bash
mkdir assets_test
# 将测试视频复制到 assets_test 目录
```

支持的视频格式：
- `.mp4`
- `.avi`
- `.mov`
- `.mkv`

## 🚀 运行测试

### 方法 1: 快速检查（推荐）

```bash
npm run test:video-formats:quick
```

这会检查：
- ✓ assets_test 目录是否存在
- ✓ 是否有测试视频文件
- ✓ FFmpeg 是否可用
- ✓ 输出目录是否创建

### 方法 2: 完整测试

```bash
npm run test:video-formats
```

或者：

```bash
npx ts-node scripts/test-three-formats.ts
```

这会执行完整的转换测试，包括：
1. 获取视频信息
2. 转换为 MJPEG 格式
3. 转换为 AVI 格式
4. 转换为 H.264 格式
5. 显示对比结果


## 🔍 验证要点

### MJPEG 格式
- ✓ 文件扩展名为 `.mjpeg`
- ✓ 使用 mjpeg 编码器
- ✓ 像素格式为 yuvj420p
- ✓ 文件大小较大，质量高

### AVI 格式
- ✓ 文件扩展名为 `.avi`
- ✓ 容器内使用 mjpeg 编码
- ✓ 像素格式为 yuvj420p
- ✓ 兼容性好

### H.264 格式
- ✓ 文件扩展名为 `.h264`
- ✓ 使用 libx264 编码器
- ✓ 输出为原始流格式
- ✓ 文件大小最小，压缩率高

## 📈 性能基准

### 预期转换时间（参考）

| 视频大小 | MJPEG | AVI | H.264 |
|----------|-------|-----|-------|
| 10MB | ~2s | ~2s | ~3s |
| 50MB | ~8s | ~9s | ~12s |
| 100MB | ~15s | ~17s | ~25s |

*实际时间取决于系统性能和视频复杂度*

### 预期文件大小比例

相同质量设置下：
- MJPEG: 100% (基准)
- AVI: 110% (略大于 MJPEG)
- H.264: 50-60% (压缩率最高)

## ⚠️ 常见问题

### 1. FFmpeg 未找到

**问题**: `FFmpeg not found in system PATH`

**解决**:
```bash
# 检查 FFmpeg 是否安装
ffmpeg -version

# Windows: 添加到 PATH
# Linux/macOS: 重新安装
```

### 2. 转换失败

**问题**: `Error initializing the muxer`

**可能原因**:
- 输出目录不存在
- 磁盘空间不足
- 视频文件损坏

**解决**:
- 检查输出目录权限
- 确保有足够的磁盘空间
- 尝试其他测试视频

### 3. 临时文件错误

**问题**: `Invalid argument` 错误

**解决**: 已修复，临时文件现在使用正确的命名格式（如 `birds.tmp.mjpeg`）

### 4. 转换速度慢

**可能原因**:
- 视频文件过大
- 系统性能限制
- H.264 编码参数复杂

**建议**:
- 使用较小的测试视频（< 50MB）
- 调整质量参数
- 考虑使用硬件加速
