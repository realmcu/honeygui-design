# HoneyGUI Design 架构设计

## 项目概述

HoneyGUI Visual Designer 是一个 VS Code 扩展，提供可视化拖放界面来创建 HoneyGUI UI 界面。它能生成适用于嵌入式应用的 C 代码，并支持代码保护区功能。

**关键技术：**
- TypeScript 5.3
- React 18 + Zustand（用于 webview UI）
- VS Code 扩展 API
- Webpack 5（webview 构建）
- HML（HoneyGUI 标记语言 - 自定义类 XML 格式）

## 状态管理

### 扩展端
- `HmlController` - 管理HML文档状态和版本
- `SaveManager` - 管理保存事务和状态
- `CollaborationService` - 管理协同会话状态

### Webview 端（Zustand）
- 组件增删改查操作
- 选择状态管理
- 撤销/重做（命令模式，50 步历史）
- 画布状态（缩放、平移、网格设置）
- 拖拽状态
- 项目配置

## 通信流程

### 扩展 → Webview
```typescript
{ command: 'loadHml', components: [], projectConfig: {}, designerConfig: {} }
{ command: 'showMessage', text: '', type: 'info' | 'warning' | 'error' }
{ command: 'codegenProgress', current: number, total: number }
{ command: 'codegenComplete', success: boolean, files: [] }
{ command: 'collaboration:update', components: [] }
```

### Webview → 扩展
```typescript
{ command: 'save', components: [] }
{ command: 'codegen', language: 'c' }
{ command: 'addComponent', component: {} }
{ command: 'removeComponent', id: '' }
{ command: 'updateComponent', id: '', updates: {} }
{ command: 'importAssets', files: [] }
{ command: 'deleteAsset', path: '' }
{ command: 'ready' }
```

## 关键实现

### 撤销/重做系统
位于 `src/webview/utils/undoRedo.ts`，实现命令模式：
- `AddComponentCommand` - 添加组件
- `DeleteComponentCommand` - 删除组件
- `MoveComponentCommand` - 移动组件
- `UpdatePropertyCommand` - 更新属性

### 设计器画布
位于 `src/webview/components/DesignerCanvas.tsx`：
- 从组件库拖放创建组件
- 从资源面板拖放图片
- 位置吸附到网格（默认 8px）
- 缩放（25% - 800%）
- 多级组件嵌套
- 多选和框选

### 键盘快捷键
位于 `src/webview/utils/keyboardShortcuts.ts`：
- `Ctrl+S` - 保存
- `Ctrl+Z` - 撤销
- `Ctrl+Shift+Z` / `Ctrl+Y` - 重做
- `Delete` - 删除选中
- `Ctrl+D` - 复制
- `方向键` - 微调（1px）
- `Shift+方向键` - 快速移动（10px）

### 编译仿真
- `BuildCore` - 不依赖VSCode的核心编译逻辑
- `SimulationRunner` - 在 VSCode Terminal 中运行仿真器
- `EnvironmentChecker` - 检查SDK、SCons、GCC

## 扩展命令

### 项目管理
- `honeygui.newProject` / `honeygui.createProject` - 创建新项目
- `honeygui.openProject` - 打开项目

### 设计器
- `honeygui.openDesigner` - 打开设计器
- `honeygui.openInTextEditor` - 用文本编辑器打开
- `honeygui.createNewHmlInWorkspace` - 新建设计

### 代码生成与仿真
- `honeygui.codegen` - 生成代码
- `honeygui.simulation` - 启动编译仿真
- `honeygui.simulation.stop` - 停止仿真
- `honeygui.preview` - 预览（开发中）

### 协同开发
- `honeygui.collaboration.startHost` - 开启协同（主机）
- `honeygui.collaboration.joinSession` - 加入协同（访客）
- `honeygui.collaboration.stop` - 停止协同

## 构建系统

使用双构建系统：
- **扩展**：TypeScript 编译器 (tsc) 输出到 `out/`
- **Webview**：Webpack 5 输出到 `out/designer/webview/`

### 构建命令
```bash
npm run compile           # 编译扩展
npm run watch             # 监听扩展变化
npm run build:webview     # 构建 webview（生产）
npm run watch:webview     # 监听 webview 变化
npm run vscode:prepublish # 发布前完整构建
```

### 开发流程
```bash
# 终端1：监听扩展代码
npm run watch

# 终端2：监听 webview 代码
npm run watch:webview

# 按 F5 启动调试
```

## HML 文件格式

```xml
<?xml version="1.0" encoding="UTF-8"?>
<hone:HoneyGUI xmlns:hone="http://www.honeygui.com">
    <HoneyGUI version="1.0">
        <hg_view id="main_view" x="0" y="0" w="480" h="272">
            <hg_button id="btn1" x="10" y="10" w="100" h="40" text="Click"/>
        </hg_view>
    </HoneyGUI>
</hone:HoneyGUI>
```

## 已知限制

1. **预览功能**：快速预览尚未实现，当前仅支持完整编译仿真
2. **性能**：大型组件树（100+组件）可能影响性能
3. **协同开发**：仅同步HML结构，资源文件需手动同步
4. **平台支持**：主要在Windows和Linux测试，macOS支持有限
