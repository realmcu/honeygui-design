# HoneyGUI Design 架构设计

## 项目概述

HoneyGUI Visual Designer 是一个 VS Code 扩展，提供可视化拖放界面来创建 HoneyGUI（HoneyGUI 标记语言）UI 界面。它能生成适用于嵌入式应用的 C++/C 代码，并支持代码保护区功能。

**关键技术：**
- TypeScript 5.3
- React 18 + ReactDOM + Zustand（用于 webview UI）
- VS Code 扩展 API
- Webpack 5（webview 构建）
- HML（HoneyGUI 标记语言 - 自定义类 XML 格式）

### 状态管理

**扩展端**：
- `HmlController` - 管理HML文档状态和版本
- `SaveManager` - 管理保存事务和状态
- `CollaborationService` - 管理协同会话状态
- `StatusBarManager` - 管理状态栏显示

**Webview 端**：React + Zustand 用于 UI 状态管理

**Zustand store 主要功能：**
- 组件增删改查操作（`addComponent`, `updateComponent`, `removeComponent`）
- 选择状态（`selectComponent`, `setSelectedComponents`, `clearSelection`）
- 撤销/重做（命令模式，50 步历史）
- 画布状态（缩放、平移、网格设置）
- 拖拽状态（`startDrag`, `drag`, `endDrag`）
- VSCode通信（`saveToFile`, `generateCode`）
- 项目配置（`setProjectConfig`, `initializeWithProjectConfig`）

**状态同步机制：**
- Webview修改 → MessageHandler → HmlController → 文件系统
- 协同模式：Webview修改 → CollaborationService → WebSocket → 其他客户端
- 文件变更 → HmlController → Webview更新

### 通信流程

#### 1. 扩展 → Webview

通过 `webview.postMessage()` 发送带类型的消息：

```typescript
// 加载HML内容
{ 
  command: 'loadHml', 
  components: Component[], 
  projectConfig: ProjectConfig,
  designerConfig: { canvasBackgroundColor: string }
}

// 显示消息
{ command: 'showMessage', text: string, type: 'info' | 'warning' | 'error' }

// 代码生成进度
{ command: 'codegenProgress', current: number, total: number }

// 代码生成完成
{ command: 'codegenComplete', success: boolean, files: string[] }

// 协同更新
{ command: 'collaboration:update', components: Component[] }
```

#### 2. Webview → 扩展

通过 `vscode.postMessage()` 发送消息：

```typescript
// 保存操作
{ command: 'save', components: Component[] }

// 代码生成
{ command: 'codegen', language: 'c' | 'cpp' }

// 组件操作
{ command: 'addComponent', component: Component }
{ command: 'removeComponent', id: string }
{ command: 'updateComponent', id: string, updates: Partial<Component> }

// 资源操作
{ command: 'importAssets', files: FileList }
{ command: 'deleteAsset', path: string }

// 通知
{ command: 'notify', message: string, type: 'info' | 'warning' | 'error' }

// 准备就绪
{ command: 'ready' }
```

#### 3. 协同通信（WebSocket）

主机和访客之间通过WebSocket实时同步：

```typescript
// 组件更新
{ 
  type: 'component:update', 
  id: string, 
  updates: Partial<Component> 
}

// 组件添加
{ 
  type: 'component:add', 
  component: Component 
}

// 组件删除
{ 
  type: 'component:remove', 
  id: string 
}

// 完整同步
{ 
  type: 'sync:full', 
  components: Component[] 
}
```

## 关键实现细节

### 撤销/重做系统

实现命令模式，位于 `src/webview/utils/undoRedo.ts`：

**命令类型**：
- `AddComponentCommand` - 添加组件
- `DeleteComponentCommand` - 删除组件
- `MoveComponentCommand` - 移动组件
- `UpdatePropertyCommand` - 更新属性

**特性**：
- 由 Zustand store 管理
- 50 步历史记录
- 支持批量撤销/重做
- 提供命令标签用于UI显示

### 设计器画布

位于 `src/webview/components/DesignerCanvas.tsx`

**支持功能**：
- 从组件库拖放创建组件
- 从资源面板拖放图片创建图片组件
- 从文件系统拖放多个图片
- 带视觉反馈的组件选择（高亮边框）
- 位置吸附到网格（默认 8px，可配置）
- 缩放（25% - 800%）
- 多级组件嵌套
- 多选和框选
- 拖拽移动和调整大小

**拖拽处理**：
- `findDropTarget()` - 查找拖放目标容器
- `createImageComponentAtPosition()` - 在指定位置创建图片组件
- 自动偏移多个组件位置（每个偏移20px）

### 键盘快捷键

通过 React hook (`useKeyboardShortcuts`) 实现，位于 `src/webview/utils/keyboardShortcuts.ts`：

**快捷键列表**：
- `Ctrl+S` - 保存
- `Ctrl+Z` - 撤销
- `Ctrl+Shift+Z` / `Ctrl+Y` - 重做
- `Delete` - 删除选中
- `Ctrl+D` - 复制
- `方向键` - 微调（1px）
- `Shift+方向键` - 快速移动（10px）
- `Esc` - 取消选择

**实现细节**：
- Hook 接收 `store` 参数以访问所有状态管理方法
- 使用 `useCallback` 避免不必要的重新渲染
- 事件监听器在 `useEffect` 中注册和清理
- 支持多选组件的移动操作
- 拦截浏览器默认行为（如 Ctrl+S 不打开保存对话框）

### HML文件处理

**HmlController** (`src/hml/HmlController.ts`)：
- 协调解析器和序列化器
- 管理文档版本和保存事务
- 支持远程更新（协同模式）
- 防止竞态条件

**HmlParser** (`src/hml/HmlParser.ts`)：
- 使用 `fast-xml-parser` 解析XML
- 将HML转换为组件树
- 处理组件属性和嵌套关系

**HmlSerializer** (`src/hml/HmlSerializer.ts`)：
- 将组件树序列化为HML格式
- 保持XML格式规范
- 支持命名空间和schema

### 代码生成

**HoneyGuiCCodeGenerator** (`src/codegen/honeygui/HoneyGuiCCodeGenerator.ts`)：
- 生成调用HoneyGUI API的C代码
- 支持代码保护区（PROTECTED START/END）
- 生成头文件、实现文件和回调文件
- 自动创建目录结构

**BatchCodeGenerator** (`src/services/BatchCodeGenerator.ts`)：
- 扫描项目中所有HML文件
- 批量生成代码
- 进度报告和错误收集
- 独立于VSCode UI，可测试

### 编译仿真

**BuildCore** (`src/simulation/BuildCore.ts`)：
- 不依赖VSCode的核心编译逻辑
- 准备编译目录（`.honeygui-build/`）
- 复制SDK和生成的代码
- 生成SCons脚本
- 执行编译

**SimulationRunner** (`src/simulation/SimulationRunner.ts`)：
- 管理仿真进程生命周期
- 处理进程输出和错误
- 支持停止和重启
- 跨平台支持（Windows/Linux）

**EnvironmentChecker** (`src/simulation/EnvironmentChecker.ts`)：
- 检查SDK路径（三级优先级）
- 验证SCons和GCC工具链
- 提供详细的错误提示

### 协同开发

**CollaborationService** (`src/core/CollaborationService.ts`)：
- WebSocket服务器（主机模式）
- WebSocket客户端（访客模式）
- 消息广播和同步
- 防止消息回环
- 连接状态管理

**消息处理**：
- MessageHandler拦截Webview操作
- 协同模式下广播而非直接执行
- HmlController支持远程更新
- 主机负责文件保存

### 资源管理

**AssetManager** (`src/designer/AssetManager.ts`)：
- 扫描项目资源目录
- 支持图片预览
- 处理资源导入和删除
- 自动删除关联组件
- 路径解析（webview URI vs 文件系统路径）

### 扩展命令

**CommandManager** (`src/core/CommandManager.ts`) 注册的主要命令：

**项目管理**：
- `honeygui.newProject` / `honeygui.createProject` - 创建新项目
- `honeygui.openProject` - 打开项目
- `honeygui.openRecent` - 打开最近项目

**设计器**：
- `honeygui.openDesigner` - 打开设计器
- `honeygui.openInTextEditor` - 用文本编辑器打开
- `honeygui.openInDesigner` - 用设计器打开
- `honeygui.createNewHmlInWorkspace` - 新建设计

**代码生成**：
- `honeygui.codegen` - 生成代码

**仿真**：
- `honeygui.simulation` - 启动编译仿真
- `honeygui.simulation.stop` - 停止仿真
- `honeygui.preview` - 预览（开发中）

**协同开发**：
- `honeygui.collaboration.startHost` - 开启协同（主机）
- `honeygui.collaboration.joinSession` - 加入协同（访客）
- `honeygui.collaboration.stop` - 停止协同

**其他**：
- `honeygui.openResourceManager` - 打开资源管理器
- `honeygui.openDocs` - 打开文档
- `honeygui.showLogs` - 显示日志
- `honeygui.clearCache` - 清理缓存

## VS Code 配置设置

扩展提供以下设置（前缀：`honeygui.`）：

### 代码生成设置
- `codegen.language`：目标语言（cpp/c），默认：cpp
- `codegen.outputDir`：代码生成输出目录，默认："src/ui"
- `codegen.outputPath`：代码输出路径，默认："src"
- `codegen.cppVersion`：C++ 版本，默认："c++17"
- `codegen.enableDebugInfo`：在生成代码中包含调试信息，默认：true

### 预览设置
- `preview.runnerPath`：HoneyGUI Runner 可执行文件路径，默认：""
- `preview.autoDownload`：缺失时自动下载 runner，默认：true
- `preview.timeoutMs`：预览超时（毫秒），默认：10000
- `preview.autoReload`：变更时自动重载预览，默认：true

### UI 设置
- `ui.gridSize`：吸附网格大小（像素），默认：8
- `ui.snapToGrid`：启用网格吸附，默认：true
- `hml.outputDir`：HML 文件输出目录，默认："ui"

### 其他设置
- `telemetry.enabled`：启用遥测收集，默认：true

## HML 文件格式

HML 是自定义类 XML 格式：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<hone:HoneyGUI xmlns:hone="http://www.honeygui.com"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://www.honeygui.com honeygui.xsd">
    <HoneyGUI version="1.0">
        <Plugin name="com.honeygui.designer" version="1.1.0"/>
        <Window width="800" height="600" title="Main Window">
            <Button id="button1" x="100" y="50" width="120" height="40" text="Click Me"/>
            <Label id="label1" x="100" y="120" width="200" height="30" text="Hello HoneyGUI"/>
        </Window>
    </HoneyGUI>
</hone:HoneyGUI>
```

## 已知限制

### 1. 包大小
- React webview包较大（约791 KiB）
- 建议：生产环境使用代码分割和懒加载

### 2. 预览功能
- 快速预览功能尚未实现
- 当前仅支持完整的编译仿真流程
- 计划：实现HML解释器直接渲染

### 3. 性能优化
- 大型组件树（100+组件）可能影响性能
- 未广泛使用 React.memo 优化
- 建议：对复杂组件使用虚拟化

### 4. 协同开发
- 仅支持HML结构同步，资源文件需手动同步
- 采用"主机为真理之源"策略，无冲突解决机制
- 断线后需手动重新加入会话

### 5. 测试覆盖
- 单元测试覆盖率较低
- 主要依赖E2E测试和手动测试
- 需要完整的HoneyGUI SDK环境

### 6. 平台支持
- 编译仿真主要在Windows和Linux测试
- macOS支持有限
- 需要图形环境（X11或Windows）运行仿真

### 7. 错误处理
- 部分错误提示不够友好
- 网络错误处理需要改进
- 需要更好的用户引导
