# 列表组件设计文档

## 概述

列表组件 (`hg_list`) 是一个特殊的容器组件，用于展示可滚动的列表项。基于 HoneyGUI SDK 的 `gui_list` API 实现。

## 设计目标

在可视化设计器中提供直观的列表编辑体验，同时符合 HoneyGUI 的列表实现机制。

## 交互方案

### 方案演进

#### 方案A：单模板 + 数据绑定
- 只编辑一个模板，运行时自动复制 N 次
- 符合 HoneyGUI 的 `note_design` 回调机制
- **缺点**：无法为每个列表项设计不同内容

#### 方案B：真实子组件列表
- 每个列表项都是独立的子组件
- 可以单独编辑每一项
- **缺点**：代码生成复杂

#### 最终方案：真实子组件列表（采用）
- 每个列表项都是独立的子组件
- 所有列表项同时显示，可以直接编辑任意一项
- 自动管理列表项的数量和位置

## 交互流程

### 1. 创建列表
1. 从组件库拖拽"列表"组件到画布
2. 自动生成 5 个列表项子组件（默认值）
3. 列表项ID格式：`{listId}_item_1`, `{listId}_item_2`, ...

### 2. 编辑列表项
1. 在组件树或画布中点击任意列表项
2. 该列表项被选中，显示选中边框
3. 可以直接拖入子组件到该列表项
4. 所有列表项同时可见，可以随时切换编辑

### 3. 切换编辑项
1. 点击组件树或画布中的其他列表项
2. 选中状态切换到新的列表项
3. 所有列表项保持可见状态

### 4. 调整列表项数量
1. 选中列表组件（不是列表项）
2. 修改属性面板的"项数量"为 3
3. 自动删除多余的列表项（列表项 4、5）
4. 如果增加数量，自动创建新的列表项

### 5. 调整布局参数
修改以下属性会自动更新所有列表项的位置：
- **项长度** (noteLength)：每个列表项的长度
- **项间距** (space)：列表项之间的间距
- **方向** (direction)：VERTICAL（垂直堆叠）或 HORIZONTAL（水平排列）

## 组件属性

### 数据属性
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| noteNum | number | 5 | 列表项数量 |
| noteLength | number | 100 | 每个列表项的长度（垂直时为高度，水平时为宽度） |
| space | number | 0 | 列表项之间的间距 |

### 样式属性
| 属性 | 类型 | 默认值 | 可选值 |
|------|------|--------|--------|
| direction | select | VERTICAL | VERTICAL, HORIZONTAL |
| style | select | LIST_CLASSIC | LIST_CLASSIC, LIST_CIRCLE, LIST_ZOOM, LIST_CARD, LIST_FADE, LIST_FAN, LIST_HELIX, LIST_CURL |

### 行为属性
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| autoAlign | boolean | true | 停止时自动对齐到列表项 |
| inertia | boolean | true | 释放后的惯性滚动效果 |
| loop | boolean | false | 循环滚动（首尾相连） |
| createBar | boolean | false | 显示滚动条 |

## 列表样式说明

基于 HoneyGUI SDK 的 `LIST_STYLE` 枚举：

| 样式 | 说明 | 适用场景 |
|------|------|----------|
| LIST_CLASSIC | 经典列表 | 标准的垂直/水平滚动列表 |
| LIST_CIRCLE | 圆形列表 | 列表项沿圆形路径排列 |
| LIST_ZOOM | 缩放中心列表 | 中心项放大，两侧缩小 |
| LIST_CARD | 卡片堆叠 | 类似卡片堆叠效果 |
| LIST_FADE | 淡入淡出 | 列表项淡入淡出效果 |
| LIST_FAN | 扇形旋转 | 列表项像扇子一样旋转 |
| LIST_HELIX | 螺旋旋转 | 列表项螺旋旋转 |
| LIST_CURL | 卷曲旋转 | 列表项卷曲旋转 |

## 实现细节

### 自动生成子组件

**触发时机**：
- 列表组件首次渲染
- 修改 `noteNum` 属性
- 修改 `noteLength`、`space`、`direction` 属性

**生成逻辑**：
```typescript
for (let i = 0; i < noteNum; i++) {
  const itemId = `${listId}_item_${i + 1}`;
  const offset = i * (noteLength + space);
  
  // 创建列表项
  const newItem: Component = {
    id: itemId,
    type: 'hg_view',
    name: `列表项 ${i + 1}`,
    position: {
      x: isVertical ? 0 : offset,
      y: isVertical ? offset : 0,
      width: isVertical ? listWidth : noteLength,
      height: isVertical ? noteLength : listHeight,
    },
    parent: listId,
    // ...
  };
}
```

### 渲染逻辑

**在 DesignerCanvas 中**：
```typescript
if (isContainer) {
  // 正常渲染所有子组件（包括列表项）
  const children = component.children?.map((childId) => {
    const child = componentList.find((c) => c.id === childId);
    return child ? renderComponent(child, componentList) : null;
  });
  
  return <Widget component={component} style={style} handlers={handlers}>{children}</Widget>;
}
```

### 容器判断

列表组件被识别为容器：
```typescript
const isContainer = ['hg_view', 'hg_window', 'hg_screen', 'hg_canvas', 'hg_list'].includes(component.type);
```

```xml
  </hg_view>
  <!-- ... 其他列表项 -->
</hg_list>
```

### C 代码输出
```c
// 列表项设计回调
static void list1_note_design(gui_obj_t *obj, void *param) {
    gui_list_note_t *note = (gui_list_note_t *)obj;
    int index = note->index;
    
    // 根据 index 创建不同的列表项内容
    switch (index) {
        case 0: // 列表项 1
            gui_button_create(obj, "btn1", 10, 10, 80, 30);
            break;
        case 1: // 列表项 2
            gui_button_create(obj, "btn2", 10, 10, 80, 30);
            break;
        // ...
    }
}

// 创建列表
gui_list_t *list1 = gui_list_create(
    parent, "list1", 0, 0, 300, 400,
    100,        // noteLength
    0,          // space
    VERTICAL,   // direction
    list1_note_design,
    NULL,
    false       // createBar
);
gui_list_set_style(list1, LIST_CLASSIC);
gui_list_set_note_num(list1, 5);
gui_list_set_auto_align(list1, true);
gui_list_set_inertia(list1, true);
```

## 用户体验优化

### 视觉反馈
- ✅ 选中的列表项：高亮边框（蓝色）
- ✅ 所有列表项同时可见，可直接编辑
- ✅ 列表项显示边框，便于区分

### 操作提示
- 首次创建列表时，显示提示："列表已创建 5 个列表项，可在组件树中查看"
- 修改列表项数量时，显示确认提示（如果会删除已有内容）

## 限制与注意事项

1. **列表项类型固定**：自动生成的列表项类型为 `hg_view`
2. **ID 命名规则**：列表项 ID 必须遵循 `{listId}_item_{index}` 格式
3. **不支持手动添加子组件**：列表的子组件由系统自动管理，不能手动拖入
4. **循环滚动限制**：只在列表总长度大于容器尺寸时有效
5. **卡片样式限制**：LIST_CARD 样式不支持循环滚动
