# 批量代码生成架构

## 架构设计

### 职责分离

**BatchCodeGenerator 服务类** (`src/services/BatchCodeGenerator.ts`)
- 负责扫描项目中的 HML 文件
- 执行批量代码生成逻辑
- 收集和报告生成结果
- 独立于 VSCode UI 层，可测试

**DesignerPanel** (`src/designer/DesignerPanel.ts`)
- 负责 UI 交互和用户反馈
- 调用 BatchCodeGenerator 服务
- 显示进度和结果

### 核心改进

1. **消除 Workspace 依赖**
   - 使用 `ProjectUtils.findProjectRoot()` 基于 project.json 定位项目
   - 支持在任意文件上下文中执行批量生成

2. **错误处理优化**
   - 收集所有错误而不是遇到第一个就停止
   - 提供详细的错误报告（哪个设计稿失败及原因）
   - 部分成功时也能看到成功数量

3. **可测试性**
   - 业务逻辑与 VSCode API 解耦
   - 可以独立测试 BatchCodeGenerator

4. **可扩展性**
   - 清晰的接口定义（BatchGenerationResult, BatchGenerationProgress）
   - 易于添加新功能（如取消操作、过滤文件等）

## API

### BatchCodeGenerator

```typescript
class BatchCodeGenerator {
    // 扫描项目中的所有 HML 文件
    scanHmlFiles(projectRoot: string): Promise<string[]>
    
    // 批量生成代码
    generateAll(
        projectRoot: string,
        onProgress?: (progress: BatchGenerationProgress) => void
    ): Promise<BatchGenerationResult>
}
```

### 数据结构

```typescript
interface BatchGenerationResult {
    success: boolean;           // 是否全部成功
    successCount: number;       // 成功数量
    totalFiles: number;         // 生成的文件总数
    errors: Array<{             // 错误列表
        designName: string;
        error: string;
    }>;
}

interface BatchGenerationProgress {
    current: number;            // 当前进度
    total: number;              // 总数
    designName: string;         // 当前处理的设计稿名称
}
```

## 使用示例

```typescript
const generator = new BatchCodeGenerator();
const projectRoot = ProjectUtils.findProjectRoot(filePath);

const result = await generator.generateAll(projectRoot, (progress) => {
    console.log(`${progress.current}/${progress.total}: ${progress.designName}`);
});

if (result.success) {
    console.log(`成功生成 ${result.successCount} 个设计稿`);
} else {
    console.log(`部分失败: ${result.errors.length} 个错误`);
}
```
