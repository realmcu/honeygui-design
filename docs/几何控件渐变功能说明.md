# 几何控件渐变功能说明

## 功能概述

设计器现已全面支持为几何控件（弧形、矩形、圆形）配置渐变效果，基于 HoneyGUI SDK 的完整渐变 API 实现。

## 支持的控件

- **hg_arc（弧形）**：✅ 角度渐变（沿弧线方向）
- **hg_circle（圆形）**：✅ 径向渐变 + 角度渐变
- **hg_rect（矩形）**：✅ 线性渐变（4个方向）

## 使用方法

### 1. 启用渐变

在属性面板的"样式"分组中，勾选"启用渐变"选项。

### 2. 选择渐变类型/方向

根据控件类型，选择合适的渐变类型或方向：

#### 圆形（hg_circle）
- **径向渐变（radial）**：颜色从圆心向边缘辐射变化
- **角度渐变（angular）**：颜色沿圆周方向变化
  - 需要配置起始角度和结束角度

#### 矩形（hg_rect）
- **水平渐变（horizontal）**：从左到右
- **垂直渐变（vertical）**：从上到下
- **对角渐变（diagonal_tl_br）**：从左上到右下
- **对角渐变（diagonal_tr_bl）**：从右上到左下

#### 弧形（hg_arc）
- 自动使用角度渐变，沿弧线方向变化
- 渐变范围与弧形的起始/结束角度一致

### 3. 配置渐变色标

启用渐变后，会出现"渐变设置"分组：

1. 点击"+ 添加色标"按钮添加渐变色标
2. 每个色标包含两个属性：
   - **位置 (0-1)**：色标在渐变中的位置，0 表示起点，1 表示终点
   - **颜色**：该位置的颜色值（支持颜色选择器和手动输入 #RRGGBB）

3. 至少需要 2 个色标才能形成渐变效果
4. 可以添加多个色标创建复杂的渐变效果
5. 点击色标右侧的"✕"按钮可以删除该色标

## 渐变类型详解

### 弧形（hg_arc）- 角度渐变

弧形使用角度渐变，颜色沿着弧线方向变化：
- 渐变起始角度和结束角度默认与弧形的起始角度、结束角度一致
- 色标位置 0 对应起始角度，位置 1 对应结束角度
- **自动补偿**：对于完整圆弧（如 270-270 或 0-360），SDK 会自动将渐变角度 +1° 以触发圆角封端绘制

**示例配置：**
```
弧形属性：
- 起始角度：270°
- 结束角度：270°（完整圆弧）
- 启用渐变：是

渐变色标：
- 位置 0.0 → 红色 (#FF0000)
- 位置 0.5 → 黄色 (#FFFF00)
- 位置 1.0 → 绿色 (#00FF00)
```

生成的代码：
```c
arc1 = gui_arc_create(parent, "arc1", 120, 120, 80, 270, 270, 8, gui_rgb(255, 0, 0));
// 设置角度渐变（SDK 自动补偿为 270-631，触发封端绘制）
gui_arc_set_angular_gradient(arc1, 270, 270);
gui_arc_add_gradient_stop(arc1, 0.0f, gui_rgba(255, 0, 0, 255));
gui_arc_add_gradient_stop(arc1, 0.5f, gui_rgba(255, 255, 0, 255));
gui_arc_add_gradient_stop(arc1, 1.0f, gui_rgba(0, 255, 0, 255));
```

**Apple Watch 活动环效果**：
- 设置弧形角度为 270-270（从顶部开始的完整圆）
- 设置渐变角度也为 270-270
- SDK 会自动补偿为 270-631，在终点绘制圆角封端
- 实现类似 Apple Watch 活动环的平滑过渡效果

### 圆形（hg_circle）

#### 径向渐变（radial）

颜色从圆心向边缘辐射变化：
- 色标位置 0 对应圆心
- 色标位置 1 对应边缘

**示例配置：**
```
圆形属性：
- 半径：80
- 渐变类型：径向（radial）
- 启用渐变：是

渐变色标：
- 位置 0.0 → 白色 (#FFFFFF)
- 位置 1.0 → 蓝色 (#0000FF)
```

生成的代码：
```c
circle1 = gui_circle_create(parent, "circle1", 120, 120, 80, gui_rgb(255, 255, 255));
// 设置径向渐变
gui_circle_set_radial_gradient(circle1);
gui_circle_add_gradient_stop(circle1, 0.0f, gui_rgba(255, 255, 255, 255));
gui_circle_add_gradient_stop(circle1, 1.0f, gui_rgba(0, 0, 255, 255));
```

#### 角度渐变（angular）

颜色沿圆周方向变化：
- 需要配置起始角度和结束角度
- 色标位置 0 对应起始角度，位置 1 对应结束角度

**示例配置：**
```
圆形属性：
- 半径：80
- 渐变类型：角度（angular）
- 起始角度：0°
- 结束角度：360°
- 启用渐变：是

渐变色标：
- 位置 0.0 → 红色 (#FF0000)
- 位置 0.5 → 绿色 (#00FF00)
- 位置 1.0 → 蓝色 (#0000FF)
```

生成的代码：
```c
circle2 = gui_circle_create(parent, "circle2", 120, 120, 80, gui_rgb(255, 0, 0));
// 设置角度渐变
gui_circle_set_angular_gradient(circle2, 0, 360);
gui_circle_add_gradient_stop(circle2, 0.0f, gui_rgba(255, 0, 0, 255));
gui_circle_add_gradient_stop(circle2, 0.5f, gui_rgba(0, 255, 0, 255));
gui_circle_add_gradient_stop(circle2, 1.0f, gui_rgba(0, 0, 255, 255));
```

### 矩形（hg_rect）- 线性渐变

矩形支持 4 个方向的线性渐变：

#### 水平渐变（horizontal）
从左到右变化

#### 垂直渐变（vertical）
从上到下变化

#### 对角渐变（diagonal_tl_br）
从左上角到右下角变化

#### 对角渐变（diagonal_tr_bl）
从右上角到左下角变化

**示例配置：**
```
矩形属性：
- 宽度：200
- 高度：100
- 渐变方向：水平（horizontal）
- 启用渐变：是

渐变色标：
- 位置 0.0 → 红色 (#FF0000)
- 位置 1.0 → 蓝色 (#0000FF)
```

生成的代码：
```c
rect1 = gui_rect_create(parent, "rect1", 100, 100, 200, 100, 0, gui_rgb(255, 0, 0));
// 设置线性渐变
gui_rect_set_linear_gradient(rect1, RECT_GRADIENT_HORIZONTAL);
gui_rect_add_gradient_stop(rect1, 0.0f, gui_rgba(255, 0, 0, 255));
gui_rect_add_gradient_stop(rect1, 1.0f, gui_rgba(0, 0, 255, 255));
```

## 测试指南

### 测试超过 360° 的渐变效果

1. **在设计器中创建测试弧形**：
   - 添加一个弧形组件
   - 设置弧形角度：0° 到 360°（完整圆弧）
   - 启用渐变
   - 设置渐变角度：0° 到 361°（超过 360°）
   - 添加两个色标：
     - 位置 0.0：红色 (#FF0000)
     - 位置 1.0：蓝色 (#0000FF)

2. **生成的代码示例**：
```c
// 创建完整圆弧
gui_arc_t *arc = gui_arc_create(parent, "test_arc",
                                 240, 240, 100,  // 中心 x, y, 半径
                                 0, 360,         // 弧形角度：0° 到 360°
                                 20,             // 线宽
                                 gui_rgb(255, 0, 0));

// 设置渐变角度超过 360°
gui_arc_set_angular_gradient(arc, 0, 361);  // 渐变角度：0° 到 361°
gui_arc_add_gradient_stop(arc, 0.0f, gui_rgba(255, 0, 0, 255));  // 红色
gui_arc_add_gradient_stop(arc, 1.0f, gui_rgba(0, 0, 255, 255));  // 蓝色
```

3. **预期效果**：
   - 弧形从 0° 位置开始显示红色
   - 沿着圆周方向逐渐过渡到蓝色
   - 在 360° 位置（即起点）附近，颜色平滑过渡回红色
   - **关键**：结束位置的封端也显示渐变色，而不是单一的蓝色

4. **对比测试**：
   - 不超过 360°：设置渐变角度为 0° 到 360°
     - 结束封端显示单一的蓝色
   - 超过 360°：设置渐变角度为 0° 到 361°
     - 结束封端显示渐变色，实现平滑过渡

## 技术细节

### 数据存储

渐变配置存储在组件的属性中：

```typescript
// 渐变开关
component.style.useGradient = true;

// 圆形：渐变类型
component.style.gradientType = 'radial' | 'angular';

// 矩形：渐变方向
component.style.gradientDirection = 'horizontal' | 'vertical' | 'diagonal_tl_br' | 'diagonal_tr_bl';

// 圆形角度渐变：起始/结束角度
component.data.gradientStartAngle = 0;
component.data.gradientEndAngle = 360;

// 渐变色标
component.data.gradientStops = [
  { position: 0.0, color: '#FF0000' },
  { position: 1.0, color: '#0000FF' }
];
```

### 代码生成

代码生成器会检查 `style.useGradient` 和 `data.gradientStops`，如果启用渐变且有至少 2 个色标，则生成相应的渐变设置代码。

#### 弧形代码生成
```c
gui_arc_set_angular_gradient(arc, startAngle, endAngle);
gui_arc_add_gradient_stop(arc, position, color);
```

#### 圆形代码生成
```c
// 径向渐变
gui_circle_set_radial_gradient(circle);
gui_circle_add_gradient_stop(circle, position, color);

// 角度渐变
gui_circle_set_angular_gradient(circle, startAngle, endAngle);
gui_circle_add_gradient_stop(circle, position, color);
```

#### 矩形代码生成
```c
gui_rect_set_linear_gradient(rect, direction);
gui_rect_add_gradient_stop(rect, position, color);
```

## 注意事项

1. **最少色标数量**：至少需要 2 个色标才能形成渐变效果
2. **位置范围**：色标位置必须在 0-1 之间
3. **颜色格式**：目前仅支持 #RRGGBB 格式（6 位十六进制）
4. **SDK 版本**：确保使用支持渐变功能的 HoneyGUI SDK 版本
5. **性能考虑**：渐变会增加渲染开销，建议在低性能设备上谨慎使用
6. **圆形角度渐变**：起始角度和结束角度的范围通常是 0-360 度
7. **矩形渐变方向**：选择合适的方向以达到预期的视觉效果

## SDK 底层实现

### 完整圆弧的自动补偿机制

**核心特性**：SDK 会自动检测完整圆弧并补偿渐变角度，无需用户手动设置。

**实现位置**：
1. `HoneyGUI/realgui/widget/gui_geometry/gui_arc.c` - `gui_arc_set_angular_gradient` 函数
2. `HoneyGUI/realgui/widget/gui_geometry/lite_geometry.c` - `draw_arc_df_aa_gradient` 函数

**自动补偿逻辑**：
```c
// 在 gui_arc_set_angular_gradient 中
void gui_arc_set_angular_gradient(gui_arc_t *this, float start_angle, float end_angle)
{
    // 计算原始角度跨度（归一化之前）
    float original_span = end_angle - start_angle;
    
    // 检测完整圆：跨度为 0 或接近 360°
    bool is_full_circle = (fabsf(original_span) < 1.0f) || 
                          (fabsf(fabsf(original_span) - 360.0f) < 1.0f);
    
    // 自动补偿：完整圆时自动 +1° 触发封端绘制
    if (is_full_circle)
    {
        end_angle = start_angle + 361.0f;
    }
    
    // 存储渐变角度
    gradient->angular_start = start_angle;
    gradient->angular_end = end_angle;
}
```

**封端绘制逻辑**：
```c
// 在 draw_arc_df_aa_gradient 中
// 检测渐变角度是否超过 360°
float gradient_span = gradient_end - gradient_start;
bool gradient_over_360 = (gradient_span > 360.0f);

// 在封端区域的颜色处理
if (!in_arc_range)
{
    // 当渐变超过 360° 时，结束封端使用渐变色
    if (gradient_over_360 && is_full_circle && dist_end_sq < dist_start_sq)
    {
        stroke_color = gradient_get_color_at_angle(gradient, angle);
    }
    else
    {
        // 正常情况：使用单一颜色
        stroke_color = (dist_start_sq < dist_end_sq) ? start_color : end_color;
    }
}
```

**用户体验优化**：
- 用户只需设置 `270-270` 或 `0-360`，SDK 自动处理封端
- 设计器端也实现了相同的自动补偿逻辑，保持一致性
- 无需手动设置 `270-631` 或 `0-361`，简化使用复杂度
