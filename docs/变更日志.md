## [2025-11-29] v1.2.1 - 编译架构重构

### 重构
- **BuildCore 抽离**：将编译核心逻辑从 BuildManager 中抽离
  - `BuildCore` - 不依赖 VSCode 的核心编译逻辑
  - `BuildManager` - VSCode 插件专用，继承 BuildCore
  - 支持在测试脚本和 CLI 工具中复用编译逻辑

### 新增
- ✅ `BuildCore.generateAutogenSConscript()` - 自动生成 autogen 目录的 SConscript
- ✅ `Logger` 接口 - 日志抽象，支持不同环境的日志输出

### 改进
- e2e 测试脚本现在使用 BuildCore，与插件共享相同的编译逻辑
- SConscript 生成逻辑统一，避免代码重复

### 架构
```
BuildCore (src/simulation/BuildCore.ts)
├── setupBuildDir()           # 准备编译目录
├── copyGeneratedCode()       # 拷贝代码 + 生成 SConscript
├── generateAutogenSConscript() # 生成 autogen/SConscript
├── compile()                 # 执行 scons 编译
└── getExecutablePath()       # 获取可执行文件路径
         │
         ▼
BuildManager extends BuildCore (src/simulation/BuildManager.ts)
├── VSCodeLogger 适配器       # VSCode OutputChannel 日志
├── updateGitignore()         # 更新 .gitignore
└── dispose()                 # 清理资源
```
